# story-webtoon-planner: 웹툰 연재 기획

웹툰의 **뼈대**를 세우는 스킬이다. 세계관·연재구조·시리즈 아크·훅 전략을 정해, 뒤따르는 상태 원장(`story-series-bible`)·캐릭터(`story-character-sheet`)·회차(`story-webtoon-episode`)가 흔들리지 않을 기획서를 만든다. 기획이 새면 회차가 갈수록 표류한다.

## 1. 개요

이 스킬은 **"무엇을, 어떤 구조로 연재할 것인가"**라는 기획 판단에 집중한다. 캔버스 가로폭·회차 컷 수·데뷔 경로 같은 **규격 사실은 여기서 정하지 않고** `story-webtoon-spec` 허브를 참조한다. 규격은 시기·정책에 따라 바뀌므로 한 곳(허브)에서만 관리한다.

## 2. 이 스킬이 하지 않는 것

- **규격 수치 단정 안 함** — 캔버스 가로폭·회차 세로 길이·컷 수·데뷔 경로 승격 기준은 `story-webtoon-spec`이 소유한다. 이 스킬은 그 값을 참조만 한다.
- **캐릭터 비주얼 시트 안 만듦** — 외형 8항목 표·일관성 앵커·중립 기본형은 `story-character-sheet` 소관.
- **회차 컷 분할·연출 안 함** — 컷 대본은 `story-webtoon-episode`, 식자는 `story-webtoon-lettering`.
- **파일 저장·상태 갱신 안 함** — 기획서 저장과 에피소드 현황표는 `story-series-bible`가 소유한다. 이 스킬은 기획 내용을 설계해 넘긴다.

## 3. 사전 확인 — 규격 먼저 고정

기획에 들어가기 전에 규격을 먼저 확인한다. 다음 두 파일을 `story-webtoon-spec` 허브에서 참조한다.

- `${CLAUDE_PLUGIN_ROOT}/skills/story-webtoon-spec/references/platform-debut-paths.md` — 타깃 플랫폼·데뷔 경로(네이버웹툰 **도전만화 → 베스트도전 → 정식연재 3단계** 등).
- `${CLAUDE_PLUGIN_ROOT}/skills/story-webtoon-spec/references/episode-volume.md` — 시즌1 화수 규모·연재 주기(아래 시리즈 아크 표의 화수 열을 이 값으로 채운다).

> ⚠️ 시리즈 아크의 화수는 임의로 "20화 완결"처럼 단정하지 않는다. 한국 정식연재 시즌1 규격은 이와 다를 수 있으므로 `episode-volume.md` 값(또는 `[미확인]` 플래그)을 그대로 반영한다.

## 4. 워크플로우

### Step 1 — 세계관 설계

배경·시대·핵심 룰·금기를 정리한다. "이 세계에서 특별히 다른 점"이 독자에게 진입 장벽이 아니라 후크가 되도록 한 문장으로 요약한다.

### Step 2 — 웹툰 네이티브 독자 판정 (2문항)

기획이 세로 스크롤 웹툰에 맞는지 두 물음으로 검증한다. 하나라도 "아니오"면 구조를 다시 본다.

1. **다음화 버튼을 누르는 이유가 매 화 분명한가?** — 무엇이 궁금해서 넘기는가.
2. **지하철에서 5분 남았을 때 이 웹툰을 클릭하는가?** — 짧은 틈에 소비되는 리듬인가.

### Step 3 — 연재구조 3분류 + 장르 매핑

어떤 구조로 연재할지 먼저 고른다. 구조가 회차 설계·훅 배치를 전부 좌우한다.

| 구조 | 특징 | 적합 장르 |
|------|------|-----------|
| **에피소드 독립형** | 각 화 완결. 언제 읽어도 OK | 일상·힐링 |
| **연속 서사형** | 전체 스토리가 이어짐 | 드라마·판타지 |
| **혼합형** | 화 단위 완결 + 관통하는 큰 흐름 | 대부분의 인기작 |

장르별 인물 트리·갈등 구조·훅 성향 프리셋은 `references/genre-presets.md`.

> ⚠️ **옴니버스(에피소드 독립형)도 관통 주제가 없으면 이탈이 빠르다.** "그냥 에피소드 모음"은 붙잡는 힘이 약하다 — Step 5의 관통 주제를 반드시 정한다.

### Step 4 — 시리즈 아크 설계

전체 연재를 구간으로 나눠 각 구간의 역할·핵심 사건·감정 톤을 배치한다. **화수 열은 `episode-volume.md` 값으로 채운다**(아래 숫자는 예시 골격일 뿐 단정 아님).

| 구간 | 화수 | 역할 | 핵심 사건 | 감정 톤 |
|------|------|------|-----------|---------|
| 도입부 | `[episode-volume.md 참조]` | 세계관·캐릭터 소개 | | |
| 전개부 | `[참조]` | 관계 형성, 갈등 시작 | | |
| 클라이맥스 | `[참조]` | 핵심 갈등 폭발 | | |
| 해결부 | `[참조]` | 해소, 성장 | | |

### Step 5 — 관통 주제 1문장

여러 구간을 지나도 흔들리지 않는 핵심 주제를 한 문장으로 못박는다.

> "이 웹툰이 결국 하고 싶은 말은 **___**이다."

### Step 6 — 훅 전략 + 레귤러 인원 상한

- **훅 유형 배치:** 1~3화는 캐릭터 훅 + 감정 훅으로 붙잡고, 중반부터 궁금증 훅을 점진적으로 심는다. 유료 전환 지점과 훅을 맞추는 것은 `story-webtoon-episode`의 `monetization-hooks.md` 참조 단계에서 정렬한다.
- **레귤러 인원 상한:** 매 화 등장하는 레귤러는 **최대 5명 이내**로 선언한다. 인원이 늘수록 화마다 일관성 유지 부담과 (웹툰화 시) 생성 크레딧이 함께 늘어난다 — 인원 상한은 비용 통제와 직결된다. 초과분은 게스트로 돌린다.

## 5. 출력 형식

기획 내용은 마스터 기획서 형식으로 정리해 `story-series-bible`에 넘겨 저장한다(이 스킬은 저장하지 않는다).

```markdown
# [작품제목] 기획서 (초안)

## 세계관
- 배경·시대·핵심 룰·금기

## 연재구조
- 분류: [에피소드 독립형 / 연속 서사형 / 혼합형]
- 장르: [ ] → genre-presets 적용

## 관통 주제
> 이 웹툰이 결국 하고 싶은 말은 ___이다.

## 시리즈 아크
| 구간 | 화수(spec 참조) | 역할 | 핵심 사건 | 감정 톤 |
| ... |

## 훅 전략
- 구간별 훅 유형·강도

## 레귤러 상한
- 레귤러 N명(≤5) / 나머지는 게스트
```

→ 저장·에피소드 현황표 생성은 `story-series-bible`로 넘긴다.

## 6. 주의사항

- **규격 복제 금지.** 캔버스·데뷔·화수 수치를 이 스킬 본문에 박지 않는다 — 허브 참조.
- **화수 환각 금지.** "20화 완결" 같은 임의 숫자를 사실처럼 쓰지 않는다.
- **레귤러 5명 상한을 넘기면** 일관성·비용이 함께 무너진다. 게스트 계층으로 분리한다.
- **독자 판정 2문항을 통과 못 하면** 소재는 좋아도 세로 스크롤 리듬에 안 맞는다 — 구조부터 다시 본다.

## 7. 관련 스킬

### After
- `story-series-bible` — 기획서 저장 + 에피소드 현황표 생성(상태 확립)
- `story-character-sheet` — 캐릭터 비주얼 시트·일관성 앵커
- `story-webtoon-episode` — 회차 플롯·컷 분할

- `moai-coworker:general-ai-slop-reviewer` — AI 슬롭 검수 (필수)
- `moai-writer:general-humanize-korean` — AI 티 정밀 윤문 (필수)
