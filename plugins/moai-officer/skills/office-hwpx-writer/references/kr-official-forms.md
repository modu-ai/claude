# 한국형 공식 문서 서식 SSOT (kr-official-forms)

> 공문(시행문)·기안문·품의서 3종의 실서식 정의 — 「행정 효율과 협업 촉진에 관한 규정」(대통령령) 및 같은 규정 시행규칙(행정안전부령)의 별지 서식을 기반으로 합니다.
> 이 파일은 한국형 문서 규격의 단일 기준(SSOT)입니다. hwpx·docx·pdf 생성 스킬과 보고·제안 계열 스킬은 두문·본문·결문 구조와 표기 규칙을 이 파일에서 가져다 씁니다.
> 구체 레이아웃(여백·폰트·플레이스홀더)은 `references/templates/gonmun.md` 등 템플릿 파일을 함께 참조하세요.

---

## 1. 공통 표기 규칙

### 1.1 문서번호 (문서분류·시행 번호)

- 형식: `{처리과 기관코드 또는 분류기호}-{일련번호}` (예: `총무과-1234`)
- 결문의 "시행"·"접수" 줄에 표기합니다: `시행 총무과-1234 (2026. 7. 24.)`
- 출처: 「행정 효율과 협업 촉진에 관한 규정」 별지 제1호 서식(기안문·시행문)

### 1.2 날짜·시행일자 표기

- 숫자로만 표기하고 연·월·일 글자 대신 마침표를 찍습니다: `2026. 7. 24.` (마지막 마침표 포함, 숫자 뒤 한 칸 띄움)
- 출처: 「행정 효율과 협업 촉진에 관한 규정 시행규칙」 제2조(문서 작성의 일반원칙)

### 1.3 수신·경유

- `수신: ○○○장관(○○과장)` — 수신자가 기관장이면 직위, 괄호 안에 처리 담당 부서·직위를 병기합니다.
- 수신자가 여럿이면 `수신: 수신자 참조`로 쓰고 결문 아래에 수신자란을 따로 둡니다.
- 경유 기관이 있으면 수신 다음 줄에 `(경유:)` 표기합니다.

### 1.4 항목 번호체계 (개조식 계층)

계층이 내려갈 때마다 아래 순서로 부호를 바꿉니다. 하위 항목은 상위 항목 첫 글자에서 2타(한글 1자) 들여 씁니다.

```
1. → 가. → 1) → 가) → (1) → (가) → ① → ㉮
```

- 항목이 하나뿐이면 번호를 붙이지 않습니다.
- 출처: 「행정 효율과 협업 촉진에 관한 규정 시행규칙」 제2조

### 1.5 붙임·끝. 표시

- 붙임이 있으면 본문 마지막 줄 다음에 표기: `붙임  1. ○○계획서 1부.` / `2. ○○명단 1부.  끝.`
- 붙임이 없으면 본문이 끝난 곳에서 한 글자(2타) 띄우고 `끝.` 표시.
- 본문·붙임이 표로 끝나는 경우: 표의 마지막 칸까지 기재사항이 차면 표 아래 왼쪽 기본선에서 `끝.` 표시, 중간에서 끝나면 마지막 기재 다음 칸에 `이하 빈칸` 표시.
- 출처: 「행정 효율과 협업 촉진에 관한 규정 시행규칙」 제4조(문서의 "끝" 표시)

### 1.6 발신명의

- 행정기관 명의: `○○부장관`, `○○시장` 등 기관장 명의를 문서 하단 중앙에 크게(통상 본문보다 큰 글자) 표기하고 직인(관인)을 찍습니다.
- 기업 내부 문서: `주식회사 ○○ 대표이사 ○○○` 또는 부서장 명의. 내부 결재 문서(기안문·품의서)는 발신명의를 생략할 수 있습니다.

## 2. 공문(시행문) 레이아웃

대외 발송용 문서. 두문 → 본문 → 결문 3단 구조.

```
┌──────────────────────────────────────────────┐
│ [두문]  기관명 (중앙, 로고 가능)              │
│         수신: {수신자}                        │
│         (경유:)                               │
├──────────────────────────────────────────────┤
│ [본문]  제목: {제목}                          │
│         1. 관련: {근거 문서·법령}             │
│         2. {본문 내용 — 항목 번호체계 적용}   │
│         붙임  1. {붙임명} 1부.  끝.           │
├──────────────────────────────────────────────┤
│ [결문]  {발신명의}                (직인)       │
│         기안자·검토자·결재권자 직위/서명       │
│         시행 {처리과}-{번호} ({시행일자})      │
│         접수 {처리과}-{번호} ({접수일자})      │
│         우 {우편번호} {주소}  /{홈페이지}      │
│         전화 {번호} 팩스 {번호} /{이메일}      │
│         /{공개 구분: 공개·부분공개·비공개}     │
└──────────────────────────────────────────────┘
```

- 출처: 「행정 효율과 협업 촉진에 관한 규정」 제4조·별지 제1호 서식. 세부 여백·폰트는 `templates/gonmun.md`.

## 3. 기안문 레이아웃

내부 의사결정용 초안 문서. 시행문과 같은 서식을 쓰되(기안문·시행문 통합 서식), 결재란이 핵심입니다.

```
┌──────────────────────────────────────────────┐
│ 결재란: │ 기안 │ 검토 │ 협조 │ 전결/대결 │ 결재 │
│         │ 담당 │ 팀장 │ 관련부서 │ (표기) │ 기관장 │
├──────────────────────────────────────────────┤
│ 수신: 내부결재 (대외 시행 시 수신자 기재)      │
│ 제목: {제목}                                  │
│ 1. 관련(근거): {지시·법령·계획}               │
│ 2. 목적·경위                                  │
│ 3. 내용(안): {항목 번호체계 적용}             │
│ 4. 조치·시행 사항                             │
│ 붙임  {n}부.  끝.                             │
└──────────────────────────────────────────────┘
```

- 결재 표기 규칙:
  - **전결**: 위임전결권자가 결재하는 경우, 전결권자 서명란에 "전결" 표시 후 서명. 결재하지 않는 상위자의 서명란은 만들지 않습니다.
  - **대결**: 결재권자 부재 시 직무대행자가 "대결" 표시 후 서명하고, 사후 보고가 필요하면 결재권자 서명란을 남겨 둡니다.
  - 출처: 「행정 효율과 협업 촉진에 관한 규정」 제10조(문서의 결재)

## 4. 품의서 레이아웃

기업 실무에서 지출·구매·계약 등 승인을 구하는 내부 문서. 법정 서식은 아니며(기업 관행 서식), 기안문 구조에 금액·집행 정보를 더합니다.

```
┌──────────────────────────────────────────────┐
│ 품 의 서            문서번호: {부서}-{번호}    │
│ 결재란: │ 기안 │ 검토 │ 합의 │ 전결 │ 승인 │  │
├──────────────────────────────────────────────┤
│ 제목: {품의 제목}                             │
│ 1. 품의 목적(사유)                            │
│ 2. 내용: 품목/수량/단가/금액 표               │
│    - 금액은 원 단위, 부가세 포함 여부 명기     │
│ 3. 집행 예정일·지급 조건·거래처               │
│ 4. 예산 과목·잔액 (해당 시)                   │
│ 붙임  견적서 등 {n}부.  끝.                   │
│ 작성일: {YYYY. M. D.}   기안자: {서명}        │
└──────────────────────────────────────────────┘
```

- 합의(협조)란: 예산 부서·법무 부서 등 관련 부서 확인이 필요할 때 결재선 중간에 둡니다.
- 전결 기준(금액 한도별 전결권자)은 각 사 위임전결 규정을 따릅니다 — 기본값 없음, 사용자에게 확인 필요.

---

## 5. 결재선 표준 흐름

```
기안(담당) → 검토(팀장/부서장) → 합의·협조(관련 부서) → 전결 또는 최종 결재(결재권자)
```

- 위임전결 규정이 있으면 그 금액·사안 기준을 우선 적용합니다. 규정 정보가 없으면 결재선을 임의로 확정하지 말고 사용자에게 조직의 결재 권한 범위를 확인합니다.
- 전자결재 시스템 연동 실무는 `moai-coworker:business-process-manager`의 "전자결재·결재선 설계" 절을 참조하세요.

## 6. 검수 체크리스트

- [ ] 날짜가 `YYYY. M. D.` 마침표 표기인가
- [ ] 항목 번호가 `1.→가.→1)→가)` 순서를 지키는가
- [ ] 본문 끝에 `끝.` 표시가 있는가 (붙임 있으면 붙임 뒤)
- [ ] 붙임 표기가 `붙임 n. ○○ 1부.` 형식인가
- [ ] 결재란에 실제 결재 단계만 있는가 (전결 시 상위자 칸 없음)
- [ ] 발신명의·시행 번호·연락처(결문)가 채워졌는가

> 정형 서식 보호: 이 서식들은 개조식·항목 번호 체계가 필수입니다. AI 슬롭 검수 시 금지어 치환만 수행하고 구조는 재작성하지 않습니다.
