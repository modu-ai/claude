# story-cover-art

> 표지·일러스트를 생성하는 스킬. 인쇄(DPI 300+)와 웹 썸네일 양쪽 해상도를 모두 다룬다.

## 1. 개요

표지는 **무엇을** 보여줄지(주 인물·분위기·핵심 소품)와 **언제** 시선을 끌지(구도·대비)가 클릭률을 결정한다. 인물 일관성은 Soul ID 학습 ID로 유지한다.

## 크레딧 고지 (사전 고지 의무)

이 작업은 **Higgsfield 크레딧을 소모**합니다. 생성 전 예상 소모량을 사용자에게 고지하고 확인을 받는다.

| 항목 | 예상 크레딧 |
|------|-----------|
| 표지 1점 (Soul 2.0 + 업스케일) | 약 4~8 크레딧 |

진행하기 전 사용자에게 "위 크레딧이 소모됩니다. 계속 진행할까요?"로 확인한다. 사용자가 거부하면 생성을 중단하고 프롬프트 온리 모드로 전환한다.

## 모델 라우팅 표

| 용도 | 1순위 Higgsfield 모델 |
|------|---------------------|
| 표지·일러스트 | Soul 2.0 (고품질 인물·배경) |
| 인쇄 해상도 | 업스케일 (DPI 300+) |
| 인물 일관성 | Soul ID (story-character-sheet 학습 ID) |

## MCP 미연결 폴백 (프롬프트 온리 모드)

Higgsfield MCP에 연결할 수 없을 때는 생성을 생략하고 완성 프롬프트를 텍스트로 출력한다. 출력 끝에 "Higgsfield 웹(https://higgsfield.ai)에 위 프롬프트를 붙여넣으세요" 안내를 추가한다. 서버 과부하 시 잠시 대기 후 재시도 안내를 함께 표시한다.

## 3. 워크플로우

### Step 1: 콘셉트 확정
주제·분위기·타깃 독자·포맷(단행본·웹썰·웹소설)을 정한다.

### Step 2: 크레딧 고지 + 확인
표지 점수 × 크레딧 + 업스케일 비용을 고지하고 승인받는다.

### Step 3: Soul 2.0 생성
콘셉트 프롬프트 + 캐릭터 ID로 표지 후보 2~3안을 생성.

### Step 4: 업스케일
선택안을 인쇄 해상도로 업스케일.

### Step 5: 타이포그래피 가이드
제목·저자명 자리를 별도 메모로 남긴다(폰트는 편집·디자인 단계).

## 4. 주의사항

- 웹 썸네일은 모바일 축소 시 가독성 우선 — 얼굴·제목을 크게.
- 인쇄 표지는 DPI 300+와 여백(제본 여유) 필수.
