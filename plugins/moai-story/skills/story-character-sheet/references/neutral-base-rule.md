# 중립 기본형 · 실루엣 극단 차별 · 색 반복 · 참조 재생성

> `story-character-sheet` Step 4에서 로드한다. 캐릭터 참조(Soul ID 시드)의 품질과 재사용성을 지키는 4개 원칙. AI 작화에서 캐릭터가 컷마다 흔들리는 실제 빈발 실패를 막는다.

## 1. 중립 기본형 원칙 (재사용성)

캐릭터 참조는 **중립 기본형으로만** 뽑는다.

- **넣는 것:** 고정 외형 + 기본 의상 + 중립 포즈 + 기본 표정 1~2개.
- **절대 넣지 않는 것:** 손에 든 물건, 특정 동작, 특정 감정 연기, 장면 소품.
- **이유:** 참조에 소품·포즈가 박히면 그 인물을 쓰는 **다른 모든 컷에 그 소품이 따라붙는다**. 장면 소품·상황 포즈는 참조가 아니라 컷 프롬프트(`story-webtoon-art`)에서 그 컷에만 부여한다.
- **각도:** 정면·측면·반측면 + 표정 변화. (비표준어 "반쯔렴"이 아니라 **반측면**.)

## 2. 실루엣 극단 차별 (인물 혼동 방지)

비슷한 두 인물은 독자·AI 모두 헷갈린다.

- 실루엣·체형을 **극단적으로 다르게** 설계한다 — 참조 이미지와 프롬프트 양쪽에서.
- **프롬프트 강조만으론 부족하다.** 강조로 안 갈리면 한쪽을 거구·빡빡머리·애꾸 등으로 **외형 자체를 바꿔 참조를 다시 뽑는다**(에스컬레이션).

## 3. 색 반복 명시 (색 표류 방지)

머리색·의상색은 명시하지 않으면 **컷마다 바뀐다**.

- 머리색·의상색을 **일관성 앵커에 넣고**, 참조 이미지와 **매 컷 프롬프트 양쪽에 반복** 명시한다(예: "갈색 단발, 베이지 셔츠").
- 없는 요소도 명시한다 — "안경 없음"처럼 부정형으로 박아야 엉뚱한 요소(안경 등)가 새어 들지 않는다.

## 4. 참조 재생성 에스컬레이션 (사다리)

일관성이 계속 깨지면 프롬프트 손질에 매달리지 말고 단계를 올린다.

```
1단계: 컷 프롬프트에 고정 외형·색을 반복 명시
2단계: 앵커를 더 강한 식별 요소로 교체(예: 눈 밑 점 → 안대)
3단계: 참조 이미지 자체를 실루엣·체형을 바꿔 다시 뽑는다 ← 근본 해결
```

> 규칙 명시(1~2단계)로 안 잡히는 혼동은 **참조 재생성(3단계)**이 답이다. 이 판단은 `story-webtoon-qc`의 "캐릭터 혼동" 결함 처리와 연결된다.
