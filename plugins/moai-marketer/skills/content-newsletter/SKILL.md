# 뉴스레터 (Newsletter)

## 개요

이메일 뉴스레터를 기획하고 작성합니다. 뉴스레터 기획부터 발행까지 전 과정, 구독자 확보 전략, 오픈율·클릭률 최적화, 위클리·먼슬리 콘텐츠 캘린더 연동까지 종합적인 뉴스레터 서비스를 제공합니다.

## 트리거 키워드

뉴스레터, 이메일 뉴스레터, 구독자, 뉴스레터 발행, 위클리, 먼슬리, 콘텐츠 구독

## 실전 뉴스레터 패턴 (한국 시장)

### 1. 오픈율 30%+ 제목 패턴

| 패턴 | 예시 | 오픈율 |
|---|---|---|
| 숫자 + 혜택 | "이번 주 읽어야 할 AI 뉴스 3" | ★★★ |
| 호기심 갭 | "넷플릭스가 OO를 포기한 이유" | ★★★★ |
| 개인화 | "[이름]님이 놓친 ~" | ★★★ |
| 시간 제한 | "오늘 자정 마감, ~ 신청을 안 하셨다면 지금" | ★★★★ |
| 질문형 | "왜 ~는 ~할까?" | ★★★ |
| 역설·반전 | "성공하지 마세요" | ★★★★ |
| 트렌드 차용 | "2026 OO 결산" | ★★★ |

40자 이내 + 이모지 1개 이하 + 첫 단어에 핵심.

### 2. 뉴스레터 본문 4 구조 (Hook → Story → Insight → CTA)

```
[Hook]    1~2 문장 — 시선 끌기 (질문·통계·역설)
[Story]   5~10 문단 — 본문 (한 인사이트 = 한 단락)
[Insight] 1~2 문단 — 독자 적용 가능한 한 줄 결론
[CTA]     1 행동 — 구체적 다음 단계 (1개만)
```

### 3. 발행 주기 매트릭스

| 주기 | 적정 분량 | 적합 컨셉 |
|---|---|---|
| 주간 (월 4회) | 800-1500자 | 트렌드·뉴스 큐레이션 |
| 격주 (월 2회) | 1500-3000자 | 깊이 있는 분석 |
| 월간 (월 1회) | 3000-5000자 | 종합 리포트·시즌 가이드 |
| 일간 (주 5회) | 500자 이내 | 짧은 인사이트·팁 |

같은 주기 1년 유지가 가장 큰 성장 요인.

### 4. 구독자 페르소나 매핑

뉴스레터는 1명에게 1:1 편지처럼 — 페르소나 1명 명확히.

```yaml
persona:
  name: "30대 후반 마케팅 매니저"
  job: "B2B SaaS 회사 마케팅 팀장"
  pain: "데이터 분석 시간 부족·트렌드 따라가기 어려움"
  habit: "주간 출퇴근길 10분 읽기"
  preference: "실전 적용 가능한 1~2 인사이트"
```

페르소나 1명 → 모든 콘텐츠 결정 (제목·분량·톤·발행 시간).

### 5. 구독자 증가 5 채널

1. **기존 채널 크로스 프로모션** — 블로그·SNS에 가입 폼
2. **추천 시스템** — "친구에게 추천 시 ~"
3. **랜딩 페이지 강력화** — 가입 후 첫 이메일 미리보기
4. **콘텐츠 부분 공개** — 풀버전 가입자 전용
5. **콜라보 — 유사 뉴스레터 상호 추천** (1만 구독 이상부터 가능)

## 워크플로우

### 1단계: 뉴스레터 목적 파악
- 목적: 브랜드 인지 / 리드 육성 / 고객 관계 / 수익화
- 발송 주기: 주간 / 격주 / 월간
- 타겟 독자: 직군, 관심사, 구독 동기

### 2단계: 뉴스레터 구조 설계
```
[제목 (Subject Line)]
- 문자 수: 40자 이내 (모바일 잘림 방지)
- 이모지: 1개 이내 (너무 많으면 스팸 처리)
- 개인화: "[이름]님," 접두어 (오픈율 +20%)
- 공식: [숫자/질문/혜택] + [키워드]
  예: "이번 주 읽어야 할 AI 뉴스 3가지"

[프리헤더 (Preview Text)]
- 40~100자 이내 (제목 보완 설명)

[본문 구조]
① 인트로 (3~5문장): 이번 호 핵심 메시지
② 메인 콘텐츠 (1~3개 섹션):
   - 큐레이션: 업계 뉴스 + 편집자 코멘트
   - 인사이트: 독자가 얻을 수 있는 핵심 관점
   - 실용 정보: 도구, 팁, 체크리스트
③ CTA (1개만): 명확한 다음 행동 유도
④ 마무리: 다음 호 예고 + 구독 유지 동기
```

### 3단계: 오픈율 최적화

**높은 오픈율 제목 공식:**
- 질문형: "왜 ~가 실패하는가?"
- 숫자형: "이번 주 놓치면 안 될 5가지"
- 개인화: "[독자가 관심 있는 주제] 완벽 정리"
- 긴박감: "오늘만 공개하는 ~"
- 호기심: "아무도 말하지 않는 ~의 진실"

**피해야 할 제목:**
- "안녕하세요, [회사명] 뉴스레터입니다"
- "이번 달 소식을 전해드립니다"
- 과도한 이모지 (🔥💯🎁🚀 남발)

### 4단계: 구독자 확보 전략

**리드 마그넷 (구독 유인):**
- 무료 템플릿, 체크리스트, 가이드 PDF
- 독점 정보, 할인 코드
- 첫 호 특별 콘텐츠 예고

**구독 폼 최적화:**
- 입력 필드 최소화 (이메일 1개 → 가장 높은 전환율)
- "스팸 안 보냄" 명시
- 발송 주기와 내용 예시 공개

**배포 채널:**
- 웹사이트 팝업 / 블로그 하단 / SNS 프로필 링크
- 파트너십: 유사 뉴스레터 상호 교차 홍보

### 5단계: 발송 플랫폼 추천

**국내/글로벌 이메일 발송 도구:**
- **Stibee (스티비)**: 국내 최적화, 한국어 지원, 소규모 무료
- **Mailchimp**: 글로벌 표준, 구독자 500명 무료
- **ConvertKit**: 크리에이터 특화, 자동화 강점
- **Substack**: 유료 구독 모델, 빌트인 결제
- **Ghost**: 블로그+뉴스레터 통합, 자체 운영

### 6단계: 콘텐츠 캘린더 설계 (월간)

**월간 뉴스레터 캘린더 예시 (주간 발행):**
```
1주: 업계 트렌드 요약 (큐레이션 중심)
2주: 심층 인사이트 (오리지널 콘텐츠)
3주: 실용 도구·팁 (How-to 중심)
4주: 독자 사례·Q&A + 다음 달 예고
```

## 사용 예시

- "스타트업 위클리 뉴스레터 첫 호 써줘"
- "이메일 제목 후보 10가지 만들어줘 (오픈율 높이기)"
- "구독자 1,000명 확보 전략 기획해줘"
- "먼슬리 뉴스레터 콘텐츠 캘린더 만들어줘"
- "업계 트렌드 뉴스레터 구성 짜줘"

## 출력 형식

- 뉴스레터 초안 (제목·프리헤더·본문·CTA)
- 제목 후보 (10-15가지, 오픈율 최적화)
- 구독자 확보 전략 가이드
- 콘텐츠 캘린더 (월간/분기별)
- 발송 플랫폼 추천

## 주의사항

⚠️ **AI 생성 콘텐츠 주의**: AI가 생성한 뉴스레터 내용은 사실 확인 후 발송하세요. 업계 통계, 링크, 인물 정보는 반드시 원출처를 검증하세요.

**스팸 필터 걸림**: 과도한 이모지, "무료", "할인" 등 단어 남발은 스팸으로 분류될 수 있습니다. 제목과 본문에서 스팸 유발 단어를 최소화하세요.

**플랫폼 규격 변경**: 이메일 클라이언트(Gmail, 네이버 메일)의 렌더링 방식은 수시로 변경됩니다. 발송 전 주요 클라이언트에서 테스트하세요.

**브랜드 가이드 미제공**: 브랜드 톤앤매너, 색상, 로고 사용 지침을 제공하면 더 일관된 뉴스레터 템플릿을 설계할 수 있습니다.

## 발행 (WordPress MCP)

ai-slop 검수 → humanize 체인까지 마친 완성 뉴스레터 본문은, 이 플러그인의 `.mcp.json`에 등록된 WordPress MCP 커넥터(mcp.wordpress.com)로 바로 발행할 수 있습니다(웹 아카이브용 포스트 발행 시).

- 흐름: **draft 저장 → 사용자 검토 → 발행** (제목·본문·태그 반영)
- 커넥터 미연결 시 본문 마크다운만 반환하고 "Settings > Connectors에서 WordPress 연결" 안내
- Stibee·Mailchimp 등 이메일 발송 플랫폼은 발행 자동화 대상이 아니므로, 발송용 본문은 그대로 복사해 사용합니다
- SNS 예고·동시 발행이 필요하면 같은 `.mcp.json`의 post-bridge·typefully 커넥터로 크로스포스트할 수 있습니다
