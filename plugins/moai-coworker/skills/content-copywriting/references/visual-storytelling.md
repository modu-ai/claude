# 비주얼 스토리텔링 (visual-storytelling)

> MoAI-Cowork v1.0.0 하네스 참고자료

## 개요

스토리 설계, 텍스트 작성, AI 이미지 생성, HTML 레이아웃을 통해 비주얼 스토리텔링을 협력하여 제작하는 스킬입니다.

## 전문가 역할

- **스토리 아키텍트**: 내러티브 구조, 장면 구성, 시각-텍스트 균형 설계
- **에세이 작성자**: 비주얼 스토리텔링의 텍스트 부분 작성, 장면별 이모션 호호흡
- **이미지 프롬프터**: Gemini 이미지 생성, 장면 간 시각 일관성 유지
- **레이아웃 빌더**: HTML/CSS로 텍스트와 이미지 통합, 반응형 페이지 제작
- **검수자**: 스토리-텍스트-이미지-레이아웃 일관성 교차 검증

## 워크플로우

- 기존 콘텐츠 및 이미지 수집

## 브랜드 내러티브 AI틱 가드 (L-1~L-8)

스토리 텍스트 작성(작성자)·검수(검수자) 단계에서 아래 8종 AI틱을 회피·검출한다. SSOT: `moai-coworker:general-humanize-korean` references/ai-tell-taxonomy.md L 카테고리.

| ID | 패턴 | 처방 방향 |
|----|------|----------|
| L-1 | 클리셰 도입부·기계적 기승전결 ("어느 날", "그렇게 시작되었습니다") | 구체 시점·장면·수치로 도입 |
| L-2 | 상투적 감정 표현 — 직접 명명형("뭉클했다")이든 신체 은유형이든 | 검증 가능한 행동·디테일로 |
| L-3 | 교훈 강박 종결 ("~을 배웠습니다/깨달았습니다") | 실제 바뀐 행동·프로세스·수치 |
| L-4 | 클리셰 전환구 ("하지만 그때", "그 순간", "운명처럼") | 시점·사건 명시 전환 |
| L-5 | 지나치게 매끄러운 결말 | 잔여 갈등·현재진행 과제 공개 |
| L-6 | 창업자 신화 템플릿 ("작은 사무실에서 시작해") | 검증 가능한 고유 디테일 |
| L-7 | 가짜 구체성 (검증 불가 통계·전문성 어필) | 출처·모수 있는 실측 |
| L-8 | 광고형 고객 후기 스토리 | 구체 행동 중심 서술 |

**양성 원칙 (더할 것)**: ① 고객을 주인공으로(브랜드는 가이드) ② 완벽 대신 갈등·결핍 ③ 검증 가능한 구체성 ④ TMI 나열 금지 ⑤ 기승전결 강박 버리기 ⑥ 창업자 서사→고객 중심 확장 ⑦ 실패 공유는 톤 관리와 함께 ⑧ 지킬 수 있는 약속만 ⑨ 스토리 버전 A/B 테스트. 상세 레시피: `general-humanize-korean` references/rewriting-playbook.md § L.
