# 개인정보 마스킹 규격 (kr-pii-masking)

> moai-coworker | general-ai-slop-reviewer 소유 공유 규격
> 이력서·급여대장·고객 문의·계약서 원본처럼 개인정보가 포함된 입력을 처리하는 모든 스킬(career·recruiter·cs·accountant·lawyer 라인)이 이 파일 하나를 참조합니다.
> 근거 법령: 개인정보 보호법(제3조 최소처리 원칙, 제28조의8 국외 이전), 표준 개인정보 보호지침.

## 0. 대원칙 — 산출물 재출력 금지

1. **입력에 포함된 개인정보 원본을 산출물에 그대로 재출력하지 않습니다.** 산출물에 식별자가 필요하면 아래 마스킹 형식으로만 표기합니다.
2. 업무 수행에 **불필요한 개인정보는 산출물에서 아예 제거**합니다 (마스킹보다 삭제가 우선 — 개인정보 보호법 제3조 최소처리 원칙).
3. 마스킹 전 원문을 예시·인용 형태로도 남기지 않습니다 ("원본: …" 병기 금지).

## 1. 항목별 마스킹 자릿수 규칙

| 항목 | 규칙 | 표기 예 |
|------|------|---------|
| 주민등록번호 | 생년월일 6자리만 표시, **뒤 7자리 전체 마스킹** | `900101-*******` |
| 외국인등록번호 | 주민등록번호와 동일 규칙 | `900101-*******` |
| 계좌번호 | **뒤 4자리만 표시**, 나머지 마스킹 | `***-****-1234` |
| 카드번호 | **뒤 4자리만 표시**, 나머지 마스킹 | `****-****-****-5678` |
| 휴대전화·전화번호 | **중간 자리 마스킹** | `010-****-5678` |
| 이메일 | 로컬부 앞 2~3자만 표시하고 나머지 마스킹 (도메인은 유지 가능) | `ho***@example.com` |
| 주소 | 시·군·구까지만 표시, **상세주소(동·호수·번지) 제거** | `서울시 마포구` |
| 성명 | 문맥상 필요 시 가운데 글자 마스킹 또는 이니셜 | `홍*동`, `H씨` |
| 사업자등록번호 | 법인·사업자 공개 정보이므로 원칙적으로 마스킹 대상 아님. 단, **개인사업자 번호가 개인 식별과 결합되는 문맥**(예: 급여·채용 문서)에서는 뒤 5자리 마스킹 | `123-45-*****` |
| 운전면허·여권번호 | 뒤 4자리만 표시 | `**-**-****56` |

## 2. 처리 절차 (입력 처리 단계)

1. 입력 수신 시 위 표의 항목을 스캔합니다.
2. 산출물 작성 전에 마스킹/삭제를 적용하고, 이후 모든 중간·최종 산출물에서 동일 규칙을 유지합니다.
3. 사용자가 "마스킹 해제"를 명시적으로 요청해도, 산출물이 제3자 공유용(공고문·답변서·보고서)이면 마스킹 유지를 권고하고 이유를 알립니다.

## 3. 국외 이전 고지 조건 (개인정보 보호법 제28조의8)

개인정보처리자가 개인정보를 **국외로 이전**(해외 클라우드·해외 SaaS 저장 포함)하려면 원칙적으로 정보주체에게 다음을 알리고 **별도 동의**를 받거나 법정 예외(계약 이행에 필요한 처리위탁·보관의 고지 등)에 해당해야 합니다:

1. 이전되는 개인정보 항목
2. 이전되는 국가, 시기·방법
3. 이전받는 자의 성명(법인명)과 연락처
4. 이전받는 자의 이용 목적과 보유·이용 기간
5. 이전을 거부하는 방법·절차와 거부 시 불이익

→ 실무 함의: 개인정보 원본을 해외 서비스에 붙여넣어 처리하는 행위 자체가 국외 이전 이슈가 될 수 있으므로, **애초에 마스킹 후 입력**하는 것이 안전합니다. 처리방침에 국외 이전 고지가 있는지 확인을 안내하세요.
