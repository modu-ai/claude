# story-character-sheet: 캐릭터 비주얼 시트 + 일관성 앵커

캐릭터의 **외형을 흔들리지 않게 고정**하는 시트를 만드는 스킬이다. 서사 정보(성격·역할·말투)와 비주얼 시트(이미지 생성용 외형 디테일)를 함께 정리하고, 화마다 반드시 지킬 **일관성 앵커**를 못박는다. 이 시트는 Soul ID 학습의 시드가 되고, `story-webtoon-qc`가 일관성 위반을 판정하는 기준이 된다.

## 1. 개요

이 스킬은 **"무엇을, 어떤 외형으로, 무엇을 고정해 학습시킬지"**를 정한다. 실제 이미지 생성·Soul ID 학습 실행·크레딧 고지는 하지 않는다 — 그건 `moai-media`가 라이브 조회·비용 프리플라이트로 처리한다(§4-생성 실행 위임).

## 2. 이 스킬이 하지 않는 것

- **이미지 생성·Soul ID 학습 실행 안 함** — 생성·학습·크레딧 고지·모델 선택은 `moai-media:media-higgsfield-core` 계약에 위임한다.
- **크레딧 수치 단정 안 함** — "학습 20장 = N크레딧" 같은 상수를 박지 않는다. 비용은 moai-media의 `get_cost` 프리플라이트가 조회한다.
- **컷 프롬프트 조립 안 함** — 컷별 작화 프롬프트는 `story-webtoon-art` 소관. 이 스킬은 캐릭터 참조까지만.
- **회차 서사 안 씀** — 스토리 본문은 `story-webtoon-episode` / `story-webnovel-writer`.

## 3. 워크플로우

### Step 1 — 서사 정보 정리

성격 키워드(3개 이내)·역할·말투·"독자가 이 캐릭터를 좋아하는 이유"를 정리한다. 외형 디테일은 여기 섞지 않고 Step 2 비주얼 시트에 모은다(비주얼-서사 분리).

### Step 2 — 비주얼 시트 작성 (인물 8항목 / 동물 8항목)

인물이면 인물 8항목 표, 동물이면 동물 8항목 표를 채운다. 표 골격·레귤러/게스트 2계층 예산은 `references/visual-sheet-tables.md`.

### Step 3 — 일관성 앵커 결정 (짧고 필수)

비주얼 시트 항목 중 **화마다 반드시 고정할 1~2가지**를 정한다.

- 예: `은발 + 왼쪽 눈 밑 점`, `치즈태비 + 빨간 방울 목걸이`.
- 앵커가 흔들리면 독자는 **다른 캐릭터로 인식**한다. 머리색·의상색은 앵커에 넣고, 참조 이미지와 매 컷 프롬프트 양쪽에 반복 명시한다(`references/neutral-base-rule.md`).
- 이 앵커는 `story-webtoon-qc`가 "일관성 위반 0건"을 판정하는 기준으로 소비한다 — 반드시 명시적으로 적는다.

### Step 4 — 중립 기본형 참조 세트 사양

Soul ID(또는 참조 이미지)에 학습시킬 세트를 **중립 기본형으로만** 설계한다.

- **소품·상황 포즈 절대 금지.** 고정 외형 + 의상 + 중립 포즈 + 기본 표정 1~2개까지만. 손에 든 물건·특정 동작·특정 감정 연기는 넣지 않는다 — 그건 컷 프롬프트(`story-webtoon-art`)에서 그 컷에만 부여한다.
- 참조에 소품을 박으면 그 인물을 쓰는 **다른 모든 컷에 소품이 따라붙어** 재사용성이 깨진다.
- 각도 목록: 정면·측면·반측면·표정 변화. (비표준어 "반쯔렴"은 쓰지 않는다 → **반측면**.)
- 비슷한 두 인물은 실루엣·체형을 극단적으로 차별한다 — 프롬프트 강조로 안 되면 참조를 다시 뽑는다(에스컬레이션, `references/neutral-base-rule.md`).

### Step 4-생성 실행 — moai-media 위임

시드 이미지 생성과 Soul ID 학습, 비용 사전 고지, 모델 선택은 `moai-media:media-higgsfield-core` 계약(`models_explore` 라이브 조회 + `get_cost` 사전 고지)에 위임한다. 이 스킬은 "무엇을 학습시킬지"(중립 기본형 세트 구성·각도 목록·일관성 앵커)까지만 정한다.

> moai-media 미설치 시: 완성된 캐릭터 참조 프롬프트를 텍스트로 출력하고 Higgsfield 웹(https://higgsfield.ai)에서 직접 생성하도록 안내한다.

## 4. 출력 형식

```markdown
# [캐릭터 이름] 캐릭터 시트

## 서사 정보
- 성격 키워드 / 역할 / 말투 / 애착 포인트

## 비주얼 시트 ([인물 8항목 / 동물 8항목] — visual-sheet-tables 참조)
| 항목 | 내용 |
| ... |

## 일관성 앵커 (화마다 고정 · qc 판정 기준)
- [1~2가지 — 예: 은발 + 왼쪽 눈 밑 점]

## 중립 기본형 참조 세트 사양
- 각도: 정면·측면·반측면 / 표정 1~2개
- 소품·상황 포즈: 없음(컷 프롬프트에서 부여)
→ 생성·학습 실행: moai-media:media-higgsfield-core 위임
```

## 5. 주의사항

- **레귤러/게스트 2계층 예산을 지킨다.** 레귤러는 전체 시트, 게스트는 그 화 외형만 간략히(여러 화 일관성 부담 없음). 상세는 `references/visual-sheet-tables.md`.
- **일관성 앵커를 빼먹지 않는다.** 앵커 없는 시트는 qc가 판정할 기준이 없다.
- **중립 기본형 원칙을 어기면** 재사용성이 무너진다 — 참조엔 소품 금지.
- **크레딧 수치를 지어내지 않는다.** 비용은 moai-media가 조회한다.

## 6. 관련 스킬

- `story-webtoon-art` — 컷 프롬프트에 캐릭터 참조·앵커 삽입
- `story-webtoon-qc` — 일관성 앵커 위반 판정

### 위임
- `moai-media:media-higgsfield-core` — 시드 생성·Soul ID 학습·비용 프리플라이트
