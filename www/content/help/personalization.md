새 노트 앱을 처음 켜면 기본 화면 그대로 쓰다가, 며칠 지나면 글꼴을 바꾸고 다크 모드를 켜고 자주 쓰는 기능을 손에 익히게 됩니다. Claude도 똑같습니다. 처음엔 기본 설정으로 시작하지만, 화면 색과 글꼴, 언어, 답변 방식까지 내 손에 맞게 하나씩 조정할 수 있습니다.

이 페이지에서는 Claude의 외형 설정과 언어 변경, 모델·사고 옵션, 그리고 답변 스타일과 작업 맥락을 기억시키는 개인화 기능을 차례대로 살펴봅니다. 대부분 설정은 한 번 바꿔두면 모든 대화에 자동으로 적용됩니다.

## 외형 설정 — 화면 색과 글꼴 바꾸기

화면을 오래 보는 만큼, 눈이 편한 색과 글꼴을 고르는 것만으로도 사용감이 크게 달라집니다. Claude는 웹 화면에서 색 모드와 글꼴을 직접 고를 수 있습니다.

### 색 모드와 글꼴 옵션

| 항목 | 선택할 수 있는 옵션 |
|---|---|
| **색 모드(Color Mode)** | 라이트 / 시스템 설정 따르기 / 다크 |
| **채팅 글꼴(Chat Font)** | 기본 / 시스템 설정 따르기 / 난독증 친화 글꼴(Dyslexic Friendly) |

### 외형 바꾸는 방법

1. Claude에 로그인합니다.
2. 왼쪽 아래의 이니셜(또는 이름)을 클릭합니다.
3. **Settings(설정)** 로 이동합니다.
4. **Appearance(외형)** 탭을 엽니다.
5. 원하는 색 모드를 고릅니다(라이트 / 시스템 설정 / 다크).
6. 원하는 채팅 글꼴을 고릅니다(기본 / 시스템 설정 / 난독증 친화).
7. 사이드바는 오른쪽 위 버튼으로 펼치거나 접을 수 있습니다.

{{< hint type="note" >}}
사이드바는 완전히 없앨 수는 없고 접어두는 것만 가능합니다. 또한 이 외형 설정은 웹 화면에만 적용됩니다.
{{< /hint >}}

## 언어 설정 — 화면 언어 바꾸기

Claude는 화면(인터페이스) 언어를 한국어를 포함한 11개 언어로 제공합니다. 영어, 프랑스어, 독일어, 힌디어, 인도네시아어, 이탈리아어, 일본어, 한국어, 포르투갈어(브라질), 스페인어(라틴아메리카·스페인)를 고를 수 있습니다.

여기서 중요한 점이 하나 있습니다. **화면 언어와 대화 언어는 서로 별개**입니다. 즉, 메뉴와 버튼은 한국어로 두면서도 Claude와는 영어나 일본어 등 어떤 언어로든 대화할 수 있습니다.

### 화면 언어 바꾸는 방법

1. 왼쪽 아래의 프로필 아이콘을 클릭합니다.
2. 설정에서 **Language(언어)** 섹션으로 이동합니다.
3. 원하는 언어를 선택합니다.
4. 화면이 자동으로 새로고침되며 선택한 언어로 바뀝니다.

이 기능은 웹과 데스크톱 앱에서 모두 사용할 수 있습니다.

## 모델·노력·사고 설정

같은 질문이라도 어떤 모델로, 얼마나 깊이 생각하게 하느냐에 따라 답변의 꼼꼼함과 속도가 달라집니다. 이 세 가지는 모두 **전송 버튼 옆 메뉴**에서 조정합니다.

### 세 가지 설정 한눈에 보기

| 설정 | 무엇을 정하나 | 선택지 |
|---|---|---|
| **모델(Model)** | 어떤 Claude 모델로 답변받을지 | 모델 이름 클릭 → 목록에서 선택 ('More models'로 더 많은 모델 확인) |
| **노력(Effort)** | 얼마나 꼼꼼하게 답할지 | Low / Medium / High(기본) / Max |
| **확장 사고(Extended Thinking)** | 답하기 전에 추론 과정을 거칠지 | 켜기/끄기 |

### 노력(Effort) 수준 이해하기

노력 수준을 높일수록 더 꼼꼼한 답변을 받을 수 있지만, 처리 시간이 길어지고 토큰도 더 많이 사용합니다.

{{< hint type="note" >}}
노력 설정은 일부 모델(Opus 4.8, 4.7, 4.6, Sonnet 4.6)에만 적용됩니다.
{{< /hint >}}

### 확장 사고(Extended Thinking)란

확장 사고를 켜면 Claude가 답하기 전에 문제를 단계별로 추론합니다. 켜져 있으면 펼칠 수 있는 'Thinking' 영역이 표시되어, Claude가 어떻게 풀어가는지 그 과정을 볼 수 있습니다. 다음과 같은 작업에 특히 도움이 됩니다.

- 수학 증명
- 코딩 문제
- 프로젝트 계획 수립
- 문서 분석
- 여러 단계를 거치는 기술 문제

### 설정 바꾸는 방법

1. 전송 버튼 옆 메뉴를 찾습니다.
2. 모델 이름을 클릭해 원하는 모델을 고릅니다(또는 'More models'에서 추가 선택).
3. 노력 수준을 고릅니다(Low / Medium / High / Max).
4. 확장 사고를 켭니다. 노력 설정이 되는 모델은 'Thinking'을, 그 외 모델은 'Extended'를 토글합니다.
5. 메시지를 보냅니다. 바뀐 설정은 다음 답변부터 적용됩니다.

{{< hint type="warning" >}}
안전 시스템이 유해할 수 있는 내용을 감지하면 확장 사고가 중간에 끊겨 보일 수 있습니다. 이럴 때는 질문을 다른 방식으로 바꿔서 다시 시도해 보세요.
{{< /hint >}}

## 개인화 기능 — Claude를 내 방식에 맞추기

설정 메뉴 너머에는 Claude가 나를 더 잘 이해하게 만드는 개인화 도구가 있습니다. 크게 세 가지이며, 따로 써도 되고 함께 써도 됩니다.

### 세 가지 개인화 도구

| 도구 | 적용 범위 | 쓰임새 |
|---|---|---|
| **프로필 지침(Profile Instructions)** | 계정 전체의 모든 대화 | 나에 대한 기본 정보·선호를 한 번 적어두기 |
| **프로젝트 지침(Project Instructions)** | 특정 프로젝트 안에서만 | 그 작업에 필요한 맥락만 따로 주기 |
| **스킬(Skills)** | 대화·작업 단위 | 답변의 톤·형식 조정과 특정 작업 능력 추가 |

프로젝트 지침은 무료 사용자를 포함한 모든 사용자가 쓸 수 있으며, 무료 계정은 프로젝트를 최대 5개까지 만들 수 있습니다.

### 프로필 지침 설정하기

프로필 지침은 계정 전체에 적용되어 모든 대화에 반영됩니다(특정 대화에만 따로 적용할 수는 없습니다).

1. 왼쪽 아래의 이니셜을 클릭합니다.
2. **Settings(설정)** 를 엽니다.
3. **Instructions for Claude(Claude에게 줄 지침)** 로 이동합니다.
4. 계정 전체에 적용할 프로필 지침을 적습니다.

### 프로젝트 지침으로 작업 맥락 주기

특정 작업에만 맥락을 주고 싶다면 프로젝트를 만들고 그 안에 지침을 적습니다. 이 지침은 해당 프로젝트 안에서만 적용됩니다.

1. 새 프로젝트를 만듭니다.
2. 그 프로젝트 안에서 프로젝트 지침을 추가합니다.
3. 해당 작업에 필요한 맥락(말투, 배경 정보, 규칙 등)을 적어둡니다.

프로젝트 자체에 대한 자세한 내용은 [Projects 기능](/chat/projects/) 페이지를 참고하세요.

## 스타일에서 스킬로의 전환

예전에 답변 말투를 고르던 '스타일(Styles)' 기능은 '스킬(Skills)'로 옮겨가고 있습니다. 정리하면 다음과 같습니다.

| 기존 스타일 | 전환 후 상태 |
|---|---|
| 직접 만든 사용자 스타일 | 스킬로 자동 변환(단, 기본으로 꺼져 있어 직접 켜야 함) |
| Concise / Explanatory / Formal | 제공 종료 |
| Learning | 마켓플레이스 스킬로 유지 |

변환된 스킬은 슬래시 명령 형식(`/{스타일이름}-style`)을 사용합니다. 이때 스타일 이름은 소문자로 바뀌고 공백은 하이픈으로 바뀌며, 문장 부호·이모지·악센트는 제거됩니다. 이름이 겹치면 뒤에 숫자가 붙어 구분됩니다.

### 스킬 켜는 방법

1. **Customize(맞춤 설정) > Skills(스킬)** 로 이동합니다.
2. **'+'** 를 눌러 스킬을 추가합니다.
3. **'Browse skills(스킬 둘러보기)'** 를 선택합니다.
4. 원하는 스킬을 검색합니다(예: 'Learning').
5. **'+'** 또는 **'Install(설치)'** 를 눌러 스킬을 켭니다.

{{< hint type="tip" >}}
전환이 끝나면 스타일 메뉴는 Claude에서 사라집니다. 기본 스타일(Concise·Explanatory·Formal)이 익숙했다면, 원하는 말투를 프로필 지침이나 프로젝트 지침에 직접 적어두는 방법으로 대신할 수 있습니다.
{{< /hint >}}
