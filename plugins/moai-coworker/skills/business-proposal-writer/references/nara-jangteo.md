# 나라장터(G2B) 전자입찰 가이드 — 입찰자(공급자) 관점

> `business-proposal-writer`가 공공조달 입찰 제안서·적격심사 서류를 작성할 때 로드하는 참조 자료입니다.
> 발주자(구매 기관) 측 조달 문서 작성은 `moai-coworker:business-process-manager`의 `references/procurement-docs.md`를 사용하세요.

## 나라장터란

나라장터(G2B, Government to Business)는 조달청이 운영하는 국가 전자조달 시스템으로, 모든 공공기관의 물품·용역·공사 구매를 처리합니다. 2026년 기준 연간 약 200조 원 규모의 조달이 이 시스템을 통해 이루어집니다(확인 필요 — 조달청 연간 통계로 재확인 권장).

---

## 나라장터 등록 절차

### 1단계: 업체 등록

1. 나라장터(g2b.go.kr) 접속 → 공급자 회원 가입
2. 사업자등록증 사본 업로드
3. 대표자 공동인증서(범용) 또는 조달청 G-PIN 준비
4. 입찰 참가 자격 등록 신청 (조달청 지사 방문 또는 온라인)
5. 등록 완료 후 조달업체 코드 발급 (처리 기간: 3-5 영업일, 확인 필요)

### 2단계: 입찰 참가

1. 나라장터 공고 검색 (조달 유형·분류 번호·발주기관 필터)
2. 입찰 공고 상세 확인 (납품 조건, 규격, 평가 기준)
3. 입찰 참가 신청 및 보증금 납부 (입찰보증금: 입찰금액의 100분의 5 이상 — 국가계약법 시행령 제37조, 보증보험증권 대체 가능)
4. 전자입찰서 제출 (마감 시각 30분 전까지 제출 권장)
5. 개찰 결과 확인

### 3단계: 계약 체결

1. 낙찰 통보 수신 후 계약 서류 제출 (기한은 공고문 기준 — 확인 필요)
2. 계약보증금 납부 (계약금액의 100분의 10 이상 — 국가계약법 제12조·시행령 제50조)
3. 전자계약 체결 (나라장터 시스템 내 서명)
4. 납품 → 검사·검수 → 대금 청구 순서

## 조달 유형 (입찰자가 만나는 공고 종류)

| 유형 | 설명 | 적용 기준 |
|------|------|-----------|
| 일반경쟁 | 자격 요건을 갖춘 누구나 참가 | 제한 없음 (원칙 — 국가계약법 제7조) |
| 제한경쟁 | 지역·실적·규모 등 참가 자격 제한 | 발주기관이 요건 설정 |
| 수의계약 | 경쟁 없이 특정 업체와 계약 | 소액(국가계약법 시행령 제26조 기준) 또는 긴급 사유 |
| 지명경쟁 | 발주기관이 지명한 업체만 참가 | 특수 기술·보안 요구 시 |
| 다수공급자계약(MAS) | 조달청 사전 계약, 수요기관 직접 구매 | 소프트웨어·IT 장비 등 |

## 입찰 참가 자격 요건 (제안 전 셀프 체크)

- 사업자등록 상태 유효 (폐업·정지 불가)
- **국세·지방세 완납** 확인 — 납세증명서 유효기간 내 발급본 (홈택스·위택스 발급)
- **부정당업자 제재 기간이 아닐 것** — 나라장터에서 자사 제재 이력 사전 조회
- 업종별 면허·등록 유효 (건설업: 건설업 등록증, IT: 소프트웨어사업자 신고 등)
- **중소기업 우대**: 중소기업 확인서(중소벤처24 발급)로 적격심사 가점 또는 중소기업자 간 경쟁제품 분리 공고 혜택

## 필수 서류 목록

### 입찰 참가 자격 등록 시

- [ ] 사업자등록증 사본
- [ ] 법인등기부등본 (법인의 경우)
- [ ] 납세증명서 (국세·지방세 완납 확인)
- [ ] 건강보험·국민연금 완납 증명서
- [ ] 업종별 면허·등록증 (해당 업종)

### 입찰 참가 시

- [ ] 입찰 참가 신청서 (전자 제출)
- [ ] 사업자등록증 사본
- [ ] 수행 실적 증명서 (제한경쟁·적격심사 시)
- [ ] 입찰 보증보험증권 (현금 대체 가능)
- [ ] 기술·가격 제안서 (협상에 의한 계약 시)
- [ ] 중소기업 확인서 (가점 대상 시)

## 전자입찰 시스템 사용법 요약

1. **공동인증서 준비**: 범용 공동인증서(구 공인인증서)·금융인증서 또는 G-PIN
2. **나라장터 로그인**: 차세대 나라장터(2024~)는 별도 플러그인(ActiveX) 없이 웹 표준 환경에서 공동인증서·금융인증서로 로그인
3. **입찰 공고 검색**: 메인 → 입찰정보 → 조건별 검색 (품명/공고번호/발주기관)
4. **전자입찰 참가**: 공고 상세 → 입찰 참가 신청 → 금액 입력 → 전자서명 제출
5. **결과 확인**: 개찰 후 입찰 내역 → 결과 조회

> 참고: 2026년부터 일부 공공기관은 모바일 앱(나라장터 앱)을 통한 입찰 결과 확인을 지원합니다(확인 필요).

## 제안(응찰) 준비 워크플로우 — 요건 분석 → 역량 매칭 → 기술·가격 제안

공고문·제안요청서(RFP)를 받은 뒤에는 아래 순서로 준비합니다.

1. **요건 분석**: RFP 요건을 항목 단위로 분해하고 평가 방식·배점·가중치(적격심사 기준표, 기술·가격 배분 예: 90:10·80:20)를 파악합니다. 자사 현황과의 격차를 식별하고 승산 있는 영역에 집중합니다.
2. **역량 매칭**: 과거 수행 실적을 요구 실적과 매칭하고(실적증명서 발급 가능 여부 확인), 필요 인력·기술 격차는 파트너·하도급 전략으로 보강합니다. 경쟁사 대비 차별화 포인트를 명시합니다.
3. **기술 제안(기술평가서)**: 요구사항별 해결 방법·적용 방법론·품질 보증·리스크 완화·일정(마일스톤)을 평가 항목 순서대로 작성합니다 — 심사위원이 배점표 순서로 채점하므로 목차를 평가 기준표와 1:1 대응시킵니다.
4. **가격 제안**: 원가 항목별 추정 → 이윤·리스크 마진 설정 → 예정가격 대비 투찰률 전략. 적격심사제 공고는 낙찰하한율 구간을 공고문에서 확인합니다(공고별 상이 — 확인 필요).
5. **최종 검증**: 기술-가격 정합성, 요구사항 전항목 충족 여부, 페이지 제한·제출 규격 준수를 점검하고 마감 30분 전 제출을 목표로 합니다.
