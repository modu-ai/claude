# 리텐션 (Retention)

## 개요

신규 획득보다 **기존 고객 유지가 저렴하고 수익성이 높다**는 전제 위에서, 마케팅이 할 수 있는 리텐션을 설계하는 스킬입니다. 이탈 조기 신호를 잡아 사전 개입하고, 떠나려는 순간을 세이브 오퍼로 붙잡고, 이미 떠난 고객을 윈백 시퀀스로 되돌리며, 라이프사이클 단계별 메시지로 관계를 유지합니다.

이 스킬은 **메시지·캠페인·지표** 측면에 집중합니다. 구매 주기 계산이나 구독 요금제 오퍼레이션, 단위경제 모델링은 다루지 않고 moai-seller 커머스 스킬로 위임합니다(아래 책임 경계 참고).

참고 원본: github.com/coreyhaines31/marketingskills (MIT) — churn-prevention 스킬의 방법론을 마케팅 라이프사이클 관점으로 재구성했습니다.

## 트리거 키워드

리텐션, 고객 유지, 이탈 방지, 이탈률, 처닝, 윈백, 재활성화, 재참여, 라이프사이클 메시지, 헬스 스코어, 취소 플로우, 세이브 오퍼, 리텐션 커브, 코호트, 재방문 유도

## 실전 리텐션 패턴

### 1. 이탈 유형 2분류 (Churn Classification)

이탈은 원인이 다르면 대응도 달라집니다.

| 유형 | 비중 | 원인 | 마케팅 대응 |
|---|---|---|---|
| **자발적 이탈**(Voluntary) | 50-70% | 고객이 직접 취소 | 취소 플로우·이탈 설문·세이브 오퍼 |
| **비자발적 이탈**(Involuntary) | 30-50% | 결제 실패·카드 만료 | 던닝(dunning) 시퀀스·사전 알림 |

> 마케팅은 주로 **자발적 이탈**의 라이프사이클 메시지를 담당합니다. 비자발적 이탈의 결제 재시도 로직·던닝 운영은 커머스·구독 운영 영역과 겹치므로, 던닝 메일 카피만 이 스킬에서 지원하고 재시도 정책은 위임합니다.

### 2. 이탈 조기 신호 & 헬스 스코어

이탈은 취소 2-6주 전부터 신호가 나옵니다. 사후 윈백보다 **사전 개입**이 ROI가 높습니다.

**선행 지표(Leading indicators)**:
- 로그인·방문 빈도 50%+ 감소
- 핵심 기능·핵심 행동 사용 중단
- 지원 문의 패턴 변화
- 요금·해지 페이지 방문 증가
- 데이터 내보내기 시작 (강한 이탈 신호)

**헬스 스코어**(참여·사용·지원 감정·결제 건강 가중 합산):
| 점수 | 등급 | 대응 |
|---|---|---|
| 80-100 | 건강 | 옹호·추천 유도 |
| 60-79 | 주의 필요 | 가치 재강조 콘텐츠 |
| 40-59 | 위험 | 재참여 캠페인 |
| 0-39 | 심각 | 즉시 개입·세이브 오퍼 |

### 3. 취소 플로우 아키텍처 (Cancel Flow)

떠나려는 순간의 마지막 접점. 기본 취소 플로우만 있어도 **10-15% 세이브** 가능.

```
트리거 → 이탈 설문 → 동적 세이브 오퍼 → 확인 → 취소 후(윈백 트리거)
```

**이탈 설문**(5-8개 사유): 가격 부담 · 사용 저조 · 기능 부족 · 경쟁사 이동 · 기술 문제 · 일시적 불필요 · 사업 종료.

**동적 세이브 오퍼**(사유에 매칭):
| 사유 | 세이브 오퍼 |
|---|---|
| 가격 부담 | 2-3개월 20-30% 할인 또는 요금제 다운그레이드 |
| 사용 저조 | 1-3개월 일시정지(pause) 또는 온보딩 재지원 |
| 기능 부족 | 로드맵 미리보기 + 예상 시점 |
| 경쟁사 이동 | 비교 데이터 + 인센티브 |
| 기술 문제 | 즉시 지원 에스컬레이션 |

> **핵심**: 죄책감 유발("정말 떠나실 건가요?") 대신 **가치 재강조 + 맞춤 오퍼**. 취소 후에도 쉬운 재활성화 경로를 눈에 띄게.

### 4. 윈백·재참여 시퀀스 (Winback / Re-engagement)

**재참여**(아직 고객, 활동 저조) vs **윈백**(이미 이탈)은 다릅니다.

**재참여 시퀀스**(헬스 스코어 하락 트리거):
| 시점 | 목적 |
|---|---|
| 신호 감지 D+0 | 가치 재상기 (놓친 핵심 기능·혜택) |
| D+3 | 성공 사례·사회적 증거 |
| D+7 | 개인화 인센티브 또는 1:1 지원 제안 |

**윈백 시퀀스**(취소 직후 트리거):
| 시점 | 목적 |
|---|---|
| 취소 D+0 | 감사 + 쉬운 재활성화 링크 |
| D+7 | "그동안 달라진 것"(신규 기능·개선) |
| D+30 | 강한 컴백 인센티브 (한정) |
| D+90 | 마지막 오퍼 또는 정리 |

### 5. 라이프사이클 메시지 (Lifecycle Messaging)

인지→활성화→습관화→충성→이탈위험 각 단계에 맞는 메시지를 트리거 기반으로 자동화합니다. 사용량 급감·기능 한도 도달·지원 티켓 에스컬레이션 같은 트리거에 **가치 강화 콘텐츠**로 선제 대응합니다(죄책감 메시지 금지).

### 6. 리텐션 지표 프레임

| 지표 | 정의 | 참고 목표 |
|---|---|---|
| **월 이탈률** | 이탈 고객 / 월초 고객수 | B2C < 5% |
| **매출 이탈률** | (이탈 MRR − 확장 MRR) / 월초 MRR | 음수(net negative) 지향 |
| **취소 세이브율** | 세이브 / 전체 취소 세션 | 25-35% |
| **리텐션 커브** | 코호트별 기간 경과 잔존율 | 평탄화(flatten) 지향 |

**코호트 세분화**: 획득 소스·요금제·가입 기간·이탈 사유별로 이탈 패턴을 쪼개 구조적 취약점을 찾습니다. 리텐션 커브가 초반 급락 후 평탄해지면 건강, 계속 우하향이면 PMF·가치 전달 문제.

### 리텐션 안티 패턴

- 사후 윈백에만 의존, 조기 신호·사전 개입 없음 (더 비쌈)
- 죄책감·강요형 메시지 ("정말 떠나실 거예요?")
- 취소 플로우에 세이브 오퍼 없음 (10-15% 세이브 기회 상실)
- 재참여와 윈백을 같은 시퀀스로 처리 (상태가 다름)
- 광고성 재참여 메일에 `(광고)`·수신거부 누락 (정보통신망법 위반)
- 리텐션 커브·코호트 없이 총이탈률만 봄 (원인 구간 못 찾음)

## 워크플로우

### 1단계: 리텐션 목표·대상 확인

개선 지표(이탈률/세이브율/재활성화율), 이탈 유형(자발/비자발), 대상 세그먼트, 현재 라이프사이클 자동화 유무를 확인합니다.

### 2단계: 이탈 진단

이탈 유형 비중, 조기 신호 정의, 헬스 스코어 설계, 코호트별 이탈 패턴(리텐션 커브)을 정리해 어디에 개입할지 정합니다.

### 3단계: 개입 지점 설계

사전 개입(헬스 스코어 트리거), 취소 순간(취소 플로우 + 동적 세이브 오퍼), 사후(윈백 시퀀스) 중 우선순위를 정하고 각 트리거·조건을 정의합니다.

### 4단계: 메시지·시퀀스 작성

재참여·윈백 시퀀스의 시점별 목적·제목·본문 초안, 세이브 오퍼 카피, 라이프사이클 메시지를 작성합니다. **광고성 발송이면 정보통신망법 §50 요건**(제목 `(광고)`·전송자 정보·무료 수신거부·야간 발송 제한)을 반영합니다.

### 5단계: 지표·측정 설계

이탈률·세이브율·재활성화율·리텐션 커브 목표를 설정하고, 세이브 오퍼 유형·카피의 A/B 계획과 오퍼 수용 LTV 추적을 정의합니다.

## 사용 예시

**예시 1**: "구독 서비스 이탈이 심해. 취소 플로우에 세이브 오퍼 붙여줘"
→ 이탈 설문 5-8개 → 사유별 동적 오퍼 → 확인·재활성화 경로 → 세이브율 목표

**예시 2**: "90일 미접속 고객 윈백 이메일 시퀀스 짜줘"
→ 윈백 트리거 → D+0/7/30/90 시퀀스 → 컴백 인센티브 → 정보통신망법 체크

**예시 3**: "이탈 조기 신호로 헬스 스코어랑 사전 개입 캠페인 만들어줘"
→ 선행 지표 정의 → 4등급 스코어 → 등급별 재참여 메시지 → 트리거 자동화

## 출력 형식

- 리텐션 진단 요약 (이탈 유형 비중 · 조기 신호 · 리텐션 커브 해석)
- 헬스 스코어 설계 (지표 가중 · 등급별 대응)
- 취소 플로우 + 동적 세이브 오퍼 매핑
- 재참여 / 윈백 시퀀스 (시점별 목적 · 제목 · 본문 초안)
- 리텐션 지표 KPI 설계 (이탈률 · 세이브율 · 코호트)
- 정보통신망법 컴플라이언스 체크리스트 (광고성 발송 시)

## 주의사항

| 상황 | 대응 |
|------|------|
| 고객 행동·이탈 데이터가 없는 경우 | 업종 벤치마크로 일반 시퀀스 구성 후 실데이터 확보 시 세분화 권고 |
| 광고성/정보성 발송 구분이 모호한 경우 | 보수적으로 광고성 기준(정보통신망법 §50)으로 설계 후 법무 검토 권고 |
| 재구매 주기·구독 요금제·단위경제 계산 요청 | 본 스킬 범위 밖. 아래 책임 경계의 moai-seller 스킬로 안내 |
| 비자발적 이탈의 결제 재시도 정책 요청 | 던닝 카피만 지원, 재시도 로직은 구독 운영 영역으로 위임 |

## 후처리 (필수 체인 — 텍스트 산출물만)

재참여·윈백 이메일 본문·제목, 세이브 오퍼 카피, 라이프사이클 메시지 등 **텍스트 산출물**은 발송 전 반드시 후처리 체인을 거칩니다. 헬스 스코어 표·리텐션 커브·이탈률 KPI 등 구조·수치 산출물은 대상이 아닙니다.

**권장 체인(텍스트)**: `marketing-retention(텍스트) → moai-coworker:general-ai-slop-reviewer → moai-writer:general-humanize-korean`

## 관련 스킬

| 스킬 | 사용 시점 |
|------|----------|
| `moai-marketer:content-email-sequence` | 윈백·재참여를 실제 이메일 자동화 시퀀스로 제작 (법적 요건 포함) |
| `moai-marketer:marketing-campaign-planner` | 리텐션을 전체 캠페인·CRM 전략으로 확장 |
| `moai-marketer:marketing-performance-report` | 리텐션 지표·코호트 성과 보고서 |
| `moai-seller:commerce-repurchase-timer` | 구매 주기 기반 재구매 타이밍(커머스 구매 시점 예측) |
| `moai-seller:commerce-subscription-strategist` | 구독 운영·요금제·플랜 설계 |
| `moai-seller:commerce-ltv-cac-architect` | 단위경제(LTV/CAC) 모델링 |
| `moai-coworker:general-ai-slop-reviewer` | 사후 체이닝(필수, 텍스트): 리텐션 메시지의 AI 패턴 검수 |
| `moai-writer:general-humanize-korean` | 사후 체이닝(필수, 텍스트): general-ai-slop-reviewer 다음 단계로 한국어 AI 티 제거 |

## 이 스킬을 사용하지 말아야 할 때

- 구매 주기 기반 재구매 타이밍 예측: `moai-seller:commerce-repurchase-timer` 사용
- 구독 요금제·플랜·구독 운영 설계: `moai-seller:commerce-subscription-strategist` 사용
- 단위경제(LTV/CAC) 계산·모델링: `moai-seller:commerce-ltv-cac-architect` 사용
- 실제 이메일 자동화 시퀀스 제작(법적 요건 포함): `moai-marketer:content-email-sequence` 사용
