# context-collector.md — 맥락 수집 프로토콜 (project 스킬)

## 개요

project 스킬의 스킬 체인 실행에 필요한 사용자 맥락을 체계적으로 수집하는 프로토콜이다. 맥락 충분성 등급, 심화 인터뷰, 모호성 감지, 반복 제한을 통해 효율적인 수집을 구현하고, 수집된 맥락의 **적용 규칙**(§5)까지 정의한다.

---

## 1. 맥락 충분성 등급

### A등급 — 필수 (프로젝트 CLAUDE.md에서 즉시 획득)

프로젝트 `CLAUDE.md`의 "프로젝트 개요" 섹션에서 읽어온다: 프로젝트 목적, 주요 산출물, 대상 독자·고객, 톤·형식 제약, 설치 플러그인. **글로벌 프로필은 사용하지 않는다** — 프로젝트 `CLAUDE.md`가 유일한 맥락 소스다.

### B등급 — 핵심 (80% 이상 충족 권장)

산출물별 도메인 맥락. 선택한 스킬(체인)에 맞춰 Phase 1 인터뷰 + 필요 시 추가 `AskUserQuestion`으로 수집한다.

**질문 생성 규칙**: (1) 선택한 스킬 체인이 무엇을 만드는지 파악 → (2) 이미 아는 맥락은 재질문하지 않음 → (3) 부족한 도메인 맥락만 `AskUserQuestion`으로 보강 → (4) 질문 횟수는 고정하지 않고 상황에 맞게 판단한다.

### C등급 — 보강 (심화 인터뷰로 수집)

팀 규모, 업무 일정/마감, 최근 트렌드, 주요 과제, 기술 스택, 예산 제약.

---

## 2. 맥락 수집 플로우

### 2-1. 초기 평가

```
context = (CLAUDE.md "프로젝트 개요") + (Phase 1 인터뷰 답변)
missing_context = A등급 + 필수_B등급 - context

IF missing_context.empty(): 즉시 실행
ELSE: 부족분만 AskUserQuestion으로 보강
```

### 2-2. 모호성 감지

응답이 "기타"이거나 확신도가 낮으면 follow-up 질문을 1개 추가한다.

### 2-3. 심화 인터뷰 (Deep Interview)

맥락 깊이를 B→C등급으로 향상시키기 위해 텍스트 대화 기반 열린 질문을 최대 2개 수행한다: [동기 탐색] "이 작업이 필요하게 된 배경은?", [전제 확인] "그렇게 판단하신 근거는?", [대안 탐색] "다른 접근도 고려해 보셨나요?", [영향 범위] "누구에게 어떤 영향을 줄까요?", [제약 발견] "꼭 피해야 할 것이 있나요?". `AskUserQuestion`이 아닌 일반 텍스트 대화로 수행하며, 사용자가 "빨리 진행해"라고 하면 즉시 실행으로 전환한다.

### 2-4. 충분성 재평가

충분성 80% 이상이면 실행 시작, 60-80%면 "일부 정보 부재, 진행하시겠습니까?" 확인, 60% 미만이면 재질문.

---

## 3. 반복 제한 및 방지

질문 개수·라운드 수를 고정하지 않는다. 맥락이 충분(A+B등급)해지면 종료한다. 라운드가 7회를 넘어도 충분성이 낮으면 "맥락을 더 수집할까요, 지금 진행할까요?"로 확인한다. `./CLAUDE.md` 프로젝트 개요 + 현재 세션 수집 맥락을 우선 재사용해 반복 수집을 방지한다.

---

## 4. 모호성 감지 규칙

신호(다중 선택 "기타", 저신뢰도 응답, 상충 답변, 도메인 키워드 부재, 목적-요청 불일치)가 1개면 단순 follow-up(1질문), 2개+면 다중 follow-up(최대 2질문) 또는 수동 입력 제안. 해소 시도가 2회를 넘으면 "일부 정보 불명확하지만 진행 가능합니다"로 전환한다.

## 5. 저장 및 적용

### 5-1. 저장 위치

수집된 맥락은 **Phase 6에서 CLAUDE.md 본문**(프로젝트 개요·스킬 체인 섹션)에 기록되며, 별도 컨텍스트 파일이나 글로벌 프로필을 만들지 않는다. 인터뷰 답변 → `CLAUDE.md` "프로젝트 개요"에 반영. 산출물별 스킬 체인 → `CLAUDE.md` "프로젝트 워크플로우" 섹션에 기록. 세션 종료 후에는 `CLAUDE.md`가 단일 소스다.

### 5-2. 맥락 적용 규칙 (HARD — 수집된 맥락의 사용)

저장된 맥락(`CLAUDE.md` 프로젝트 개요 + `.moai/context.md` + 세션 임시 저장)을 **사용**할 때의 규칙:

1. **선택적 적용** — 맥락은 현재 요청과 관련 있을 때만 골라 쓴다.
2. **자연스러운 반영** — 적용할 때는 원래 알던 것처럼 녹인다. 어디서 가져왔는지 티 내지 않는다.
3. **메타 코멘터리 금지** — "저장된 맥락에 따르면", "제가 기억하기로는", "프로젝트 개요를 보니" 같은 회수 서술을 응답·산출물에 쓰지 않는다.
4. **재진입(resume) 세션 주의** — 설치 후 `/project resume`으로 돌아온 세션은 캐시된 맥락을 다시 읽는 순간이라 메타 코멘터리가 특히 빈발한다. 재개 인사에서도 맥락은 자연스럽게만 반영한다.

이 규칙은 생성 `CLAUDE.md`의 `맥락 적용 규칙` HARD 블록과 쌍이다(`claudemd-generator.md` 참조) — 셋업 시점(본 파일)과 런타임(생성 CLAUDE.md) 양쪽에서 같은 규칙이 적용된다.

---

## 6. 컨텍스트 갱신 트리거

사용자가 `CLAUDE.md`를 직접 수정한 경우, 새 산출물 유형(스킬 체인)을 추가하는 경우, 평가 점수가 낮은 경우, 사용자가 명시적으로 재수집을 요청한 경우.
