# 한국어 윤문 처방집 (Rewriting Playbook)

탐지한 AI 티 구간을 실제 문장으로 고칠 때 따르는 전환 규칙집. 분류 체계 SSOT인 `ai-tell-taxonomy.md`의 각 패턴별 처방을 **실행 가능한 치환 레시피**로 확장한다.

## 0. 대원칙 (The Prime Directives)

1. **의미 불변(Fidelity)**: 사실·주장·수치·고유명사·인용·인과관계는 글자 단위로 보존한다. 모호해도 임의 보강 금지.
2. **톤 유지(Tone Match)**: 입력이 격식체면 격식체로, 에세이면 에세이로. 윤문이 원래 글을 "다른 장르"로 바꾸지 않는다.
3. **국소성(Locality)**: 문장을 한꺼번에 전부 재작성하지 않는다. AI 티가 있는 구간만 수술적으로 고친다.
4. **자연성 우선, 완벽성 차순(Natural > Perfect)**: 과하게 문학적으로 고치지 않는다. 일상 한국어 필자의 중간값 리듬을 목표.
5. **근거 기반(Span-Grounded)**: 모든 변경은 탐지 리포트의 span에 연결된다. 탐지 없는 구간을 건드리지 않는다.
6. **과윤문 경고(Over-Polish Alarm)**: 전체 문장의 50% 이상이 바뀌면 내용이 훼손됐을 가능성이 크다. 변경률을 모니터링.

**카피 모드 예외 (Copywriting Exception)**: 마케팅 카피·헤드라인·CTA·랜딩페이지·슬라이드·브랜드 스토리에서는 "의미 불변"과 "과윤문 경고"를 다음과 같이 해석한다:
- **사실 앵커 보존** — 수치·날짜·가격·고유명사·제품명·법적 표기·직접 인용은 글자 단위로 보존.
- **핵심 약속의 의미 보존** — 브랜드의 중심 메시지·혜택·가치제안의 의미는 유지하되, 표현·문장 구조는 재작성 가능.
- **구조 재편성 허용** — 같은 혜택을 더 강하게, 더 짧게, 더 구체적으로. 3인칭→2인칭 전환, 헤드라인 전면 재작성 포함.
- **변경률은 기록만** — 변경률 30%/50% 가드는 카피 모드의 경고·중단 트리거가 아니다. 감사 기록(audit trail)으로만 남긴다. 대체 가드 = 사실 앵커 체크 (SKILL.md 4대 철칙·Phase 5 카피 모드 등급 기준과 정합).
- **예**: "자동화가 나 대신 일합니다" → "프롬프트 1도 몰라도, 여러분의 컴퓨터가 AI 직원이 됩니다" (사실 앵커·약속 유지, 구조·인칭·톤 재편성 — 변경률이 50%를 넘어도 정상).

## 1. 카테고리별 치환 레시피

### A. 번역투 레시피

| 원문 패턴 | 윤문 예시 |
|-----------|----------|
| X에 대해 논의한다 | X를 논의한다 / X를 이야기한다 |
| X를 통해 Y한다 | X로 Y한다 / X해서 Y한다 / X함으로써 Y한다 |
| X에 있어서 | X에서 / X를 볼 때 / X에서는 |
| X라는 점에서 | X해서 / X라는 이유로 / X이기 때문에 |
| X와 관련하여 | X에서 / X에는 / X를 두고 |
| X에 기반하여 / X을 바탕으로 | X로 / X를 근거로 / X를 보고 |
| 경쟁력을 가지고 있다 | 경쟁력이 있다 / 경쟁력이 강하다 |
| 판단되어진다 | 판단된다 / 판단한다 |
| AI에 의해 생성된 | AI가 만든 |
| 높일 수 있다 | 높인다 (사실 서술일 때) / 높일 여지가 있다 (가능성일 때) |
| X을 위해 Y한다 | X하려고 Y한다 / X하도록 Y한다 |
| 합의가 이루어졌다 | 합의했다 / 합의에 이르렀다 |
| 기술 발전 속도 가속화 | 기술의 발전 속도가 빨라진다 |
| 그리고 (문두) | (삭제) / "-고" 연결어미로 압축 |

#### v2.2~v2.5 카피·IT 장르 번역투 레시피 (A-20~A-25)

워크플로우·자동화·시스템 주제 카피에서 AI가 반복 생산하는 실전 번역투. 일반 산문보다 카피·헤드라인·CTA·렌딩문에서 빈도 높음. v2.2(A-20~A-23) → v2.4(A-24 직역 부사) → v2.5(A-25 인칭 호회) 순 확장.

| 원문 패턴 | 윤문 예시 | 패턴 ID |
|-----------|----------|---------|
| 말 한마디에 굴러가는 자동화 (← run/roll) | 말 한마디로 작동하는 AI 직원 | A-20 (+ 구체 비유) |
| 자동화가 굴러간다 / 굴리는 네 가지 (← run/roll) | 자동화가 작동한다 / 만드는 네 가지 | A-20 |
| 무엇으로 굴러가나요? (← what runs it) | 어떻게 작동하나요? | A-20 |
| 흩어진 일을 하나의 흐름으로 | 흩어진 일을 하나로 이어서 | A-21 |
| 하나의 흐름이 됩니다 / 흐름으로 잇습니다 | 하나로 이어집니다 / 하나로 이어 씁니다 | A-21 |
| 한 흐름으로 잇는 자동화 설계자 | 하나로 이어 설계하는 자동화 설계자 | A-21 |
| 같은 워크플로우가 일을 시작합니다 | 같은 흐름으로 일이 이어집니다 | A-21 (+ 워크플로우→흐름) |
| 자동화가 나 대신 일합니다 / 나와 함께 일합니다 | 자동화가 대신 일해 줍니다 | A-22 |
| 하나의 워크플로우로 손발을 맞춥니다 | 한 번에 맞물려 움직입니다 | A-23 (직역 관용구 → 동사) |
| X는 Y의 엔진이다 / 심장이다 / 날개를 달다 | X가 Y를 움직인다 / 이끈다 / 빠르게 한다 | A-23 (metaphor → 직접 동사) |
| 결과물의 얼굴을 만들고 (← face of) | 결과물의 톤과 스타일을 디자인하고 | A-23 (face 직역 비유) |
| 같은 페이지를 본다 (← on the same page) | 같은 방향을 본다 / 맞춘다 | A-23 |
| 워크플로우 (workflow calque) | 흐름 / 자동화 / 과정 (맥락별) | A-21 보조 / B-2 참조 |
| 더는 혼자가 아닙니다 (← no longer alone) | 이제는 혼자가 아닙니다 | A-24 (문두 직역 부사) |
| 더는 걱정하지 않아도 됩니다 | 이제는 걱정하지 않아도 됩니다 | A-24 |
| 자동화가 ~합니다 (헤드라인/CTA) | 여러분이 AI 직원에게 ~을 위임하세요 | A-25 (3인칭→2인칭 호소) |
| 워크숍이 끝나도 자동화가 내 컴퓨터에 그대로 남습니다 | 강의가 끝나면 여러분만의 AI 직원이 생깁니다 | A-25 (3인칭 무주어→2인칭+구체 혜택) |

**업종별 A-20~A-25 레시피 적용 사례**:

| 업종 | 원문(AI 티) | 윤문(A-20~A-25 적용) | 패턴 |
|------|-----------|------------------|------|
| **뷰티 커머스** | 이 제품은 피부 개선의 새로운 흐름을 만듭니다 | 7일 만에 피부톤이 한 단계 밝아집니다 (구체 혜택+기간) | A-21(흐름→달성) + A-25(2인칭 혜택) |
| **SaaS(자동화)** | 더는 이메일 자동화로 혼동하지 않아도 됩니다 | 이제 여러분이 AI 직원에게 이메일 캠페인을 맡기세요 | A-24(더는→이제) + A-25(무주어→2인칭) |
| **푸드테크** | 식재료 주문이 간단한 하나의 과정이 됩니다 | 아침에 주문하면 저녁에 신선한 식재가 도착합니다 | A-21(과정→구체 시간표) |

#### 카피 톤 강화 레시피 (행동 종결·헤드라인·구체 혜택)

단일 패턴 ID로 잡히지 않는 카피 톤 결함 — 평서 종결·약한 헤드라인·추상 혜택 — 의 처방. A-25(2인칭 호회)와 짝으로 쓴다. 출처: 사용자 스펙 `ai-tell-ko-copy-spec.md` 교정 원칙 5·7.

| 원문 패턴 | 윤문 예시 | 적용 원칙 |
|-----------|----------|-----------|
| 2일 만에 완성합니다 (평서 종결) | 2일 만에 위임하세요 (행동 유도) | 행동·결과 중심 끝맺음 |
| 네 가지 (헤드라인 추상 명사 종결) | 프롬프트 1도 몰라도 AI 직원에게 모든 업무를 위임합니다 | 약한 끝맺음 → 구체 혜택 |
| 자동화가 대신 일해 줍니다 (추상 혜택) | 프롬프트/컨텍스트 엔지니어링 1도 몰라도 위임 | 구체 혜택 명시 (측정·체감) |

> **비유 일관성 가드**: 한 번 정한 비유("AI 직원")를 페이지 전체에 일관 적용 — "자동화/AI 팀/AI 직원" 혼용 금지(교정 원칙 8). A-25 2인칭 호회 전환 시 비유를 함께 고정한다.

> **비유·과장·의인화 → 한국인 정서 연계**: A-23(직역 비유)·D-4(과장)·D-5(의인화)로 탐지한 구간은 곧바로 K-1(구체 사실·장면)·K-2(절제·곡언) 양성 처방으로 대체한다 — 비유·과장을 지우기만 하면 밋밋해지므로, 구체 근거로 채우는 것이 한국인 정서 카피의 핵심이다(§ K 레시피 참조). 예: "압도적 성과" → (D-4 탐지) → "기대를 적잖이 넘겼다"(K-2 곡언).

### B. 영어 인용·용어 처방

- **괄호 병기**: 일반 독자 대상 → 첫 등장 1회만 영어 병기, 이후 한국어만. 전문 독자 대상 → 영어 병기 유지하되 매번 반복하지 않음.
- **영어 단어 번역표 (빈출)**:
  - pipeline → 파이프라인 (유지 OK) / 흐름 / 공정
  - framework → 체계 / 틀 / 구조
  - leverage → 활용하다 / 기대다 / 끌어올리다
  - seamless → 매끄러운 / 끊김 없는
  - robust → 튼튼한 / 견고한
  - scalable → 확장성 있는
  - insight → 통찰 / 눈 / 시사점
  - impact → 영향 / 파장
  - holistic → 전체적 / 총체적
- **영어 인용구**: 원문 어감이 핵심이면 유지하고 한국어 번역 병기. 그렇지 않으면 한국어로 풀어쓰고 출처만 각주.

### C. 구조 레시피

- **기계적 병렬 "첫째/둘째/셋째"**: 
  - 열거가 핵심이면 → "우선 / 이어서 / 마지막으로" 등 어휘 변주.
  - 열거가 장식이면 → 산문으로 녹이기: "A다. B도 마찬가지다. 여기에 C가 더해진다."
- **불릿 → 산문 전환 예시**:
  - 원문:
    ```
    - 속도가 빠르다
    - 비용이 저렴하다
    - 확장성이 높다
    ```
  - 윤문: "속도는 빠르고 비용도 낮다. 무엇보다 확장 여지가 크다."
- **헤딩 제거**: 에세이·칼럼에서는 H2 이상 헤딩 자체를 없애고 문단 간 흐름으로 처리.
- **문단 첫 문장 요약 공식 해체**: 매 문단이 topic sentence로 시작하지 않도록, 일부 문단은 장면·수치·질문으로 시작.
- **이모지 전량 삭제** (에세이·리포트 문맥). 제품 카피·SNS면 유지 가능.

### D. 관용구 처방 (삭제 우선)

| 삭제 대상 | 대안 |
|----------|------|
| 결론적으로 | (삭제) — 마지막 문단 자체가 결론이므로 라벨링 불필요 |
| 요약하면 / 정리하자면 | (삭제) 또는 "한 줄로 말하면" |
| ~라고 할 수 있다 | ~이다 (단언 가능하면) / ~로 보인다 (관측이면) |
| 매우 중요하다 | 구체 근거로 대체: "X 없이는 Y가 성립하지 않는다" |
| 시사하는 바가 크다 | (삭제) 또는 "의미는 분명하다" |
| 주목할 만하다 | (삭제) — 이미 문장이 주목하게 만드는 내용이면 불필요 |
| 혁신적인 / 획기적인 | 대부분 삭제. 필요하면 "처음 시도한" / "이전과 다른" 같이 구체화 |
| ~의 지평을 열다 / ~시대가 도래했다 | 삭제 후 실제 변화를 서술 |

### E. 리듬 처방

- **입력 분석**: 탐지 단계에서 평균 문장 길이·표준편차를 계산한다 (E-1 시그널, `metrics.py`로 측정).
- **균일성 감지 시 처방**:
  - 단문(10-15자) 1-2개를 문단마다 투입: "맞다. 그게 핵심이다."
  - 긴 문장(80자+) 1개 허용.
- **종결어미 변주**: 4-5문장 연속 같은 종결어미 사용 금지. "~다 / ~았다 / ~인 것 / 명사형 종결"을 섞음.

### F. 수식 처방

- 정도부사("매우", "정말", "대단히") → 기본 90% 삭제. 강조가 필요하면 구체 수치·비교.
- 동의어 이중 수식("중요하고 핵심적인") → 하나만.
- "~적 / ~성 / ~화" 접사 → 구체 동사·명사로 풀기.
  - "근본적 변화" → "뿌리부터 바뀐다"
  - "구조적 문제" → "구조가 문제다" / "구조 자체가 문제다"
- "-tion·-ment·-ness·-ity" 영어 명사화 직역 → 동사·형용사로 풀기 ("the implementation of the policy" → "정책 시행" / "정책을 시행하기").

### G. Hedging 처방

- **완곡 3단계 다운그레이드**:
  - "~할 수 있을 것으로 보인다" → "~로 보인다" → "~일 것이다" → "~이다"
- 단언 가능 지점이면 두세 단계 내려가서 단언. 사실 모호할 때만 1단계 완곡 유지.

### H. 접속사 처방

- 문두 접속사 3개 이상 연속 → 70% 삭제.
- "또한" → 대부분 삭제. 꼭 필요하면 "여기에"·"거기에"·"더해" 등 어휘 변주.
- "따라서 / 그러므로" → 인과가 자명하면 삭제. 필요하면 "그래서"로 교체.
- "하지만 / 그러나" 반복 → 교차 사용하거나 한쪽을 "그런데"로.

### I. 형식명사 처방

- "것이다" 종결 → 종결어미 직결로.
  - "변화가 크다는 것이다" → "변화가 크다"
- "~할 필요가 있다" → "~해야 한다" / "~할 만하다" / 구체 행위 지시.
- "~이 필요하다" → 주어·동사로 구체화. "혁신이 필요하다" → "이 회사가 제품을 다시 만들어야 한다" (맥락 허락 시).

### J. 장식 처방

- **볼드**: 본문에서 거의 전량 제거. 목차·제목급에만 허용.
- **따옴표**: 인용·특수 용례에만 한정.
- **대시(—)**: 1문서 1-2회 이하. 나머지는 쉼표·괄호·문장 분리.

### K. 한국적 정서·결 레시피 (양성 축)

A~J가 "고칠 것"이라면 K는 "더할 것"이다. 탐지 리포트에 K-1~K-4 신호가 있거나 글이 매끄럽지만 밋밋하면 아래 레시피로 한국적 정서를 채운다. **장르 가드** — 격식 보고서·공문·계약서는 K-1만 약하게, K-3는 적용하지 않는다.

| 신호 | 처방 한 줄 | 예 |
|------|-----------|-----|
| K-1 정서 온도 0(추상·일반론만) | 추상 단언마다 구체 장면·수치·사례 1개(일반론 1:구체 2). 대명사(A-16) 없이 상황으로 공감 | "고객 경험이 중요하다" → "한 번 불편을 겪은 손님은 돌아오지 않는다" |
| K-2 과장(D-2·D-4) | 강조어→사실+여백, 무게는 곡언(이중부정·완서법)으로 | "압도적 성과" → "기대를 적잖이 넘겼다" |
| K-3 빳빳한 완결(에세이·SNS) | 일부 문장을 말끝 흐림·여백·미완결로(격식문 제외) | "그게 문제입니다" → "그게 제일 걸리더라고요" |
| K-4 요약 서두+공식 결말 | 서두=장면·질문·공감·반전·결핍 / 결말=여운·구체 단언 | "~할 때입니다" → "결국 자리가 아니라 신뢰의 문제였다" |

> **K 가드 3원칙**: (1) 격식 장르 과적용 금지(K-1 약하게·K-3 미적용), (2) A~J와 충돌 시 A~J 우선(정서 살린다고 대명사·과장 늘리지 않음), (3) 변경률 30% 가드 유지 — K를 핑계로 다른 글로 바꾸지 않는다. **화계 거리(E-7 3단계)**: 격식 합니다 / 정중 해요 / 친근 해체 중 독자·장르에 맞는 한 단계를 끝까지 유지하되, 거리 조절이 정서를 만든다.

### L. 스토리텔링 처방 (L-1~L-8, ai-tell-taxonomy SSOT)

AI 글쓰기에서 자주 감지되는 스토리텔링 패턴별 처방. `ai-tell-taxonomy.md`의 L 카테고리 SSOT를 참고.

#### 패턴별 처방 (한 줄 수정 가이드)

| L 패턴 ID | 탐지 신호 | 한 줄 처방 |
|-----------|----------|-----------|
| **L-1: 클리셰 도입** | "어느 날", "그렇게 ~는 시작됐다" 식 전형 도입 | 구체 날짜·실제 사건(예: "2019년 3월, 보증금 500만원짜리 원룸이 창고가 됐다") |
| **L-2: 강요된 감정** | 신체감각 + 직접 명명 동시 ("눈물이 흘렀고 감동했다") | 관찰 장면으로만 (예: "편지에는 '재구매하겠다'는 문장이 세 번 반복돼 있었다") |
| **L-3: 교훈 강박** | "이를 통해 저희는 ~을 배웠습니다" | 실제 행동으로 전환 (예: "그 실패 이후 반품 프로세스를 바꿨다. 지금은 90%가 24시간 내 처리된다") |
| **L-4: 클리셰 전환구** | "그날 이후", "그 순간", "운명처럼" | 구체 시간/원인으로 (예: "그 주문 이후 주당 배송량이 3배로 늘었다") |
| **L-5: 지나치게 매끄러운 결말** | "모든 어려움을 극복하고 완벽을 이루었다" | 진행형/불완전성 (예: "지금도 매달 1~2건은 반품이 온다. 원인을 기록해 다음 생산에 반영한다") |
| **L-6: 창업자 신화** | "작은 원룸에서 시작해 오늘날 우리가 되었다" | 검증 가능 디테일 (예: "2020년, 보증금 500만원짜리 원룸이 창고 겸 사무실. 택배 상자를 침대 밑에 쌓아두고 잤다") |
| **L-7: 가짜 구체성** | "114편의 논문과 후기 2042건 분석", 확인 불가 전문성 | 실제 구체성 (예: "재구매 고객 120명 설문에서 87명이 '배송 속도'를 이유로 꼽았다") |
| **L-8: 광고처럼 읽히는 후기** | 동일 포맷 반복, 전문직 코스프레, "문의가 너무 많아서" | 실제 고객의 구체 장면 (예: "김OO 님은 매주 화요일마다 재주문한다. 이유를 물으니 '설거지가 편하다'고 했다") |

#### 긍정 원칙: 브랜드 서사 양성 처방

L 패턴 삭제만으로는 밋밋해진다. 아래 9개 원칙으로 강력한 브랜드 서사를 **작성**한다:

1. **고객을 주인공으로, 브랜드는 가이드로** — 브랜드 자신이 주인공이 되는 것이 가장 흔한 실수.
2. **완벽 대신 갈등·결핍** — "우리는 완벽합니다" 자랑이 진부함의 원인. 해결 과정이 본체.
3. **검증 가능한 구체성 우선** — 숫자·실제 프로세스·인용구(창업 과정, 페르소나, 의사결정)가 "진짜다움"을 만듦.
4. **TMI 나열 금지** — 제품 스펙 설명 대신 "왜 신경 쓰는지, 어떻게 돕는지, 어떤 과정을 거쳤는지".
5. **문학적 완결성 강박 버리기** — 창업 스토리는 소설 아님. 기승전결 억지 완비 불필요.
6. **창업자 서사 → 브랜드 서사 확장** — 성장 시 고객 중심으로 유연한 진화 (에어비앤비: "에어매트리스"→"소속감").
7. **"빌딩 인 퍼블릭" 톤 관리** — 도전·실패 공유는 진정성. 톤 조절 실패 시 브랜드 가치 훼손.
8. **지킬 수 있는 가치만 약속** — 약속-이행 일치가 진정성을 만든다.
9. **다양한 버전 A/B 테스트** — 하나의 스토리 고집 금지. 주요 고객층별로 다른 앵글 시도.

### M. 슬라이드·프레젠테이션 처방 (M-1~M-3, ai-tell-taxonomy SSOT)

AI 글쓰기에서 감지되는 슬라이드 구조 패턴별 처방. `ai-tell-taxonomy.md`의 M 카테고리 SSOT를 참고.

#### 패턴별 치환 방향

| M 패턴 ID | 탐지 신호 | 치환 방향 | 예 |
|-----------|----------|---------|-----|
| **M-1: 대시 대비 헤드라인** | 대시(—)로 "X — Y" 두 부분을 분할하는 제목 | 대시 제거, 한 문장 통합 또는 두 문장 분리 (파이프·콜론 등 다른 분할 기호로 대체 금지) | "복붙에서 위임으로 — 목표만 주면" → "목표만 주면, AI 직원이 나머지를 처리합니다" |
| **M-2: 조사·체언 종결 조각문** | 서술어 없이 조사·명사형으로 끝나는 제목 ("성공의 열쇠") | 서술어 포함 완전문 또는 완결 명사구로 | "성공의 열쇠 — 자동화" → "자동화가 성공의 열쇠입니다" |
| **M-3: "A에서 B로" 전환 공식** | "엑셀에서 노션으로"류 전환 공식으로 여는 오프닝 | 구체 사례·수치·변화로 시작 | "엑셀에서 노션으로, 바뀐 것" → "노션 전환 후: 자동 갱신 1초, 협업 시간 50% 단축" |

#### 명사구·불릿 밀도 가이드

- **장당 메시지 수** — 슬라이드 1장 = 1 메시지. 불릿 3±1개, 각 불릿은 근거·수치 위주 (읽기 인지 부하 관리).
- **명사구 허용 경계** — 완결형 명사구 제목("2026년 Q1 사업 보고")은 허용, 서술이 잘린 조각문("성공의 열쇠", "하나의 흐름으로")은 금지. 경계 사례는 taxonomy M 카테고리 참조.
  - ❌ "지속 가능한 비즈니스 모델 혁신을 위한 기술 투자 전략의 우선순위 결정 프레임워크" (과밀 명사구)
  - ✅ "기술 투자 우선순위: 비용·효과·위험" (완결 라벨)
- **이미지 + 텍스트** — 이미지가 있으면 텍스트는 라벨 수준(10자 내외), 없으면 완결 헤드라인 1줄 + 부연 1-2줄.
- **발표 자막·스크립트는 구어(카피 모드), 도표 라벨·출처 표기는 정보성(불변).**

## 2. 변경률 모니터링

### 산문 모드 (산문·칼럼·리포트·에세이)

- 변경 전후 텍스트의 **레벤슈타인 거리 / 원문 길이**를 계산해 변경률을 기록한다.
- 권장 범위: 5-30%.
- 30% 초과: 과윤문 가능성 → 재검토.
- 5% 미만: 저윤문 → S1 패턴이 남아 있는지 재확인.

### 카피 모드 (마케팅 카피·헤드라인·CTA·랜딩페이지·슬라이드·브랜드 스토리)

- 변경률 계산은 동일(레벤슈타인 거리)하되 **감사 기록(audit trail)으로만** 남긴다 — 권장 범위·경고·중단 트리거가 없다.
- 이유: 헤드라인·CTA는 단문이라 30%가 몇 글자 수준이고, 정당한 카피 윤문(2인칭 전환·헤드라인 재작성)은 본질적으로 50%를 넘는다.
- **대체 가드** (하나라도 위반 시 해당 edit 롤백):
  1. 사실 앵커 손실 0건 — 수치·날짜·가격·고유명사·법적 표기를 before/after 대조.
  2. 핵심 약속·혜택의 의미 보존 — 쌍 단위 대조.
  3. 정보성 구간 무변경 — 정보성/호소성 분류(SKILL.md Phase 3-2b)에서 정보성으로 판정된 구간은 산문 모드 규칙 적용.

## 3. 어휘 대체 위험 (Do-NOT list)

이들은 문체상 AI 티로 보여도 **건드리면 의미가 바뀌는 표현**이므로 보존한다:

- 전문 고유명사·제품명·모델명(GPT-4, Claude 3, Gemini 등)
- 수치·단위·날짜
- 직접 인용된 문장(큰따옴표 "" 내부)
- 법률·규정 조문 인용
- 학술 개념어가 불가피한 경우 (예: "확률적 앵무새", "창발")

| **마케팅 카피** | 행동·결과 중심, 구체 혜택, A-25 호소형 | 추상 명사 종결, 약한 헤드라인 | L-7(가짜 구체성) 강조, A-20~A-25 + K-1 적극 |
| **랜딩페이지** | 신뢰성 신호(숫자·고객평), CTA 명확 | 과장·약속 미달성, 모호한 링크 | L-6~L-8 강조(창업신화·후기·구체), 단문+명사 |
| **슬라이드** | 1장 1메시지, 명사형, 이미지 중심 | M-1~M-3 패턴 전량 제거, 과도한 텍스트 | M-1~M-3 SSOT 적용 + 밀도 가이드 |
| **브랜드 스토리** | 시간축(과거→현재→미래), 실패·갈등, 고객 중심 | L-1~L-6 패턴(클리셰·신화) | L 긍정 원칙 9개 모두 적극 적용, K-1~K-4 필수 |

입력 첫 100자를 읽고 장르를 추정한 뒤 이 표로 허용/금기 선을 조정한다. **K 패턴(한국적 정서·결)은 칼럼·에세이·블로그에 적극 적용, 리포트는 K-1만 약하게, 공적 연설·공문은 K-3·K-4 미적용**(격식·완결 우선).

## 5. 반복 윤문 방침

- 1차 윤문 후 자체검증에서 잔존 S1/S2 패턴이 발견되면 2차 윤문을 진행한다.
- 최대 3회. 3회 후에도 잔존하면 해당 구간을 리포트에 "사람이 직접 확인 요망"으로 표시한다.
