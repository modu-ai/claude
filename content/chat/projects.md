## Projects란? — "주제별 폴더" 같은 개념

Projects는 Claude와의 여러 대화와 파일을 한 주제 아래 모아서 관리할 수 있는 기능입니다.

### 예시

- **프로젝트 1**: "마케팅 캠페인 Q2"
  - 대화 1: 아이디어 브레인스토밍
  - 대화 2: 예산 계획
  - 파일: 분석 자료, 제안서

- **프로젝트 2**: "계약서 검토"
  - 대화 1: 계약서 분석
  - 대화 2: 협상 전략
  - 파일: 계약서, 유사 사례

## Project 만드는 법

1. Claude Desktop 사이드바 상단의 **Create New Project** 클릭
2. 프로젝트 이름 입력 (예: "분기별 예산 계획")
3. 선택사항: 프로젝트 설명 추가
4. **Create** 클릭
5. 이 프로젝트 내에서 대화 시작

## Project의 이점

| 이점 | 설명 |
|---|---|
| **문맥 유지** | 같은 Project 내 모든 대화가 서로의 맥락을 기억 |
| **파일 정리** | 한 주제의 파일을 모두 한 곳에 모음 |
| **재사용** | 이전 대화를 쉽게 찾아 이어서 작업 |
| **팀 공유** (Team 플랜) | 같은 Project를 여러 사람이 함께 사용 가능 |

## 3가지 실제 사용 사례

### 사용 사례 1: 업무 프로젝트

**프로젝트 이름**: "신제품 런칭 2026"

```
대화 1: "신제품의 주요 기능 3개와 가치 제안을 정리해 줄 수 있나요?"
대화 2: "위에서 정한 기능을 바탕으로 마케팅 캠페인 개요를 만들어 줄 수 있나요?"
대화 3: "위 캠페인에 맞는 SNS 포스트 5개를 작성해 줄 수 있나요?"

파일: 제품 스펙 → 마케팅 전략 → 최종 포스트
```

### 사용 사례 2: 학습 주제

**프로젝트 이름**: "Python 배우기"

```
대화 1: "Python 기초를 알려 줄 수 있나요?"
대화 2: "리스트와 딕셔너리의 차이를 설명해 줄 수 있나요?"
대화 3: "간단한 데이터 분석 코드를 보여 줄 수 있나요?"

파일: 학습 자료, 코드 예시, 연습 문제
```

### 사용 사례 3: 글쓰기 프로젝트

**프로젝트 이름**: "사업계획서 작성"

```
대화 1: "사업계획서의 구조를 알려 줄 수 있나요?"
대화 2: "Executive Summary를 작성해 줄 수 있나요?"
대화 3: "시장 분석 섹션을 보강해 줄 수 있나요?"

파일: 시장 데이터, 작성 중인 문서, 참고 자료
```
