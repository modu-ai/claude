# 마스터 기획서 골격 + 에피소드 현황표 스키마

> `[작품제목]_기획서.md` 한 파일이 시리즈의 마스터 문서다. 아래 골격으로 저장하고, 맨 아래 **에피소드 현황표**를 회차마다 갱신한다.

## 1. 마스터 기획서 골격

```markdown
# [작품제목] 기획서

> 생성: story-series-bible
> 라인: 웹툰 / 웹소설

## 독자 페르소나
- 이름(가상) / 나이·직업
- 웹툰 보는 상황 (언제·어디서·어떤 기분)
- 이 작품에서 원하는 것 (어떤 감정을 충전하러 오는가)
- "다음화 버튼"을 누르는 이유

## 세계관
- 공간 / 시간대 / 분위기 / 세계관 규칙
- 공간 상세 (소품·색감·조명·계절 영향·독자에게 주는 감정)

## 캐릭터
### 주인공
- 서사 정보: 이름 / 성격 키워드(3개 이내) / 역할 / 말투 / 독자가 좋아하는 이유
- 비주얼 시트: 헤어·얼굴/눈·체형·피부톤·기본 복장·상징 소품·전체 인상 한 줄
- ★ 일관성 앵커: 화마다 고정할 1~2가지 (예: "은발 + 왼쪽 눈 밑 점")

### 레귤러 조연 (최대 5명)
- 이름 / 성격 / 등장 빈도 / 서사 역할 / 비주얼 시트 / 일관성 앵커

### 게스트 원칙
- 옴니버스면 그 화 주인공. 전체 기획서엔 이름·역할만, 외형 상세는 에피소드 원고에.

### 캐릭터 관계도

## 시리즈 아크
| 구간 | 화수 | 역할 | 핵심 사건 | 감정 톤 |
|------|------|------|----------|--------|
| 도입부 | 1~3화 | 세계관·캐릭터 소개 | | |
| 전개부 | 4~10화 | 관계 형성·갈등 시작 | | |
| 클라이맥스 | 11~15화 | 핵심 갈등 폭발 | | |
| 해결부 | 16~20화 | 해소·성장 | | |

## 관통 주제
> "이 작품이 결국 하고 싶은 말은 ___이다."

## 훅 전략
| 훅 유형 | 설명 |
|--------|------|
| 감정 훅 | 이 감정이 더 필요해 |
| 궁금증 훅 | 무슨 일이 일어날지 궁금해 |
| 캐릭터 훅 | 이 캐릭터가 더 보고 싶어 |
| 공감 훅 | 나도 저런 적 있는데 |
원칙: 1~3화는 캐릭터 훅 + 감정 훅. 중반부터 궁금증 훅 점진 심기.
(수익화 훅 정렬은 story-webtoon-spec:references/monetization-hooks.md)

## 에피소드 현황
| 화 | 상태 | 제목 | 핵심 감정 | 훅 포인트 | 파일 | 날짜 |
|----|------|------|-----------|-----------|------|------|
```

## 2. 에피소드 현황표 스키마

| 열 | 뜻 |
|----|-----|
| 화 | ep## (2자리) |
| 상태 | ⬜ 예정 / 🟡 작성중 / ✅ 작성됨 |
| 제목 | 에피소드 제목 |
| 핵심 감정 | 이번 화가 남기는 감정 |
| 훅 포인트 | 다음 화를 궁금하게 만드는 장치 요약 |
| 파일 | `[작품제목]_EP##_[에피소드제목].md` |
| 날짜 | 작성일 |

## 3. 갱신 규칙

- 회차 원고를 만들면 해당 행을 **✅ 작성됨**으로 갱신하고 파일명·날짜를 채운다.
- 예정 회차는 미리 ⬜로 깔아 두면 아크 진행을 한눈에 본다.
- **에피소드 생성 = 현황표 갱신, 항상 세트.** (SKILL.md §4 불변식)
