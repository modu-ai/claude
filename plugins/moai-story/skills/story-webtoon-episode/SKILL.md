# story-webtoon-episode: 회차 플롯 + 컷 분할 + 컷 연출

한 회차의 **리듬을 설계**하는 스킬이다. 회차 플롯을 짜고, 세로 스크롤 컷으로 나누고, 컷마다 연출을 지시한다. **말풍선·식자는 이 스킬이 하지 않는다** — 컷별로 어떤 말풍선·SFX가 필요한지 자리만 잡고, 실제 스니펫·서체·배치는 `story-webtoon-lettering`으로 넘긴다.

## 1. 개요

이 스킬이 결정하는 것은 **무엇을 보여줄지(컷)**와 **언제 감정을 밀고 당길지(리듬)**다. 세로 스크롤은 페이지 그리드가 아니라 **위→아래 연속 스크롤**이므로, 페이지 만화의 "6칸 그리드·거터·왼→오 읽기" 전제를 쓰지 않는다. 컷 높이와 여백 띠(칸새)가 호흡의 레버다.

### 비주얼-서사 분리

컷 대본에는 컷 연출·시각 지시를 담고, 성격·내면 같은 서사 정보는 시트(`story-character-sheet`)·기획서(`story-series-bible`)에 둔다. 스토리 본문에 시각 정보를 나열하지 않는다 — 이야기는 이야기로, 그림 지시는 컷 표로.

## 2. 이 스킬이 하지 않는 것

- **말풍선·식자·SFX 안 함** — 컷에 필요한 자리만 표시(`💬 필요` 등)하고, 종류·꼬리·서체·SFX 스타일은 `story-webtoon-lettering`이 소유한다. (업계 비표준어 "말칸"은 쓰지 않는다 → **말풍선**.)
- **작화 프롬프트 조립 안 함** — 컷 지시를 이미지 프롬프트로 옮기는 것은 `story-webtoon-art`.
- **규격 수치 단정 안 함** — 회차 컷 수 범위·시즌 화수는 `story-webtoon-spec` 허브 참조.
- **상태 저장 안 함** — 회차 원고 저장·현황표 갱신은 `story-series-bible`.

## 3. 사전 확인 — 규격 참조

- `${CLAUDE_PLUGIN_ROOT}/skills/story-webtoon-spec/references/episode-volume.md` — 회차 컷 수 범위·연재 주기. 컷 수의 상·하한 관행은 여기서 확인한다(임의 숫자 금지).
- `${CLAUDE_PLUGIN_ROOT}/skills/story-webtoon-spec/references/monetization-hooks.md` — 기다무 해제·미리보기 유료 전환 지점. 회차 끝 훅을 유료 경계와 정렬할 때 참조.

## 4. 워크플로우

### Step 1 — 회차 목표 + 감정 비트

이 회차가 전달할 정보·감정·회차 끝 훅을 한 줄로 정한다. 회차를 여러 **감정 비트**로 쪼갠다(각 비트 = 감정 키워드 + 강도 강/중/약).

### Step 2 — 컷 분할 (컷 수는 감정 비트가 정한다)

**"몇 컷"을 먼저 정하지 않는다.** 각 감정 비트에 맞는 컷을 배치한다 — 정적·충격 비트는 과감히 큰 컷 1개, 빠른 합은 작은 컷 여러 개. 할당량 사고("회차당 N컷")를 버린다.

세로 스크롤의 리듬 레버는 **컷 높이 변화 + 여백 띠(칸새) 길이 + 스크롤 뷰포트 1화면 설계**다. 상세 문법은 `references/cut-grammar.md`.

컷 표를 만든다(회차당 1개):

| 컷 | 장면(무엇을) | 감정강도 | 컷 높이 | 앵글/거리 | 프레임 연출 | 인물/방향/소품 | 💬 식자 자리 |
|----|--------------|----------|---------|-----------|-------------|----------------|--------------|
| 1 | | | | | | | |

### Step 3 — 컷 연출 지시 (프레임 연출 + 인물 배치)

감정 정점 컷에만 프레임 연출·인물 배치를 얹는다. 남발하면 대비가 죽어 오히려 산만해진다. 8종 프레임 연출·5종 인물 배치의 영어 프롬프트와 감정 매핑은 `references/panel-direction.md`.

> ★ **작성 위치 승격 규칙:** 배치·연출 지시는 컷 묘사 문장 안에 묻으면 무시된다. 컷 지시 직후 **`★ 레이아웃:` 줄**로 승격해 쓴다(실측: 묘사 안 삽입 2회 연속 실패 → 페이지 레벨 승격 후 성공). 상세는 `references/panel-direction.md`.

### Step 4 — 훅 배치

훅 4유형(감정·궁금증·캐릭터·공감)을 구간별 강도로 배치하고, 회차 끝 훅을 유료 전환 지점과 정렬한다. 상세는 `references/hook-taxonomy.md` + `monetization-hooks.md` 참조.

### Step 5 — 회차 흐름 점검

모든 컷 확정 후 줌아웃해 회차 전체 리듬을 한 번 점검한다(클라이맥스 위치·감정 곡선·도입 흡인·컷 높이 변주 등 7항목). 점검표는 `references/episode-template.md`.

## 5. 출력 형식

회차 문서 7섹션 템플릿을 따른다. 전체 골격은 `references/episode-template.md`.

```markdown
# [작품제목] EP## — [에피소드제목]

## 이번 화 기본 정보 (화수 / 핵심 감정 / 회차 끝 훅)
## 이번 화 등장 인물 (비주얼 요약 + 일관성 앵커 — 시트에서 가져옴)
## 이번 화 공간·장소
## 회차 플롯 (감정 비트 순서)
## 컷 표 (컷 높이·여백 띠·프레임 연출·💬 식자 자리)
## 훅 배치 (구간별 유형·강도 + 유료 경계 정렬)
## 회차 흐름 점검 (7항목)
```

## 6. 주의사항

- **"말칸" 금지 → 말풍선.** 식자 실무는 `story-webtoon-lettering`으로 넘긴다.
- **컷 수 할당량 사고 금지.** 컷 수는 감정 비트가 정한다 — 세로에서는 컷 높이·여백 띠 길이가 리듬 레버.
- **프레임 연출 남발 금지.** 감정 정점 컷에만. 같은 구간에 프레임 연출과 인물 배치를 겹쳐 쓰지 않는다.
- **연출 지시는 `★ 레이아웃:` 줄로 승격.** 묘사 안에 묻으면 무시된다.
- **규격 수치는 spec 허브 참조.** 컷 수 범위를 지어내지 않는다.
