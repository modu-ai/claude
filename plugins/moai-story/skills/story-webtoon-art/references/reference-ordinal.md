# 첨부 서수 1:1 매핑 + 동적 계산 + 자기검증

> `story-webtoon-art` Step 3에서 로드한다. 참조 이미지(캐릭터 ref·스타일 그리드·직전 컷)를 첨부할 때, 헤더의 첨부 순서와 프롬프트 내부 서수를 글자 그대로 일치시키는 표준. 이게 흔들리면 어떤 이미지를 어떤 순서로 첨부할지 알 수 없어 프롬프트가 무용지물이 된다.

## 고정할 것 (형식 — 항상)

- 각 컷 헤더에 **`첨부 순서: ① … → ② …`** 줄을 반드시 넣는다.
- 코드블록(프롬프트) **첫 문단 = 첨부한 이미지를 순서대로 "N번째 이미지는 X입니다"로 1:1 매핑**.
- **헤더 첨부 순서 ↔ 코드블록 서수가 개수·순서·이름까지 글자 그대로 일치**.
- 캐릭터는 각각 **개별 서수**로. 뭉뚱그리기 금지("첫 이미지들은 인물·고양이" ❌).

## 유동적인 것 (내용 — 매번 다름)

- **첨부 개수·종류는 컷마다 다르다.** 인물 1~N명 / 스타일 그리드가 없을 수도 / **첫 컷은 직전 결과가 없다.**
- **실제 첨부한 것만, 실제 순서대로** 서수를 다시 세서 채운다. 골든 견본의 "2명·스타일·직전"을 그대로 복제하지 않는다 — 형식만 복제하고 개수는 유동적으로.

## 동적 서수 계산 공식

```
직전 컷 서수 = 인물 수 + (스타일 그리드 있으면 1) + 1
스타일 없으면 그만큼 당긴다.
```

## 서수 변형표 (형식은 동일 · 첨부 구성만 다름)

| 상황 | 첨부 순서(헤더 = 코드블록 서수) | 직전 결과 서수 |
|------|------|------|
| 인물1 · 스타일 · 직전 | ① 인물 → ② 스타일 → ③ 직전 | 3번째 |
| 인물2 · 스타일 · 직전 | ①② 인물 → ③ 스타일 → ④ 직전 | 4번째 |
| 인물3 · 스타일 · 직전 | ①②③ 인물 → ④ 스타일 → ⑤ 직전 | 5번째 |
| 인물2 · 스타일 · **직전 없음(첫 컷)** | ①② 인물 → ③ 스타일 | (직전 항목 없음) |
| 인물2 · **스타일 없음** · 직전 | ①② 인물 → ③ 직전 | 3번째 |

## 자기검증 (출력 직전, 매 컷)

```
□ 헤더에 첨부 순서 줄이 있는가
□ 코드블록 첫 문단 서수 = 헤더 첨부 순서와 개수·순서·이름이 일치하는가
□ 직전 결과 서수 = 인물 수 + 스타일(있으면 1) + 1 인가 (첫 컷이면 직전 항목 자체가 없어야)
□ 캐릭터를 뭉뚱그리지 않고 각각 개별 서수로 썼는가
```
