## 독립 실행 워크플로우

전략 가이드 파일 없이도 다음 단계로 실행 가능합니다:

1. **요청 파악**: 문서 유형(SOP/회의록/조달/매뉴얼) 및 대상 업무 확인
2. **구조 설계**: 목적, 적용 범위, 담당자, 단계별 절차 초안 작성
3. **내용 작성**: 각 절차를 구체적인 액션 단위로 기술 (담당자·도구·산출물 명시)
4. **검토 포인트 제시**: 법적 준수 사항, 결재 라인, 비상 대응 절차 포함 여부 확인
5. **최종 확인**: 사용자 검토 후 최종본 제공

## 전자결재·결재선 설계

프로세스 문서(SOP·품의·구매 요청 등)에 결재 절차를 포함할 때는 아래 표준 흐름으로 결재선을 설계합니다.

```
기안(담당) → 검토(팀장/부서장) → 합의·협조(관련 부서) → 전결 또는 최종 결재(결재권자)
```

1. **기안**: 담당자가 문서를 작성하고 결재를 상신합니다. 문서번호·시행일자는 서식 규칙을 따릅니다.
2. **검토**: 직속 상급자(팀장·부서장)가 내용·형식을 확인합니다.
3. **합의(협조)**: 예산·법무·보안 등 이해관계 부서의 확인이 필요한 안건은 결재선 중간에 합의란을 둡니다. 합의 부서가 반려하면 기안 단계로 되돌립니다.
4. **전결**: 위임전결 규정에 따라 결재 권한이 위임된 사안은 전결권자가 "전결" 표기 후 결재하며, 그 상위자의 결재란은 만들지 않습니다. 결재권자 부재 시 직무대행자가 "대결" 표기 후 결재합니다. (근거: 「행정 효율과 협업 촉진에 관한 규정」 제10조 — 행정기관 기준이며, 기업은 각 사 위임전결 규정을 따릅니다)

**위임전결 규정 설계 가이드**: 사안 유형(지출·계약·인사·홍보)과 금액 구간별로 전결권자를 표로 정의합니다. 금액 구간과 전결권자 배치는 조직마다 다르므로 기본값을 임의로 확정하지 말고 사용자 조직의 규정을 확인합니다(확인 필요). 규정이 없는 조직에는 "소액 반복 지출은 부서장 전결, 신규 계약·예산 외 지출은 대표 결재"와 같은 관행적 출발점을 초안으로 제시하고 검토를 요청합니다.

공문·기안문·품의서의 결재란·전결 실서식은 `moai-officer:office-hwpx-writer`의 `references/kr-official-forms.md`(한국형 서식 SSOT)를 참조하세요.

## 문제 해결

| 상황 | 해결 방법 |
|------|-----------|
| 결재 라인 정보 부족 | 조직도 또는 결재 권한 범위를 알려주시면 맞춤 작성 가능합니다 |
| 조달 규정 확인 불가 | 일반적인 내부 구매 기준으로 작성 후 담당 부서 검토를 권장합니다 |
| 기존 프로세스와 충돌 | 현행 절차 문서를 공유해 주시면 개선안으로 작성해 드립니다 |
| 업종·도메인 특화 요구 | 해당 업종 규정이나 컴플라이언스 기준을 함께 제공해 주세요 |

## 관련 스킬 (후처리 체인)

SOP, 운영 매뉴얼, 회의록, 조달 문서 등 **서술형 산출물**은 다음 체인으로 마무리합니다.

```
business-process-manager → moai-coworker:general-ai-slop-reviewer → moai-writer:general-humanize-korean
```

- `moai-coworker:general-ai-slop-reviewer` — 문서 본문의 AI 티 패턴을 검수·수정합니다.
- `moai-writer:general-humanize-korean` — 검수된 본문을 자연스러운 한국어 실무 문체로 다듬습니다.

## 이 스킬을 사용하지 말아야 할 때

- **법적 구속력이 있는 계약서 작성** → 법률 전문가 검토가 필요합니다. moai-lawyer:legal-contract-review를 함께 사용하세요
- **실제 발주 처리 또는 시스템 입력** → ERP나 조달 시스템에 직접 접속해야 합니다
- **인사·노무 관련 절차** → moai-recruiter:business-people-operations 스킬이 더 적합합니다
- **재무·예산 승인 절차** → moai-coworker 스킬을 사용하세요
