# context-collector.md — 맥락 수집 프로토콜 (project 스킬)

## 개요

project 스킬의 스킬 체인 실행에 필요한 사용자 맥락을 체계적으로 수집하는 프로토콜이다. 맥락 충분성 등급, 2-Stage 일괄 설문(S1/S2), 모호성 감지, 반복 제한을 통해 효율적인 수집을 구현하고, 수집된 맥락의 **적용 규칙**(§5)까지 정의한다.

---

## 1. 맥락 충분성 등급

### A등급 — 필수 (프로젝트 CLAUDE.md에서 즉시 획득)

프로젝트 `CLAUDE.md`의 "프로젝트 개요" 섹션에서 읽어온다: 프로젝트 목적, 주요 산출물, 대상 독자·고객, 톤·형식 제약, 설치 플러그인. **글로벌 프로필은 사용하지 않는다** — 프로젝트 `CLAUDE.md`가 유일한 맥락 소스다.

### B등급 — 핵심 (80% 이상 충족 권장)

산출물별 도메인 맥락. **S1 일괄 진단 라운드에 함께 배치**해 수집하고, 남은 공백만 S2 보강 라운드로 넘긴다.

**질문 생성 규칙**: (1) 선택한 스킬 체인이 무엇을 만드는지 파악 → (2) 이미 아는 맥락은 재질문하지 않음 → (3) 부족한 도메인 맥락만 질문 슬롯에 올림 → (4) 라운드 수를 고정하지 않고 충분성으로 판정한다.

### C등급 — 보강 (S2 슬롯에 합류)

배경·동기, 판단 근거, 팀 규모, 업무 일정/마감, 주요 과제, 예산 제약. **별도의 텍스트 대화 심화 인터뷰를 열지 않는다** — 필요한 축을 S2 라운드의 선택지 질문으로 변환해 함께 낸다.

---

## 2. 맥락 수집 플로우 (2-Stage)

**[HARD] 한 라운드 = `AskUserQuestion` 1회 호출.** 질문을 1개씩 쪼개 연속 호출하지 않는다. 상한은 1회 최대 4질문 × 각 4옵션(`Other` 자동 포함).

### 2-1. 사전 차감 (질문 전에 먼저 뺀다)

```
known   = (./CLAUDE.md "프로젝트 개요") + (.moai/context.md) + (진입 발화)
needed  = A등급 + 필수 B등급
missing = needed - known

IF missing.empty(): 질문 없이 즉시 Phase 2로
ELSE: missing을 정보 이득 순으로 정렬 → 상위 4개를 S1 한 라운드에 배치
```

### 2-2. S1 — 일괄 진단 라운드

`missing` 상위 4개를 **한 번의 호출에 묶어** 낸다. 남은 축이 4개 미만이면 그만큼만, 4개를 넘으면 초과분은 S2로 이월한다. 질문 축 풀과 슬롯 채우기 규칙은 `init-protocol.md` §Phase 1 참조.

### 2-3. S2 — 보강 라운드 (조건부)

발동 조건 4가지(필수 축 공백 / `Other`·저신뢰 응답 / 답변 상충 / S1 슬롯 초과 이월) 중 하나라도 해당하면 실행하고, 없으면 건너뛴다. 실행할 때도 **부족분을 전부 묶어 한 라운드**로 낸다. 소크라테스 축(배경·근거·대안·영향·제약)이 필요하면 이 라운드의 선택지 질문으로 편입한다.

### 2-4. 충분성 재평가

충분성 80% 이상이면 실행 시작, 60-80%면 그 라운드에 「지금 아는 것으로 진행」 옵션을 포함해 사용자가 선택하게 하고, 60% 미만이면 S2를 한 번 더 돈다.

---

## 3. 반복 제한 및 방지

라운드 수를 미리 고정하지 않는다. 맥락이 충분(A+B등급)해지면 종료한다. **S2가 2회를 넘어가면** 그 라운드에 「지금 아는 것으로 진행」 선택지를 반드시 포함해 사용자가 끊을 수 있게 한다. `./CLAUDE.md` 프로젝트 개요 + 현재 세션 수집 맥락을 우선 재사용해 반복 수집을 방지한다.

---

## 4. 모호성 감지 규칙

신호(다중 선택 "기타", 저신뢰도 응답, 상충 답변, 도메인 키워드 부재, 목적-요청 불일치)를 **라운드 종료 시 한꺼번에 집계**하고, 감지된 신호 전부를 **하나의 S2 라운드**로 묶어 해소한다(신호 1개당 1회 호출 금지). 해소 시도가 2회를 넘으면 "일부 정보가 불명확하지만 진행 가능합니다"로 전환한다.

## 5. 저장 및 적용

### 5-1. 저장 위치

수집된 맥락은 **Phase 6에서 CLAUDE.md 본문**(프로젝트 개요·스킬 체인 섹션)에 기록되며, 별도 컨텍스트 파일이나 글로벌 프로필을 만들지 않는다. 인터뷰 답변 → `CLAUDE.md` "프로젝트 개요"에 반영. 산출물별 스킬 체인 → `CLAUDE.md` "프로젝트 워크플로우" 섹션에 기록. 세션 종료 후에는 `CLAUDE.md`가 단일 소스다.

### 5-2. 맥락 적용 규칙 (HARD — 수집된 맥락의 사용)

저장된 맥락(`CLAUDE.md` 프로젝트 개요 + `.moai/context.md` + 세션 임시 저장)을 **사용**할 때의 규칙:

1. **선택적 적용** — 맥락은 현재 요청과 관련 있을 때만 골라 쓴다.
2. **자연스러운 반영** — 적용할 때는 원래 알던 것처럼 녹인다. 어디서 가져왔는지 티 내지 않는다.
3. **메타 코멘터리 금지** — "저장된 맥락에 따르면", "제가 기억하기로는", "프로젝트 개요를 보니" 같은 회수 서술을 응답·산출물에 쓰지 않는다.
4. **재진입(resume) 세션 주의** — 설치 후 `/project resume`으로 돌아온 세션은 캐시된 맥락을 다시 읽는 순간이라 메타 코멘터리가 특히 빈발한다. 재개 인사에서도 맥락은 자연스럽게만 반영한다.

이 규칙은 생성 `CLAUDE.md`의 `맥락 적용 규칙` HARD 블록과 쌍이다(`claudemd-generator.md` 참조) — 셋업 시점(본 파일)과 런타임(생성 CLAUDE.md) 양쪽에서 같은 규칙이 적용된다.

---

## 6. 컨텍스트 갱신 트리거

사용자가 `CLAUDE.md`를 직접 수정한 경우, 새 산출물 유형(스킬 체인)을 추가하는 경우, 평가 점수가 낮은 경우, 사용자가 명시적으로 재수집을 요청한 경우.
