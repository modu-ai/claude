# story-conti

> 시나리오 씬을 프레임 단위 콘티로 변환하는 스킬. 8프레임 시퀀스와 9앵글 탐색으로 감독·촬영감독과 소통 가능한 스토리보드를 만든다.

## 1. 개요

이 스킬은 **무엇을** 만드는가 — 씬별 프레임 시퀀스 이미지. **언제** 쓰는가 — story-screenplay의 씬이 확정된 뒤.

## 크레딧 고지 (사전 고지 의무)

이 작업은 **Higgsfield 크레딧을 소모**합니다. 생성 전 예상 소모량을 사용자에게 고지하고 확인을 받는다.

| 항목 | 예상 크레딧 |
|------|-----------|
| 8프레임 시퀀스 1세트 (Popcorn) | 약 10~16 크레딧 |

진행하기 전 사용자에게 "위 크레딧이 소모됩니다. 계속 진행할까요?"로 확인한다. 사용자가 거부하면 생성을 중단하고 프롬프트 온리 모드로 전환한다.

## 모델 라우팅 표

| 용도 | 1순위 Higgsfield 모델 |
|------|---------------------|
| 콘티 프레임 시퀀스 | Popcorn (8프레임 연속) |
| 멀티 앵글 탐색 | Shots (9앵글 그리드) |
| 캐릭터 일관성 | Soul ID (story-character-sheet와 공유) |

## MCP 미연결 폴백 (프롬프트 온리 모드)

Higgsfield MCP에 연결할 수 없을 때는 생성을 생략하고 완성 프롬프트를 텍스트로 출력한다. 출력 끝에 "Higgsfield 웹(https://higgsfield.ai)에 위 프롬프트를 붙여넣으세요" 안내를 추가한다. 서버 과부하 시 잠시 대기 후 재시도 안내를 함께 표시한다.

## 3. 워크플로우

### Step 1: 씬 분할
씬을 프레임 단위로 나누고 각 프레임의 화각·동작을 정한다.

### Step 2: 크레딧 고지 + 확인
프레임 세트 수 × 약 10~16 크레딧을 합산 고지하고 승인받는다.

### Step 3: Popcorn 시퀀스 생성
프레임 설명을 Popcorn으로 생성해 8프레임 시퀀스를 만든다.

### Step 4: Shots 앵글 탐색(필요 시)
키 프레임을 Shots 9앵글로 탐색해 최적 구도를 선택.

### Step 5: 콘티 보드 조합
프레임·대사·연출 메모를 한 보드로 조합.

## 4. 주의사항

- 프레임 간 인물·배경 일관성을 Soul ID로 유지.
- 씬 톤(밝기·색온도)을 프레임 세트 전체에 걸쳐 통일.
