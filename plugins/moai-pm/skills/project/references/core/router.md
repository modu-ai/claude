# router.md — 자연어 → 4-plugin 라우팅 프로토콜

## 개요
사용자의 자연어 요청을 분석하여 **4개 AI 직원 플러그인**(코워커·디자이너·코더·PM) 중 적합한 곳으로 라우팅하는 프로토콜. 이전 27-플러그인 분산 토폴로지는 폐기되었고, 단일 `modu-ai/claude` 마켓플레이스의 4-plugin으로 수렴했다. 라우팅된 플러그인의 `skills/` 안 SKILL.md가 곧 실행 단위다.

---

## 1. 라우팅 흐름

```
사용자 자연어 요청 (/project 또는 bare 진입)
    ↓
[1단계] 플래그 확인
    ├── --cowork / --designer / --code 명시 → 해당 분기 즉시 진입
    └── 플래그 없음 → [2단계]
    ↓
[2단계] 프로젝트 컨텍스트 확인
    ├── CLAUDE.md의 "워크플로우 체인" / .moai/config 정의 산출물 체인 있음
    │       → 해당 분기로 직접 실행
    └── 없음 → [3단계]
    ↓
[3단계] 키워드 매핑
    ├── 매칭 1개 플러그인 → 해당 분기 setup 프로토콜 진입
    ├── 매칭 2개+ → [4단계] 모호성 해소
    └── 매칭 0개 → [5단계] 폴백
```

## 3. 모호성 해소

키워드 매칭 결과 후보가 2개 이상의 플러그인에 걸칠 때:

### 3-1 자동 해소
- 산출물 유형이 명시되면 해당 산출물을 만드는 플러그인 우선
- 예: "브랜드 카드뉴스 만들어줘" → 디자인 자산보다 콘텐츠 산출물이 명시 → `--cowork`(content-card-news)
- 예: "디자인 시스템 기반 랜딩 페이지" → `--designer` 우선(시스템 셋업) 후 `--cowork`(랜딩 체인)

### 3-2 사용자 확인
자동 해소가 어려우면 AskUserQuestion(1질문, 후보 플러그인 최대 4개):

```
"어떤 작업으로 시작할까요?"
○ 코워커 — 실무·콘텐츠·작가 체인 셋업
○ 디자이너 — 브랜드·디자인 시스템 셋업
○ 코더 — 개발 환경 셋업
+ Other
```

## 4. 복합 요청 처리 (크로스 플러그인)

사용자 요청이 2개+ 플러그인에 걸칠 때:

### 4-1 순차 처리 (분기 체이닝)
```
"브랜드 디자인 잡고 카드뉴스 만들어줘"
→ /project --designer (DESIGN.md 합성) 완료 후
→ /project --cowork (content-card-news → general-ai-slop-reviewer, 디자인 시스템 적용)
```

### 4-2 동시 셋업
```
"스타트업이라 사업계획서랑 웹사이트 디자인 둘 다 필요해"
→ /project --cowork (사업계획서 체인) + /project --designer (브랜드 셋업)
   (각각 독립 CLAUDE.md 섹션 또는 순차 진행)
```

### 4-3 심층 분석 판단
복합 요청이거나 2개+ 플러그인 체인이 걸치면 `ultrathink`(심층 분석)로 최적 분기 경로를 결정한다.

## 6. 검증 깊이 연동

라우팅과 함께 **검증 깊이 등급**도 정한다 (상세: `execution-protocol.md`):
- 법률·계약·NDA·개인정보·세무·재무·정부지원·의료·규제 키워드, 또는 2개+ 플러그인 체인 → **DEEP** (근거 게이트 + 면책 + 전달 전 확인)
- 일반 산출물(보고서·카피·문서) → **NORMAL** (근거 게이트 활성)
- 단순 조회·요약 → **QUICK** (근거 게이트 스킵)

키워드는 seed일 뿐이며, 트리거 단어 없이 표현된 고위험 요청도 맥락으로 DEEP 판단한다.

## 7. 버전 정보

**4-plugin / 단일 modu-ai/claude 마켓플레이스 라우팅.** 플러그인 → 스킬(또는 스킬 체인) 모델. 키워드 매칭으로 4-plugin 중 하나를 고르고, 해당 플러그인의 셋업 프로토콜(`*-setup.md`)로 진입한다. 별도의 실행 모듈 계층은 없다.
