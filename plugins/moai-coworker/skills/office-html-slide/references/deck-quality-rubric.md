# 슬라이드 정량 QA 루브릭 — 6카테고리 가중합 + hard-fail

발표급 슬라이드가 아마추어 슬라이드와 갈리는 지점을 **사람이 보지 않고도 1차로 자동 채점**하기 위한 규칙 세트입니다. office-html-slide는 단일 파일 HTML(DOM)을 렌더하므로, 문서형 슬라이드(pptx) 채점에서 흔히 쓰는 python-pptx 파싱 관행을 **HTML/DOM 파싱·렌더 검사**로 옮겨 적용합니다.

> **핵심 원칙 — 임계값은 상수가 아니다.** 아래 모든 수치(폰트 하한, 6x6, 색상 수, 액션타이틀 길이, 명도대비)는 업계 컨센서스로 수렴한 **기본값**일 뿐 공식 규정값이 아닙니다. 브랜드·청중·발표 환경(온라인/오프라인)에 따라 조정되어야 하며, 채점기는 이 값을 설정 파일(예: `qa-config.json`)로 오버라이드할 수 있게 구현합니다. **고정 상수로 하드코딩하지 마십시오.**

---

## 채점 구조 — 3단계

1. **DOM 구조 파서** — 슬라이드마다 (a) 제목 텍스트, (b) 본문 텍스트 전체, (c) 모든 텍스트 노드의 computed `font-size`/`font-family`/`color`, (d) 모든 요소의 `getBoundingClientRect()`, (e) 인라인 SVG 차트의 시리즈 수·범례 존재 여부를 추출합니다.
2. **규칙 채점** — 아래 31개 기준을 각각 독립 함수로 구현해 슬라이드별 pass/fail과 근거 수치를 반환합니다.
3. **종합 스코어** — 규칙을 6카테고리로 그룹핑해 **가중합 점수**를 산출하고, hard-fail 규칙은 감점이 아니라 **즉시 반려**로 처리합니다.

## 6 카테고리 가중합

| 카테고리 | 포함 기준(번호) | 채점 관점 |
|---|---|---|
| **구조(structure)** | 1·2·3·4·5·6·7·8 | 액션타이틀·원 메시지·불릿 상한 |
| **타이포(typography)** | 9·10·11·12·13·28 | 폰트 하한·종류 상한·오버플로 |
| **컬러(color)** | 14·15·16·30 | 팔레트 수·강조 비율·명도대비·의미 일관성 |
| **정렬(alignment)** | 20·21·22·23·27 | 그리드 스냅·여백·화이트스페이스·표 밀도 |
| **데이터시각화(data-viz)** | 17·18·19·29 | 데이터-잉크·직접 라벨링·차트 정크 |
| **신뢰성(trust)** | 24·25·26·31 | 출처 라인·페이지 번호·발화 시간·대체텍스트 |

**합격선**은 6카테고리 가중합 기준을 두되(예: 90/100), 이 합격선 자체가 **캘리브레이션으로 조정되는 값**이며 고정 상수가 아닙니다. 카테고리 가중치·합격선은 프로젝트 설정에서 재정의합니다.

## Hard-fail 티어 (즉시 반려)

아래 6개는 가중합 감점이 아니라 **한 건이라도 걸리면 즉시 반려**입니다(발표급 아님 → 고쳐서 다시).

1. **본문 폰트 하한 미달** — 본문 텍스트가 설정된 하한(기본 24pt 상당, 온라인 전용 덱은 18pt로 하향 설정 가능) 미만.
2. **수치 슬라이드 출처라인 부재** — 숫자·차트·표가 있는데 `출처:`/`자료:`/`Source:` 텍스트 노드 없음.
3. **명도대비 WCAG 4.5:1 미달** — 본문 텍스트-배경 대비가 4.5:1 미만(큰 텍스트 3:1).
4. **텍스트 오버플로** — 텍스트 요소의 `scrollHeight > clientHeight`(박스 밖으로 넘침) 또는 자동 축소로 폰트가 하한 미만이 됨.
5. **3D 차트 효과** — SVG/CSS에 원근·`rotateX/Y`·입체 그림자 등 3D 차트 장식.
6. **그리드 정렬 편차 초과** — 동일 역할 요소(제목/푸터)의 좌표 편차가 슬라이드 너비의 허용 오차(기본 1%)를 초과.

> hard-fail 임계값도 모두 설정 오버라이드 대상입니다 — 예컨대 온라인 아카이브 덱은 폰트 하한을 18pt로 낮춰 재정의할 수 있습니다.

## 31개 정량 기준 (HTML/DOM 재해석)

측정법의 python-pptx 표현은 모두 DOM 등가물로 치환했습니다. 임계값 열의 값은 **기본값**이며 설정으로 오버라이드됩니다.

| # | 기준 | 측정법 (DOM) | 임계값(기본, 오버라이드 가능) | H/S |
|---|---|---|---|---|
| 1 | 제목 길이 상한 | 제목 요소 텍스트 word count | ≤15단어 | soft |
| 2 | 제목 줄바꿈 수 | 제목 요소 렌더 줄 수(`getClientRects().length`) | ≤2줄 | soft |
| 3 | 액션타이틀 완결성 | 제목에 서술어 존재/명사구 종결 여부 정규식·품사 | 명사 나열 제목이면 fail | soft |
| 4 | 제목 등위접속사 | 제목에 "및/와/과/그리고/and" + 두 서술어 | 검출 시 경고 | soft |
| 5 | 본문 총 단어 수 | 슬라이드 내 제목 제외 텍스트 노드 단어 합 | ≤40 권장·60 초과 fail | soft |
| 6 | 불릿 줄 수 | 본문 `<li>`/단락 개수 | ≤6줄 (6x6) | soft |
| 7 | 불릿당 단어 수 | 각 단락 단어 수 | 평균 ≤6~8·개별 12 초과 fail | soft |
| 8 | 불릿 중첩 깊이 | 중첩 `<ul>` 깊이 | ≤3단계 | soft |
| 9 | 본문 폰트 하한 | 본문 노드 computed `font-size` | ≥24pt 상당(온라인 18pt 설정) | **hard** |
| 10 | 캡션/출처 폰트 하한 | footer/출처 노드 `font-size` | ≥12pt·18pt 미만 경고 | soft |
| 11 | 폰트 크기 종류 수 | 슬라이드 내 고유 `font-size` 개수 | ≤3종 | soft |
| 12 | 폰트 패밀리 수 | 슬라이드 내 고유 `font-family` 개수 | ≤2종 | soft |
| 13 | 제목 크기 일관성 | 전체 덱 제목 `font-size` 표준편차 | 편차 ≤2pt | soft |
| 14 | 팔레트 색상 수 | 텍스트/도형/차트 고유 색(ΔE<10 병합, 그레이 제외) | 슬라이드 ≤4색·덱 ≤6색 | soft |
| 15 | 강조색 사용 비율 | 강조색 요소 수 / 전체 색상 요소 수 | 10~30% | soft |
| 16 | 텍스트-배경 명도대비 | 상대 휘도 기반 WCAG 대비비 | 본문 ≥4.5:1·큰 텍스트 ≥3:1 | **hard** |
| 17 | 데이터-잉크 비율 | SVG 차트 내 장식 요소(그림자/그라디언트/중복 범례) 개수 | 0 권장·3D면 hard-fail | **hard**(3D) |
| 18 | 범례 vs 직접 라벨 | 카테고리 ≤3개인데 별도 `<legend>` 존재 | 존재 시 fail | soft |
| 19 | 파이 카테고리 수 | 파이/도넛 데이터 포인트 개수 | ≤5 권장·7 초과 fail | soft |
| 20 | 그리드 정렬 편차 | 동일 역할 요소 x/y 좌표 표준편차(px) | 슬라이드 너비 1% 이내 | **hard** |
| 21 | 그리드 스냅 정합 | 요소 경계 좌표 클러스터링(정렬선) | 미정렬 경계 <10% | soft |
| 22 | 여백 확보 | 요소 bbox의 슬라이드 가장자리 최소 거리 | 짧은 변 ≥3% | soft |
| 23 | 화이트스페이스 비율 | 콘텐츠 bbox 점유 면적 / 전체 | 점유 ≤70%(여백 ≥30%) | soft |
| 24 | 출처 표기 존재 | 수치 슬라이드에 `출처:`/`자료:`/`Source:` 노드 | 부재 시 hard-fail | **hard** |
| 25 | 페이지 번호 존재 | `.deck-page` 등 페이지 표기 노드 렌더 여부 | 100%(표지·섹션 화이트리스트) | soft |
| 26 | 발화 시간 근사 | 총 단어 수 / 130~150 wpm | ≤60초 | soft |
| 27 | 표 밀도 상한 | `<table>` 행×열 셀 개수 | ≤36셀(6x6) | soft |
| 28 | 텍스트 오버플로 | `scrollHeight > clientHeight` 또는 축소 후 하한 미만 | 발생 시 hard-fail | **hard** |
| 29 | 장식 대 데이터 비율 | 데이터 무관 장식 아이콘/스톡 이미지 개수 | 데이터 슬라이드 0 | soft |
| 30 | 색상 의미 일관성 | 양수/음수 등 의미별 색 매핑 덱 전체 비교 | 동일 의미 다른 색이면 fail | soft |
| 31 | 대체텍스트 존재 | 정보성 SVG/이미지의 `aria-label`/`<title>`/`alt` | 100% 필요·부재 경고 | soft |

> H/S = hard(즉시 반려) / soft(가중합 감점). hard 표기는 기본 설정 기준이며, 온라인·아카이브 프로파일에서는 일부를 soft로 재분류할 수 있습니다.

## office-pptx-designer와의 공유

이 루브릭은 **office-pptx-designer와 공유**됩니다. office-pptx-designer는 pptx 네이티브 채점기([`office-pptx-designer/references/qa-checklist.md`](../../office-pptx-designer/references/qa-checklist.md), python-pptx/pptxgenjs 좌표 기준·EMU 단위)를 별도로 가지며, 본 파일은 그 6카테고리·hard-fail 개념을 **HTML/DOM 렌더 대상으로 재해석한 대응본**입니다. 두 스킬은 같은 원칙(폰트 하한·출처 라인·명도대비·오버플로)을 각자의 렌더 컨텍스트에서 검사합니다.

## 근거 (공개 1차 출처)

여기 정리한 기준은 특정 제품이 아니라 **공개된 프레젠테이션 품질 관행**에서 수렴한 값입니다: WCAG 2.2(명도대비 4.5:1 / 큰 텍스트 3:1), Barbara Minto 피라미드 원칙·액션타이틀, Edward Tufte 데이터-잉크 비율, Guy Kawasaki 10/20/30 규칙, 6×6 규칙, ARL 접근성 가이드(폰트 24pt 하한). 이들은 McKinsey/BCG/Bain 내부 문서가 아니라 공개 교육 자료·표준에서 수렴한 **컨센서스 값**입니다. 따라서 모든 임계값은 조정 가능한 설정으로 두어야 하고 고정 상수로 하드코딩하면 안 됩니다.
