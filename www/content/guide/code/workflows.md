# 일반 워크플로우

## 공식 4단계 워크플로우

**Explore → Plan → Implement → Commit**

1. **Explore** — 코드베이스 탐색. 예: "이 폴더 구조 설명해줘"
2. **Plan** — 계획 수립. 예: "이 기능 추가하려면 뭘 바꿔야 해?"
3. **Implement** — 구현. 예: "그 계획대로 코드 작성해줘"
4. **Commit** — 커밋. 예: "커밋 메시지 써줘"

## 일상 작업 패턴 (이렇게 물어보세요)

- **코드베이스 탐색** — "이 프로젝트에서 인증은 어디서 담당해?"
- **버그 수정** — 재현 경로 설명 → "원인 찾아서 고쳐줘" (재현 → 진단 → 수정 3단계)
- **리팩토링** — "이 함수 중복 제거해줘"
- **테스트 작성** — "이 함수 테스트 써줘"
- **문서화** — "이 모듈 README 써줘"

## Git 커밋·PR

Claude가 변경사항을 읽고 커밋 메시지를 자동 작성합니다. PR 설명도 생성 가능합니다.
