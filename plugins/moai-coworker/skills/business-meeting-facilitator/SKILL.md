# 회의 진행·퍼실리테이션 (Meeting Facilitator)

## 개요

회의가 길어지는 이유는 보통 사람이 많아서가 아니라 **목적이 흐릿하고 진행 규칙이 없어서**다. 이 스킬은 회의 전 목적·아젠다·산출물을 못 박고, 회의 중에는 시간과 발언을 관리하며, 회의 후에는 결정과 담당자가 남게 하는 진행 설계를 돕는다. 모이는 시간을 줄이고 남는 결론을 늘린다.

## 트리거 키워드

회의, 회의 진행, 아젠다, 퍼실리테이션, 산으로 가는 회의, 의사결정 회의, 회의록, 회의 준비, 미팅 진행, 회의 규칙, 후속 액션

## 워크플로우

### 1단계: 회의의 목적과 산출물 정의

회의를 잡기 전에 한 줄로 답한다. 답이 안 나오면 회의 대신 메시지로 끝낼 일이다.

- **이 회의가 끝나면 무엇이 남아야 하는가?** (결정 / 정렬 / 아이디어 / 공유)
- 회의 유형 구분:

| 유형 | 목적 | 끝났을 때 남는 것 |
|------|------|------------------|
| 의사결정 | 선택지 중 결정 | 결정 + 담당자 + 기한 |
| 정렬·공유 | 같은 그림 맞추기 | 합의된 이해·다음 단계 |
| 발산·아이디어 | 가능성 펼치기 | 후보 목록 + 다음 판단 기준 |
| 문제 해결 | 막힌 곳 풀기 | 원인 + 해결안 + 책임자 |

### 2단계: 아젠다 설계 (시간 박스)

안건마다 **목적·시간·산출물**을 붙인다. 막연한 "○○ 논의"는 금지.

- 각 안건: 제목 / 목적(결정·공유·발산) / 배정 시간 / 누가 발제
- 무거운 결정 안건을 앞쪽에 배치(에너지 높을 때)
- 사전 공유 자료를 미리 돌려 회의에서 "읽는 시간"을 없앤다
- 안건이 5개를 넘으면 회의를 쪼개거나 일부를 비동기로 돌린다

### 3단계: 진행 규칙 (산으로 가지 않게)

회의 시작 30초에 규칙을 합의하면 끌고 가기 쉬워진다.

- **타임키퍼·기록자 지정**: 진행자가 둘 다 하면 둘 다 놓친다
- **주차장(Parking Lot)**: 주제에서 벗어난 논의는 보드에 적어두고 나중에. "그건 주차장에 적어둘게요"
- **발언 균형**: 말 많은 사람은 요약 유도, 조용한 사람은 직접 호명해 의견 청취
- **결론 신호**: 안건마다 "그래서 우리가 정한 건 ○○ 맞죠?"로 못 박고 다음으로

### 4단계: 의사결정 끌어내기

결정이 안 나는 회의를 끝내려면 결정 방식을 미리 정한다.

- **결정 방식 합의**: 합의 / 다수결 / 책임자 단독결정(RAPID·DACI 등 단순 버전)
- **선택지를 좁히기**: 무한 토론 대신 2~3개 안으로 압축 후 비교
- **결정 못 할 땐**: 무엇이 더 있어야 결정 가능한지(데이터·승인)와 누가 언제까지를 정하고 종료
- 결정은 그 자리에서 **문장으로** 적어 모두가 같은 걸 봤는지 확인

### 5단계: 회의록 + 후속 액션

기록의 핵심은 회의 내용 받아쓰기가 아니라 **결정과 할 일**이다.

- **3줄 회의록**: 결정 사항 / 액션 아이템(담당·기한) / 보류·다음 안건
- 회의 직후(당일) 공유 — 시간이 지나면 합의가 흐려진다
- 액션 아이템은 담당자·기한·완료 기준까지. "○○ 검토"가 아니라 "○○ 금요일까지 A/B 비교안"

## 사용 예시

**예시 1**: "내일 주간 회의가 매번 1시간 넘게 늘어지는데 어떻게 줄이지?"
→ 목적 재정의 → 안건별 시간 박스 아젠다 → 타임키퍼·주차장 규칙 → 3줄 회의록 템플릿

**예시 2**: "신규 기능 우선순위를 정하는 의사결정 회의를 진행해야 해"
→ 결정 방식 사전 합의 → 후보 2~3안 압축 → 비교 기준표 → 결정 문장화 + 담당·기한

**예시 3**: "회의가 자꾸 딴 얘기로 새서 결론이 안 나"
→ 주차장 규칙 + 결론 신호 멘트 → 안건 5개 초과 시 분할·비동기화 → 진행 멘트 스크립트

## 산출물

- 회의 목적·유형·산출물 한 줄 정의
- 시간 박스 아젠다(안건·목적·시간·발제자)
- 진행 규칙 세트(타임키퍼·주차장·결론 신호)
- 의사결정 방식·비교안
- 3줄 회의록 + 액션 아이템(담당·기한) 템플릿

## 주의사항

- 모든 모임이 회의일 필요는 없다. 공유·보고는 비동기 메시지가 더 빠를 때가 많다.
- 진행자는 의견 주인이 아니라 흐름 관리자다. 본인 주장을 끌고 가면 균형이 깨진다.
- 결정을 문장화하지 않으면 회의가 끝나도 각자 다르게 기억한다.

## 관련 스킬

- **moai-coworker:business-report-speak**: 회의에서 결론 먼저 발제·보고
- **moai-coworker:business-negotiation-1on1**: 회의 전 핵심 인물과 사전 정렬·설득
- **moai-coworker:business-conflict-handler**: 회의 중 충돌·발언 독점 대응
- **moai-officer:office-docx-generator**: 회의록을 정식 문서로 정리

## 이 스킬을 사용하지 말아야 할 때

- **회의록·기획서 문서 작성**: 글 산출물은 `moai-coworker`를 쓴다. 이 스킬은 진행 설계에 집중한다.
- **개인 일정·할 일 관리**: 내 작업 계획은 `moai-productivity`를 사용한다.
- **워크숍·교육 프로그램 설계**: 장시간 퍼실리테이션 커리큘럼은 별도 설계가 필요하다.
