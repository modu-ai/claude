# story-webtoon-spec: 한국 웹툰 규격 SSOT 허브

한국 웹툰 플랫폼의 **규격·경로·용어**를 한곳에 모은 참조 허브다. 원고를 그리기 전에 "얼마나 크게, 몇 컷으로, 어디에 올리나"를 먼저 고정하는 자리다. 다른 story-* 스킬(planner·episode·art·qc·cover-art)이 규격을 각자 복제하지 않고 이 허브 하나를 바라본다.

## 1. 이 스킬이 하는 일

- **규격 고정** — 세로 스크롤 원고의 가로폭·세로 길이·분할 단위·색공간을 규격 표로 제공한다.
- **경로 안내** — 네이버웹툰·카카오 계열의 투고→승격→연재 경로를 단계로 정리한다.
- **수익화 설계 연동** — 기다무·미리보기 같은 유료 전환 지점을 이야기 훅과 맞추도록 돕는다.
- **용어 정리** — '말칸' 같은 비표준어 대신 말풍선·식자·콘티 같은 표준 실무어로 교정한다.

컷 대본·작화 프롬프트·기획은 각각 `story-webtoon-episode`·`story-webtoon-art`·`story-webtoon-planner`가 맡는다. 이 스킬은 **규격의 진실**만 책임진다.

## 2. 수치는 반드시 출처와 함께 (환각 금지 원칙)

플랫폼 수치(캔버스 가로폭, 회차 길이, 파일 용량 상한, 컷 수)는 **시기·정책에 따라 바뀐다.** 이 허브는 확인되지 않은 숫자를 사실처럼 적지 않는다.

- 구조·과정 지식(3단계 데뷔 경로, 세로 스크롤 형식, 기다무 구조)은 사실로 서술한다.
- **개별 수치는 전부 `[미확인 — 공식 안내 확인 필요]`로 표기**하고, references 표의 "확인 방법" 열에 어느 공식 안내에서 확인하는지 적는다.
- 사용자에게 답할 때도 "정확한 최신 수치는 해당 플랫폼 공식 창작 안내에서 확인하세요"를 함께 안내한다.

> 확인된 규격 표 하나보다, **[미확인] 자리표시 + 확인 경로**를 갖춘 표가 낫다. 지어낸 픽셀값·용량 상한은 창작자를 잘못된 원고로 이끈다.

### 답변 전 자가 점검 (수치가 나오는 질문마다)

- [ ] 지금 답에 픽셀·길이·용량·컷 수 **숫자**가 들어가는가? → 확인된 출처가 없으면 `[미확인]`으로 바꾼다.
- [ ] 구조·과정(3단계 경로·세로 형식·기다무 구조)만 사실로 서술했는가?
- [ ] `[미확인]` 항목에 **"어느 공식 안내에서 확인"**을 함께 안내했는가?
- [ ] 인쇄 만화 기준(CMYK·300DPI·판형 mm)을 웹툰에 잘못 적용하지 않았는가?

## 3. 데뷔·투고 경로 (구조는 사실)

한국 주요 플랫폼의 등단 경로는 **아마추어 공개 → 인기 검증 → 정식 계약** 흐름을 공유한다. 네이버웹툰은 흔히 잘못 알려진 "2단계"가 아니라 **3단계**다.

| 단계 | 네이버웹툰 | 성격 |
|------|-----------|------|
| 1 | 도전만화 | 누구나 자유 업로드하는 아마추어 게시판 |
| 2 | 베스트도전 | 반응 좋은 작품이 승격되는 상위 노출 |
| 3 | 정식연재 | 편집부 계약 프로 연재 |

- 승격 기준의 정량 수치·계약 조건은 `[미확인 — 공식 안내 확인 필요]`.
- 카카오웹툰·카카오페이지·네이버시리즈의 투고 경로와 승격 기준은 references에 정리한다.

→ 상세: `references/platform-debut-paths.md`

## 4. 원고 규격 (수치는 [미확인] 표기)

세로 스크롤 웹툰은 **가로 고정·세로로 긴 이미지**를 회차 단위로 올리고, 긴 회차는 여러 장으로 분할 업로드한다.

- **형식(사실):** 위→아래 연속 스크롤. 페이지형 컷 그리드가 아니다.
- **여백 띠(사실):** 컷과 컷 사이 세로 빈 띠가 스크롤 리듬을 만든다. 식자 배치는 `story-webtoon-lettering` 참조.
- **색공간(사실):** 화면 표시 기준 sRGB. 인쇄용 CMYK·300DPI를 적용하지 않는다.
- **수치(전부 [미확인]):** 가로폭·세로 총길이·분할 단위·용량 상한 → 공식 원고 제작 가이드 확인.

→ 상세: `references/manuscript-specs.md`

## 5. 회차 분량 관행

- **컷 수:** 세로형은 페이지 만화보다 컷이 많은 편(구조는 사실). 정확 범위 `[미확인]`.
- **화수 규모:** 시즌1 화수 `[미확인]` — 연재작 사례/편집부 안내.
- **연재 주기:** 주 1회가 일반적(큰 관행은 사실). 일일 연재(웹소설)도 존재.

→ 상세: `references/episode-volume.md`

## 6. 썸네일·표지 규격

- **노출 지점(사실):** 썸네일/타이틀 카드, 카카오페이지·네이버시리즈 표지, 웹소설 표지.
- **가독 하한(사실):** 모바일에서 크게 축소되므로 제목 글자·핵심 인물이 축소 상태에서 식별돼야 한다. 얇은 획·과밀 요소는 뭉개진다.
- **수치([미확인]):** 비율·최소 크기·포맷 → 각 플랫폼 공식 이미지 규격 안내.

→ 상세: `references/thumbnail-cover-specs.md`

## 7. 수익화 훅 정렬

- **기다무(구조는 사실):** 기다리면 한 회 무료 — 재방문 유도 + 성급한 독자의 결제 전환. 대기 시간 `[미확인]`.
- **미리보기(구조는 사실):** 유료로 다음 회 앞당겨 보기.
- **설계 관점(핵심):** 유료 경계 직전을 고구마 정점으로, 유료 회차를 사이다 해소로. 회차 끝 클리프행어를 유료 지점과 정렬한다.

→ 상세: `references/monetization-hooks.md`

## 8. 용어 사전 + 금칙어

실무 표준어를 쓰면 협업·검색·플랫폼 심사에서 오해가 줄어든다. 비표준어를 표준어로 교정하는 금칙어 표를 함께 둔다.

| 쓰지 말 것 | 대신 |
|-----------|------|
| 말칸 | 말풍선 |
| 웹썰 | 웹툰 / 웹소설 |
| 컷지 | 컷 |

사용자가 비표준어로 물어도 뜻을 알아듣고 **표준어로 교정해** 답한다.

→ 상세: `references/webtoon-glossary.md`

## 9. 사용 패턴

1. 규격이 필요한 질문을 받으면 해당 references 파일을 열어 표를 확인한다.
2. 표의 수치가 `[미확인]`이면 그대로 전하고 "확인 방법" 열의 공식 안내를 함께 안내한다.
3. 구조·과정 지식은 사실로 답한다.
4. 다른 story-* 스킬이 규격을 물으면 이 허브의 references 경로를 알려 준다(복제하지 않는다).

### 워크드 예시 — "네이버웹툰 원고 가로폭 알려줘"

> 세로 스크롤 웹툰은 가로 고정·세로로 긴 형식이라 가로폭을 먼저 맞춰야 합니다. 다만 정확한 픽셀값은 시기·정책에 따라 바뀌므로 여기서 단정하지 않습니다 — **`[미확인]`**. 네이버웹툰 공식 원고 제작 가이드에서 현재 권장 가로폭을 확인하세요. (형식·색공간 sRGB·분할 업로드 개념은 `references/manuscript-specs.md`에 정리돼 있습니다.)

숫자를 지어내 답하지 않고, **형식은 설명 + 수치는 확인 경로**로 답하는 것이 이 허브의 기본 태도다.

## 10. 자매 허브와의 경계

| 물음 | 담당 |
|------|------|
| 캔버스·데뷔·회차·수익화·용어 | 이 스킬(story-webtoon-spec) |
| 말풍선·서체·SFX·식자 | `story-webtoon-lettering` |
| 연재 상태·기획서·파일명 | `story-series-bible` |
| 컷 대본·작화 프롬프트 | `story-webtoon-episode` / `story-webtoon-art` |

경계가 겹치는 질문(예: 표지 서체)은 규격은 이 스킬, 서체 지정은 lettering으로 나눠 안내한다.
