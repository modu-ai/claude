# AI 틱 윤문 카피 스킬 — 생성 지시서 (설계 노트)

> **Status**: 설계 노트(v0.1, 2026-07-08) → **정식 구현 완료**(2026-07-08, v2.5 콘텐츠 6항 + v2.6 재설계: 다업종 예시 확장·L 스토리텔링/M 슬라이드 카테고리 신설·카피 모드 가드(사실 앵커 보존)·metrics_v2 v2.1 카피 탐지·humanize_html.py). 본 문서는 사용자 제공 원 설계 노트의 역사 기록으로 보존한다. 운영 SSOT는 `general-humanize-korean` references/ai-tell-taxonomy.md + SKILL.md. 박스 그림은 마크다운 표로 변환; 내용은 원문 축실 보존.

## 스킬 개요

- **이름 제안**: `moai-domain-ai-tell-ko-copy` (또는 기존 `general-humanize-korean`의 한국 카피 확장)
- **목적**: AI가 생성한 한국어 마케팅 카피에서 번역투·직역 비유·의인화·AI 텔을 식별해, 브랜드 톤을 유지하면서 자연스럽고 펀치 있는 한국 카피로 윤문
- **입력**: HTML/마크다운 카피 텍스트 (헤드라인·본문·CTA·메타 description)
- **출력**: 동일 구조/정보 유지 + 카피만 자연화 (before/after 표 + 일괄 치환 가능 형태)

## 1. AI 틱 패턴 카탈로그 (식별 규칙)

| 패턴 | 전형적 표현 | 문제 | 원인 |
|------|-----------|------|------|
| **직역 비유** | "굴러가는 자동화", "결과물의 얼굴", "한 흐름으로" | 한국어에 안 쓰는 비유 | "running automation", "face of the result", "one flow" 직역 |
| **의인화 직역** | "자동화가 나 대신 일합니다", "자동화가 나와 함께" | 과도한 의인화·직역투 | "automation works for me" 직역 |
| **직역 부사** | "더는 혼자가 아닙니다" | 부자연 | "no longer alone" 직역 ("이제는"이 자연) |
| **직역 숙어** | "손발을 맞추다", "한 팀으로 손발" | 어색 | "work in tandem/hand in hand" 직역 |
| **약한 끝맺음** | "네 가지", "~완성합니다" (헤드라인) | 펀치 부족 | 추상 명사/평서로 끝 |
| **감정 과장** | "더는 혼자가 아닙니다", "같은 노트북이지만" | 광고 카피처럼 안 읽힘 | 감정적 영어 카피 직역 |
| **정제 추상화** | "자동화가 대신 일해 줍니다" | 맞는 말이지만 밋밋 | AI가 안전하게 다듬은 결과 |
| **3인칭 무주어** | "자동화가 ~합니다" (2인칭 호회 부재) | 독자와 거리 | 영어 3인칭 설명체 |

> **기존 스킬 매핑**: 직역 비유→A-23, 의인화 직역→A-22/D-5, 직역 숙어→A-23, 정제 추상화→K-1(역), 감정 과장→D-4/K-2. **신규(기존 없음)**: 직역 부사 "더는/no longer" → `A-24` 후보. 3인칭 무주어/2인칭 호회 부재 → 화계·인칭 축, `I-7` 보강 또는 신규.

## 2. 교정 원칙 (톤 가이드)

1. **2인칭 직접 호회** — "여러분", "여러분의 컴퓨터" (3인칭 "자동화가" → "여러분이 AI 직원을")
2. **강렬·구체 비유로 교체** — "AI 직원", "AI 자동화 머신" (추상 "자동화"보다 구체적 이미지)
3. **구체 혜택 명시** — "프롬프트/컨텍스트 엔지니어링 1도 몰라도", "~업무를 위임" (추상 혜택 → 측정 가능/체감 가능)
4. **구어 허용** — "~수 있습니다", "~됩니다", 강조 허용 (지나친 정제 오히려 AI 틱)
5. **행동·결과 중심 끝맺음** — "위임하세요", "만듭니다", "24/7 쉬지 않고 일을 처리합니다" (평서 "합니다" → 행동 유도)
6. **정보성 카피는 건드리지 않음** — 날짜·가격·세션명·FAQ 사실·사업자 정보는 톤 변경 금지 (SEO/신뢰성 보존)
7. **헤드라인은 강한 끝맺음** — "네 가지" 같은 추상 명사로 끝내지 않음 ("~위임합니다", "~만듭니다")
8. **비유 일관성** — 한 번 정한 비유("AI 직원")를 전 페이지에 일관 적용, "자동화/AI 팀/AI 직원" 혼용 금지

## 3. before/after 예시 (실제 사례)

| before (AI 틱) | after (자연) | 적용 원칙 |
|----------------|-------------|-----------|
| 말 한마디에 굴러가는 자동화 | 말 한마디로 작동하는 AI 직원 | 직역 비유 제거 + 구체 비유 |
| 자동화가 나 대신 일합니다 | 여러분의 컴퓨터가 AI 자동화 머신이 됩니다 | 의인화 직역 → 2인칭 + 강렬 비유 |
| 더는 혼자가 아닙니다 | 이제는 혼자가 아닙니다 | 직역 부사 교정 |
| 네 가지가 한 흐름으로 손발을 맞춥니다 | 네 가지가 한 번에 맞물려 움직입니다 | 직역 숙어·비유 제거 |
| 결과물의 얼굴을 만들고 | 결과물의 톤과 스타일을 디자인하고 | 직역 비유("face") → 자연 |
| 2일 만에 완성합니다 | 2일 만에 위임하세요 | 평서 → 행동 유도 |
| 워크숍이 끝나도 자동화가 내 컴퓨터에 그대로 남습니다 | 강의가 끝나면 여러분만의 AI 직원이 생깁니다 | 직역투 → 2인칭 + 구체 혜택 |
| 네 가지 (헤드라인 끝) | 프롬프트 1도 몰라도 AI 직원에게 모든 업무를 위임합니다 | 약한 끝맺음 → 구체 혜택 |

## 4. 작업 절차 (스킬 본문에 포함)

1. **추출**: HTML에서 한국어 텍스트 노드만 추출 (태그/스타일/스크립트 제외)
2. **패턴 매칭**: 카탈로그 8종 패턴으로 스캔, 후보 식별
3. **맥락 분석**: 정보성(날짜/가격/FAQ)인지 호소성(헤드라인/CTA)인지 분류 — 정보성은 보존
4. **개선안 작성**: 원칙 8조 적용, before/after 쌍 생성
5. **일괄 적용**: Python replace/정규식으로 HTML 내 텍스트만 치환 (태그 속성·style 보존)
6. **검증**: 잔재 패턴 grep 카운트 0 확인 + strong/em 구조 유지 + HTML 문법(따옴표·태그 밸런스) 점검

## 5. 주의사항 (실패 사례 기반)

- **head 메타 누락**: body만 윤문하지 말고 `<title>`/description/og:description/JSON-LD description까지 동일 톤으로 (잔재 "굴러가는" 남지 않게)
- **HTML 문법 우선**: 그리드 깨짐 등은 CSS가 아니라 style 속성 따옴표 누락일 수 있음 — 카피 치환 전 HTML 구조 점검
- **정규식으로 자식 식별**: 동일 grid 패턴多处일 때 내용 키워드 lookahead(`(?=[\s\S]*?키워드)`)로 정확한 그리드 타겟
- **minmax(0,1fr)**: 코드창 등 min-content 큰 자식이 그리드 1열로 깨뜨리면 `minmax(0,1fr)`로 강제 축소
