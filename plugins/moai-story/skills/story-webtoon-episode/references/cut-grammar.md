# 세로 스크롤 컷 문법 — 높이 리듬 · 여백 띠 · 뷰포트

> `story-webtoon-episode` Step 2에서 로드한다. 세로 스크롤 웹툰은 페이지 그리드가 아니다. 컷을 "몇 칸"으로 나누는 게 아니라 **위→아래로 흐르는 리듬**으로 설계한다. 페이지 만화의 거터·왼→오 읽기·6칸 할당 전제는 여기서 쓰지 않는다.

## 핵심 원칙 — 컷 수는 감정 비트가 정한다

**"회차당 N컷"이라는 할당량은 없다.** 각 감정 비트에 맞는 컷을 배치한다.

- 정적·충격 비트 → 큰 컷 1개(뷰포트를 꽉 채우는 임팩트).
- 빠른 합·정보 밀집 → 작은 컷 여러 개.
- 할당량 사고("이 회차는 40컷")로 채우면 리듬이 죽는다.

세로 스크롤에서 리듬을 만드는 레버는 **페이지 만화의 "칸 수"가 아니라 아래 세 가지**다.

## 레버 1 — 컷 높이 변화

컷의 **세로 높이**가 감정 강도를 만든다.

| 감정 | 컷 높이 |
|------|---------|
| 강(클라이맥스·충격) | 큰 높이 — 스크롤 한 화면을 꽉 채우거나 넘김 |
| 중(갈등·전환) | 중간 높이 |
| 약(일상·이동) | 낮은 높이 — 빠르게 지나감 |
| 침묵·여운 | 큰 높이 + 넉넉한 여백(정적) |

→ 한 회차 안에 **큰 컷 ↔ 낮은 컷 대비** 필수. 다 같은 높이 = 밋밋한 스크롤. 클라이맥스 = 가장 큰 컷.

## 레버 2 — 여백 띠(칸새) 길이 = 호흡

컷과 컷 사이의 **세로 빈 띠**가 스크롤의 숨쉬기다.

- **감정 상승 구간**(빠른 합·추격): 여백 띠를 **짧게** → 컷이 촘촘히 이어져 속도감.
- **여운·전환 구간**: 여백 띠를 **길게** → 스크롤이 잠시 비어 호흡.
- **장면 전환**: 일반보다 더 긴 여백 띠 → 시간·장소가 바뀐다는 시각 신호.
- **침묵 컷**: 위아래로 넉넉한 여백 → 정적을 만든다.

## 레버 3 — 스크롤 뷰포트 1화면 설계

독자는 한 번에 **화면 1개 분량**만 본다. 뷰포트 단위로 무엇이 동시에 보일지 설계한다.

- **임팩트 컷은 뷰포트를 독점**하게 — 위아래 컷이 같은 화면에 걸치면 충격이 분산된다.
- 클라이맥스·반전 컷 앞에 여백 띠를 길게 둬서, 스크롤을 내리는 순간 그 컷만 화면에 등장하게 한다("스크롤 리빌").
- 한 뷰포트에 말풍선이 너무 많으면 읽는 부담이 커진다 — 식자 계획(`story-webtoon-lettering`)과 함께 조율.

## 비트 유형 → 컷 구성 결정표

컷 수 할당량은 없지만, **비트 유형별 기본 구성**은 있다. 페이지형의 "1~8컷 결정표"를 세로 스크롤로 각색한 표다.

| 비트 유형 | 컷 구성 | 효과 |
|-----------|---------|------|
| 충격·반전·결정적 순간 | **큰 컷 1개** (뷰포트 독점 스플래시) | 최대 임팩트·정적 |
| 강한 단일 비트 + 반응 | 큰 컷 + 반응 컷 1개 (전·후 대비) | 묵직한 호흡 |
| 대화·전개·감정 흐름 | 중간 높이 3~5컷 | 표준 서사 리듬 |
| 빠른 합·몽타주·정보 밀집 | 낮은 컷 촘촘히 (여백 짧게) | 속도감 — 과밀하면 가독성 붕괴 |

- **비트를 먼저 정하고 컷 구성을 맞춘다** — "이 구간의 비트는? → 거기 맞는 구성." 비트가 약하거나 정적이면 과감히 컷을 줄인다.
- 한 회차 안에서 구성을 **변주**한다. 같은 패턴만 반복되면 리듬이 죽는다.
- 스플래시 컷은 프롬프트에 **"한 컷으로 화면 전체를 채우는 스플래시"**로 명시한다 — 지정하지 않으면 균일 분할로 회귀한다.

## 리듬 조립 패턴

```
정지컷(낮은 높이) → 짧은 여백 → 정지컷 → [긴 여백 띠] → 임팩트컷(큰 높이, 뷰포트 독점) → [긴 여백] → 여운컷(큰 높이 + 넉넉한 여백)
```

> "정지 → 여백 → 임팩트 → 여백 → 여운"의 호흡이 세로 스크롤의 기본 문장이다. 컷 수를 세지 말고 이 호흡을 설계한다.

## 회귀 방지

- 컷 높이를 안 지정하면 AI/배치가 균일 높이로 회귀한다 → **큰 컷은 "화면을 꽉 채우는 큰 컷"으로 명시**.
- 여백 띠는 프롬프트로 정밀 픽셀 통제가 어렵다 → 방향성만 유도("이 컷 위아래로 넉넉한 여백, 정적인 호흡")하고 정밀 조정은 편집·뷰어에서.
