# 노션 템플릿 키트 (Notion Template Kit)

## 개요

노션을 깔았지만 어떻게 구조를 짜야 할지 막막한 사람을 위해, 올인원 업무관리·목표·회고 대시보드의 데이터베이스 구조와 뷰를 설계한다. 화려한 디자인이 아니라 "매일 열어서 쓰게 되는" 최소 구조를 우선한다. 실제 페이지 생성이 필요하면 Notion 커넥터로 연결한다.

## 트리거 키워드

노션 템플릿, 올인원 업무관리, 노션 대시보드, 노션 목표 관리, 노션 회고 템플릿, 노션 구조, 업무 데이터베이스, 노션 세팅, 노션 할 일

## 워크플로우

### 1단계: 용도 정하기

먼저 이 노션이 무엇을 담을지 정한다. 한 페이지에 다 넣지 않는다.

| 용도 | 핵심 데이터베이스 |
|------|------------------|
| 업무관리 | 할 일·프로젝트·노트 |
| 목표관리 | 목표·주간 행동·진척 |
| 회고·기록 | 회고·업무일지·배운 점 |
| 올인원 | 위를 한 대시보드에 링크로 묶기 |

→ 용도가 분명할수록 구조가 단순해진다. 처음엔 1~2개 DB로 시작한다.

### 2단계: 데이터베이스 속성 설계

각 DB에 들어갈 속성(필드)을 최소로 정한다.

- **할 일 DB**: 제목 · 상태(할 일/진행/완료) · 우선순위 · 마감일 · 프로젝트(관계형)
- **목표 DB**: 목표 · 기간(분기) · 핵심 결과 · 진척률
- **회고 DB**: 날짜 · 주기(주/월) · Keep/Problem/Try · 다음 행동
- → 속성이 많을수록 안 쓰게 된다. "정렬·필터에 꼭 필요한 것"만 남긴다

### 3단계: 뷰(View) 구성

같은 데이터를 상황별로 다르게 보는 뷰를 만든다.

- **오늘 뷰**: 마감일=오늘 또는 상태=진행 필터
- **이번 주 뷰**: 주 단위 필터·정렬
- **보드 뷰**: 상태별 칸반(할 일→진행→완료)
- **캘린더 뷰**: 마감일 기준
- → 입력은 한 번, 보기는 여러 방식. 뷰가 노션 활용의 핵심이다

### 4단계: 대시보드 묶기

자주 쓰는 뷰·링크를 첫 화면(홈)에 모은다.

- 상단: 오늘 할 일·이번 주 목표(연결된 DB 뷰)
- 중단: 진행 중 프로젝트, 최근 회고
- 하단: 자주 쓰는 링크·메모
- → 매일 여는 "홈"을 만들어 흩어짐을 막는다. `office-goal-planner`·`office-retro-builder` 산출물을 여기에 연결

### 5단계: 운영 규칙·실제 생성

- **입력 루틴**: 아침에 오늘 할 일 추가, 저녁에 회고 한 줄 → `office-habit-routine` 연계
- **주간 정리**: 완료 항목 아카이브, 다음 주 준비
- **실제 생성**: 구조가 확정되면 Notion 커넥터(`mcp__claude_ai_Notion__*`)로 페이지·DB 생성 가능
- → 단, 설계가 먼저다. 빈 템플릿부터 만들지 말고 용도→속성→뷰 순으로 정한다

## 사용 예시

**예시 1**: "노션으로 올인원 업무관리 대시보드 만들고 싶어"
→ 할 일·프로젝트 DB 설계 → 오늘/이번 주/보드 뷰 → 홈 대시보드 구성안

**예시 2**: "목표랑 회고를 노션에서 같이 관리하고 싶어"
→ 목표 DB + 회고 DB 속성 설계 → 분기 목표↔주간 회고 연결 → 진척 뷰

**예시 3**: "노션 깔았는데 뭐부터 만들어야 할지 모르겠어"
→ 용도 1개로 좁히기 → 최소 DB 1개 + 뷰 2개 → 매일 여는 홈 한 페이지

## 산출물

- 용도별 데이터베이스 구조
- DB 속성(필드) 설계
- 상황별 뷰 구성(오늘·주간·보드·캘린더)
- 홈 대시보드 레이아웃
- 입력·정리 운영 규칙

## 주의사항

- 처음부터 화려하게 만들면 유지가 안 된다 — 최소 구조부터 키운다.
- 속성·DB가 많을수록 입력 부담이 커진다. "꼭 필요한 것"만 둔다.
- 실제 페이지 자동 생성은 Notion 커넥터 연결이 필요하다. 미연결 시 구조 설계까지 제공한다.

## 관련 스킬

- **moai-cowork:office-goal-planner**: 목표·핵심 결과를 노션 DB로 관리
- **moai-cowork:office-retro-builder**: 회고 양식을 노션 회고 DB로 구조화
- **moai-cowork:office-habit-routine**: 습관 트래커를 노션 뷰로 시각화
- **moai-cowork:business-productivity-weekly-report**: 업무일지 DB를 주간보고 작성에 활용

## 이 스킬을 사용하지 말아야 할 때

- **팀 협업 워크스페이스·권한 설계**: 조직 단위 노션은 `moai-product`를 사용한다.
- **노션 외 도구(엑셀·구글시트) 표 생성**: 문서 산출물은 `moai-cowork` 계열로 보낸다.
- **코드·DB 스키마 설계**: 소프트웨어 데이터 모델은 `moai-core`/백엔드 도메인이 맞다.
