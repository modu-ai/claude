Claude를 쓰다 보면 가끔 빨간 오류 메시지가 뜨거나, 답변이 그럴듯한데 사실은 틀렸거나, 분명히 "메일을 보냈다"고 했는데 실제로는 아무 일도 일어나지 않는 경우를 만날 수 있습니다. 마치 똑똑하지만 가끔 깜빡하는 친구처럼, Claude도 완벽하지는 않습니다.

이 페이지는 그런 순간에 당황하지 않도록 흔한 문제를 유형별로 정리했습니다. 어떤 상황인지 알아보는 법, 그리고 직접 해볼 수 있는 해결 방법을 차근차근 안내합니다.

## 오류 메시지가 떴을 때

Claude가 화면에 오류를 표시하는 데는 몇 가지 정해진 이유가 있습니다. 메시지마다 의미와 대처법이 다릅니다.

| 오류 유형 | 무슨 뜻인가요 | 어떻게 하나요 |
|---|---|---|
| **사용량 한도 경고/차단** | 일정 시간 동안 보낼 수 있는 메시지 양을 다 썼습니다 | 한도가 풀리길 기다리거나, 요금제를 업그레이드 |
| **메시지 길이 제한** | 한 번에 보낸 내용이 너무 깁니다 | 내용을 작은 단위로 나눠서 보내기 |
| **로그인 오류** | 로그인 과정에 문제가 생겼습니다 | VPN·확장 프로그램 끄기, 캐시 정리 |
| **용량 제약** | 일시적으로 서버가 붐비는 상태입니다 | 몇 분 기다렸다가 다시 시도 |
| **서비스 장애** | 서비스 자체에 문제가 생겼습니다 | 상태 페이지에서 장애 여부 확인 |

### 사용량 한도 오류

무료 사용자는 5시간 단위의 세션 사용량 한도가 있고 한도에 도달하면 정해진 시간이 지나야 다시 초기화됩니다. 크레딧이 있는 유료 사용자는 5시간 한도에 도달한 뒤에도 계속 사용할 수 있으며 이때부터는 사용량만큼 요금이 부과됩니다.

1. 사용량 한도를 효율적으로 쓰는 방법을 점검합니다
2. 자주 한도에 걸린다면 요금제 업그레이드를 고려합니다

{{< hint type="tip" >}}
사용량을 아끼는 구체적인 방법은 [사용량 한도](/help/usage-limits/) 페이지에서 자세히 다룹니다.
{{< /hint >}}

### 메시지 길이 제한 오류

한 번에 너무 긴 내용을 보내면 길이 제한에 걸립니다. 이럴 때는 다음을 시도해 보세요.

1. 긴 내용을 여러 개의 작은 조각으로 나눠서 보냅니다
2. 핵심 부분만 요약해서 전달합니다
3. 어떤 부분이 중요한지 Claude에게 먼저 골라달라고 합니다
4. 그래도 안 되면 새 대화를 시작합니다

{{< hint type="note" >}}
코드 실행 기능을 쓰는 유료 사용자는 메시지가 자동으로 요약되어 길이 오류가 줄어듭니다. 다만 완전히 없어지는 것은 아니어서 너무 긴 내용은 여전히 나눠 보내는 게 좋습니다.
{{< /hint >}}

### 로그인 오류

로그인이 잘 안 될 때는 보통 네트워크나 브라우저 환경이 원인입니다. 아래 순서대로 하나씩 시도해 보세요.

1. VPN을 사용 중이라면 잠시 꺼봅니다
2. 켜져 있는 브라우저 확장 프로그램을 비활성화합니다
3. 브라우저의 캐시와 쿠키를 정리합니다
4. 그래도 안 되면 [status.claude.com](https://status.claude.com)에서 장애가 있는지 확인합니다

{{< hint type="note" >}}
로그인 문제는 지역이나 VPN 환경과 관련 있는 경우가 많습니다. 위 단계로 해결되지 않으면 다른 네트워크에서 시도해 보는 것도 방법입니다.
{{< /hint >}}

### 용량 제약과 서비스 장애

"용량 제약" 오류는 서비스가 망가진 게 아니라, 이용자가 몰렸을 때 부하를 관리하기 위한 일시적인 반응입니다. 보통 몇 분 안에 저절로 풀리니 잠시 기다렸다가 다시 시도하면 됩니다.

실시간 장애 상황은 [status.claude.com](https://status.claude.com)에서 언제든 확인할 수 있습니다.

## 답변이 틀렸거나 헷갈릴 때

Claude는 가끔 사실과 다른 내용을 자신 있게 말하기도 합니다. 이런 현상을 "환각(hallucination)"이라고 부르며 지금의 생성형 AI가 가진 본질적인 한계에서 비롯됩니다.

왜 이런 일이 생길까요?

- **최신 정보에 약합니다**: 학습 데이터에는 시점 한계가 있어 최근에 일어난 일이나 시의성 있는 주제에는 약합니다
- **그럴듯하게 지어냅니다**: 인용구처럼 보이는 문장이나 통계를 실제로는 만들어낸 채로 권위 있게 제시할 수 있습니다

### 부정확한 답변을 줄이는 방법

1. Claude를 유일한 정답의 출처로 삼지 않습니다. 특히 중요한 결정에 관한 조언일수록 그렇습니다
2. 웹 검색 기능을 켜서, Claude가 인용한 출처를 원본 사이트에서 직접 확인합니다
3. 잘못된 답변을 발견하면 👎(싫어요) 버튼이나 이메일로 구체적인 예시와 함께 신고합니다

{{< hint type="warning" >}}
건강, 법률, 금융처럼 결과가 큰 주제는 Claude의 답변만 믿지 말고 반드시 신뢰할 수 있는 출처나 전문가로 교차 확인하세요.
{{< /hint >}}

## 깨진 링크와 "했다"는 거짓 주장

Claude가 "이메일을 보냈습니다", "문서를 만들었습니다", "이 링크에서 확인하세요"라고 말했는데 실제로는 아무 일도 일어나지 않거나 링크가 열리지 않을 때가 있습니다.

이유는 분명합니다. Claude는 별도로 연동된 기능이 아니라면 Claude 플랫폼 바깥에서 어떤 작업도 수행할 수 없습니다. 그런데도 도움을 주려는 과정에서, 실제로는 없는 능력을 "있다"고 착각해 말해버리는 경우가 생깁니다.

Claude가 스스로는 할 수 없는 일의 예:

| 할 수 없는 일 | 설명 |
|---|---|
| **이메일 전송** | 메일 서비스에 직접 접근하지 못합니다 |
| **외부 문서 생성** | 워드 같은 외부 프로그램으로 파일을 만들지 못합니다 |
| **파일 전송** | 파일을 외부로 보내는 기능이 없습니다 |
| **작동하는 외부 링크 생성** | 실제로 열리지 않는 링크를 만들어낼 수 있습니다 |

{{< hint type="warning" >}}
Claude가 "외부 작업을 완료했다"고 자신 있게 말하더라도 그대로 믿지 마세요. 실제로 연동된 기능만 작동하므로 정말 그 기능이 켜져 있는지 직접 확인한 뒤에 의존해야 합니다.
{{< /hint >}}

## 모델이 바뀌어 낯설 때

쓰던 Claude 모델이 어느 날 종료(deprecation)되고 새 모델로 바뀌면, 익숙하던 말투나 느낌이 달라져 아쉬울 수 있습니다. 모델 교체는 유지 관리 부담과 용량 제약 때문에 정기적으로 일어나는 일이며 지금까지 Anthropic은 대체로 출시 약 1년 뒤에 모델을 종료해 왔습니다.

특정 모델의 성격이나 능력에 정이 든 사용자에게는 분명 아쉬운 변화입니다. Anthropic은 종료된 모델을 어떤 형태로든 다시 공개하는 "모델 보존"을 목표로 작업하고 있습니다.

### 새 모델에 부드럽게 적응하는 법

1. 가능한 한 빨리 새 모델을 직접 써보고 중요한 프롬프트로 이전 모델과 나란히 비교해 봅니다
2. 메모리 기능과 대화 검색을 활용해 새 모델이 당신의 취향을 빨리 익히도록 돕습니다
3. 맞춤 커뮤니케이션 스타일을 정의해 모델이 바뀌어도 일관된 말투를 유지합니다
4. Projects 기능에 맞춤 지침과 맥락을 넣어 상호작용을 개인화합니다
5. 대화·프로젝트·계정 전체 설정으로 모델의 행동을 원하는 방향으로 다듬습니다
6. 새 모델이 제공하는 향상된 성능과 더 나은 지시 이행 같은 새로운 기능을 살펴봅니다

{{< hint type="note" >}}
이 방법들이 이전 모델의 경험을 완벽히 되살려 주지는 못합니다. 다만 변화에 따른 불편을 최대한 줄여줄 수 있습니다.
{{< /hint >}}
