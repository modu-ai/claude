# 화풍 앵커 — 7토큰 분해 + 영어 앵커 블록 + 내용물 가드

> `story-webtoon-art` Step 1에서 로드한다. 시리즈 화풍을 컷마다 흔들리지 않게 고정하는 절차. 스타일 참조 이미지를 비전으로 읽어 7토큰으로 분해하고, 그것을 프롬프트에 박을 영어 STYLE 블록으로 옮긴다.

## 스타일 7토큰 분해

스타일 참조(그리드/샘플)가 주는 것은 "룩앤필"이다. **피사체(그리드 속 인물·집·풍경)는 무시하고, 공통된 "그리는 방식"만** 본다.

| 토큰 | 무엇을 본다 | 예시 |
|------|------------|------|
| 컬러 팔레트 | 주색·보조색·배경색(구체적으로) | "따뜻한 과슈 오렌지 지배 + 세이지 그린 보조 + 크림 배경" |
| 기법 | 펜·잉크/페인팅/연필/셀채색/벡터 | "거친 과슈 페인팅 + 약한 수채 워시" |
| 윤곽선 | 두께·질감 | "부드러운 손그림 윤곽, 진한 선 없음" |
| 색면 | 플랫/그라데이션/해칭음영 | "결이 보이는 플랫 색면" |
| 텍스처 | 그레인·해칭·붓결·스크린톤 | "종이 그레인 + 붓결" |
| 무드 | 전체 분위기 | "따뜻·아날로그·정감" |
| 폰트 무드 | 어울리는 레터링(식자 참고용) | "손글씨 레터링" |

### 추출 원칙

- **색은 정확히.** "다채로운" 같은 모호한 말 금지. 실제 두드러지는 색을 구체적으로.
- **기법·무드는 형용사로.** 여러 피사체가 같은 방식으로 그려져 있으니 **그리는 방식의 공통점**만 본다.

## 영어 앵커 블록 + 내용물 가드 (복붙 · 절대 생략 금지)

추출한 토큰을 영어 STYLE 블록으로 옮겨 프롬프트에 박는다. **내용물 가드 문장은 절대 생략하지 않는다** — 안 넣으면 참조 속 얼굴·집·풍경까지 베껴 그린다.

```
=== STYLE (match the ATTACHED reference image; use ONLY its palette / texture /
technique, and IGNORE its subject matter — faces, houses, landscapes, animals.
Do NOT copy any of those objects) ===
- Palette: [컬러 팔레트]
- Technique: [기법]
- Linework: [윤곽선]
- Color fill: [색면]
- Texture: [텍스처]
- Mood: [무드]
```

## 한국어 본문 + 영어 STYLE 혼용

한국어 프롬프트 본문 안에서도 이 블록만 영어로 박으면 화풍이 더 또렷이 잡힌다. 한국어로 "흑백 펜·잉크 화풍, 거친 해칭…"을 같이 적어줘도 좋다 — 둘 다 OK.

## 첨부 경고 배너 (산출물 맨 앞)

스타일 앵커는 그 이미지를 **함께 첨부**해야 작동한다. 안 올리면 무용지물 → 산출물 맨 위에 경고.

```
⚠️ 이 프롬프트는 [스타일 이름] 참조 이미지를 함께 첨부해야 작동합니다.
   첨부 순서: ① 캐릭터 ref → ② 스타일 그리드 → (연속 시) ③ 직전 컷 결과
```

## "웹툰 프레이밍 함정" — 스타일이 약하게 묻을 때만

"웹툰/comic"이라고만 하면 학습 관성으로 기본 웹툰체로 회귀해 화풍이 묻힐 수 있다. **결과를 보고 약할 때만** 추가한다(잘 묻으면 불필요):

- 프레이밍 역전: `a hand-painted comic page in the ATTACHED painterly style, NOT a clean flat digital webtoon.`
- 금지문: `Do NOT default to glossy flat webtoon shading; keep the brushy hand-painted texture of the reference.`

## 레지스트리 경계

아트 스타일 토큰 레지스트리(화풍·채색·선화·톤)는 `moai-designer:design-system-library`의 `styles/`가 소유한다. 이 스킬은 **레지스트리에 토큰을 두지 않고**, 위 추출 절차로 **시리즈별 화풍 앵커(7토큰 인스턴스)만** 보유한다.
