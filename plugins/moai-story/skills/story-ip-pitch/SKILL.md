# story-ip-pitch: IP 사업화

완성작을 **2차 저작 시장에 피칭**하는 문서를 만드는 스킬이다. 원작 가치·확장 경로·제안 조건을 한 제안서로 정리하고, 한국 계약 실무(표준계약서·저작권 유보·등록) 구조를 반영해 협상 문서로 효력을 갖게 한다.

## 1. 개요

IP 피칭은 **무엇을 제안할지(2차 저작 형태)**와 **누구에게 어떤 조건으로 접근할지(제작사·플랫폼·에이전시)**를 설계한다. 원작 데이터(조회수·완독률·팬덤)가 협상력의 기준이다. 계약 조항의 법적 해석은 이 스킬의 범위가 아니다.

## 2. 이 스킬이 하지 않는 것

- **법률 자문 안 함** — 이 스킬은 계약 **구조 개요**만 제공한다. 조항 검토·협상은 `moai-lawyer` + 전문가 확인.
- **작품 창작 안 함** — 완성된 원작(story-*/book-* 산출물)이 전제.
- **수치 단정 안 함** — 로열티·옵션료·기간은 계약 사안이다. `[미확인 — 계약/시장 확인]`로 둔다.

## 3. 사전 확인

- `references/expansion-paths.md` — 원작→2차 저작 경로별 적합도·난이도·선행 산출물 요구.
- `references/ip-contract-kr.md` — 문체부 표준계약서 유형·2차적저작물작성권 유보·한국저작권위원회 등록·로열티/옵션 협상 체크리스트(법률 자문 아님).

## 4. 워크플로우

### Step 1 — IP 현황 정리

원작 형태·실적(조회수·부수·완독률·팬덤 규모)·핵심 자산(캐릭터·세계관)을 정리한다. 실적 수치는 실제 데이터로만 채운다(지어내지 않음).

### Step 2 — 확장 후보 선정

2차 저작 경로별 적합도·난이도·선행 산출물 요구를 `expansion-paths.md`로 평가해 우선순위를 매긴다.

### Step 3 — 확장 시나리오 + 제안서 작성

- IP 요약(로그라인·핵심 자산).
- **확장 시나리오**(2차 저작 형태·타깃 제작사·근거).
- 협상 조건 개요(판권 범위·기간·로열티) — 구체 수치는 `[미확인]`, 심화는 `moai-lawyer`.

### Step 4 — 계약 구조 점검

`ip-contract-kr.md`로 문체부 표준계약서 유형·2차적저작물작성권 유보 여부·저작권 등록 필요성을 점검한다. **법적 판단은 `moai-lawyer`로 넘긴다.**

### Step 5 — 타깃 매핑

제작사·플랫폼·에이전시 후보를 우선순위화한다.

## 5. 출력 형식

```markdown
# [작품] IP 피칭 제안서
> 원작 형태 / 실적(실데이터) / 핵심 자산

## IP 요약
- 로그라인 / 캐릭터·세계관 자산

## 확장 시나리오
| 경로 | 적합도 | 난이도 | 선행 산출물 | 타깃 |
|------|--------|--------|-------------|------|

## 협상 조건 (개요)
- 판권 범위 / 기간 / 로열티 [미확인 — moai-lawyer 확인]
- 2차적저작물작성권 유보 여부: [점검 필요]

## 타깃 목록
- 제작사·플랫폼·에이전시 (우선순위)
```

## 6. 주의사항

- **AI 생성물 표시 의무 확인** — AI 생성 이미지·표지를 상업 플랫폼·제안서에 사용할 때 AI 생성물 표시 의무를 확인한다(상세는 `moai-lawyer`의 AI 관련 체크리스트 참조).
- **법률 자문 아님** — 계약 조항 해석·서명 판단은 반드시 `moai-lawyer` + 전문가.
- **2차적저작물작성권 유보는 되돌리기 어렵다** — 계약 서명 전 유보 범위 확인.
- **실적 수치는 실데이터만** — 조회수·부수·팬덤 규모를 부풀리거나 지어내지 않는다.
