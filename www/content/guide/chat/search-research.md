같은 "찾아봐 줘"라는 부탁이라도, 오늘 날씨를 확인하는 것과 경쟁사 시장 동향을 며칠치 자료로 정리하는 건 무게가 다릅니다. Claude에는 이렇게 서로 다른 상황에 맞춰 쓰는 세 가지 도구가 있습니다. 바로 **웹 검색**, **확장 사고**, **리서치**입니다.

쉽게 비유하면, 웹 검색은 "잠깐 인터넷을 켜서 확인하기", 확장 사고는 "골똘히 오래 생각하기", 리서치는 "여러 자료를 뒤져서 보고서를 써오기"에 가깝습니다. 어떤 상황에 어떤 도구가 어울리는지 알면, 더 빠르게 또는 더 깊게 원하는 답을 받을 수 있습니다.

## 세 가지 도구 한눈에 비교하기

| 도구 | 가장 잘하는 일 | 이런 질문에 |
|---|---|---|
| **웹 검색** | 빠르게 사실 하나 확인 | "오늘 서울 날씨는?", "이 회사 최근 뉴스 알려줘" |
| **확장 사고** | 인터넷 없이 깊게 추론 | 수학 문제 풀이, 코드 디버깅, 복잡한 분석 |
| **리서치** | 여러 출처를 모아 종합 보고 | 경쟁사 분석, 오래된 문서 업데이트 |

{{< hint type="note" >}}
세 도구는 서로 배타적이지 않습니다. 확장 사고와 리서치를 함께 쓰면, 신중하게 계획을 세우면서 동시에 폭넓은 자료를 모을 수 있어 신기술 사업 제안서나 다양한 출처가 필요한 과학 분석 같은 작업에 잘 맞습니다.
{{< /hint >}}

## 웹 검색 — 최신 정보를 바로 확인하기

웹 검색을 켜면 Claude가 실시간으로 인터넷을 검색해, 살아 있는 웹 정보를 바탕으로 답변합니다. 답변에는 출처 링크와 인용이 함께 표시되어 직접 확인할 수 있습니다.

### 웹 검색을 켜는 법

1. 채팅 입력창에서 **슬라이더(아이콘)** 버튼을 클릭합니다.
2. 드롭다운 메뉴에서 **웹 검색(Web search)** 항목을 찾습니다.
3. 토글을 켜서 활성화합니다.

### 웹 검색이 잘하는 일

- 날씨, 회사 정보, 최근 뉴스 헤드라인처럼 **시의성 있는 사실** 확인
- **웹 페치(web fetch)** — 특정 URL을 직접 불러와 내용 가져오기
- **이미지 검색**(Bing 기반)과 인터랙티브 콘텐츠 표시
- IP 주소 기반 위치 추론으로 지역에 맞는 답변

### 지원 모델과 요금제

| 항목 | 내용 |
|---|---|
| 지원 모델 | Fable 5, Opus 4.8 / 4.7 / 4.6, Sonnet 4.6, Haiku 4.5 |
| 무료 계정 | 웹 검색에 **하루 사용량 제한**이 있음 |
| Team/Enterprise | 소유자(Owner) 또는 기본 소유자가 관리자 설정 > Capabilities에서 활성화 필요 |

{{< hint type="warning" >}}
무료 계정에서는 URL을 직접 분석하면 대화의 컨텍스트(맥락) 용량을 많이 소모합니다. 또한 검색 시간은 질문 복잡도에 따라 달라지고, 웹사이트 링크가 가끔 동작하지 않을 수 있으며, 사용량은 일일 한도에 포함됩니다.
{{< /hint >}}

## 확장 사고 — 인터넷 없이 깊게 생각하기

확장 사고는 최신 웹 정보가 필요 없는 대신 **복잡한 추론**이 필요한 작업에 적합합니다. Claude가 결론에 이르기 전에 여러 각도에서 문제를 탐색하도록 해줍니다.

이런 작업에 잘 어울립니다.

- 수학 문제 풀이
- 코드 디버깅
- 철학적 분석
- 아키텍처(구조) 최적화 같은 다각도 검토

## 리서치 — 여러 출처를 모아 보고서로

리서치는 Claude가 스스로 여러 검색을 이어가며 질문을 다양한 각도에서 깊이 파고드는 기능입니다. 보통 **5번 이상의 도구 호출**을 1~3분에 걸쳐 수행하면서, 검증 가능한 출처 인용이 담긴 심층 보고서를 만들어 줍니다.

리서치는 웹과 연결된 내부 출처(Gmail, Google 캘린더, Google 문서 등 Workspace 커넥터)를 **동시에** 검색해 답을 구성합니다.

### 리서치를 켜는 법

1. 채팅 입력창 왼쪽 아래의 **'+' 버튼**을 클릭합니다.
2. 메뉴에서 **리서치(Research)**를 선택합니다.
3. 창 하단에 **파란색 표시**가 나타나는지 확인합니다.
4. 질문하면 Claude가 조사를 진행합니다.
5. 끄려면 '+' 버튼을 다시 클릭합니다.

### 지원 환경과 조건

| 항목 | 내용 |
|---|---|
| 요금제 | 유료 플랜만 지원 (Pro, Max, Team, Enterprise) |
| 플랫폼 | 웹, Claude Desktop, Claude 모바일 |
| 사전 조건 | **웹 검색이 켜져 있어야** 리서치가 작동함 |

### 리서치가 잘하는 일

- 경쟁사 분석
- 오래된 문서를 최신 내용으로 업데이트
- 캘린더/이메일 우선순위 정리

{{< hint type="tip" >}}
리서치가 바로 작동하지 않으면 "리서치 도구를 사용해 줘(please use the research tool)"처럼 명시적으로 요청해 보세요. 특정 내부 자료를 끌어와야 할 때는 "[출처]에서 관련 맥락을 가져와 줘"처럼 직접 지정하면 도움이 됩니다.
{{< /hint >}}

{{< hint type="warning" >}}
리서치는 여러 출처를 가져오기 때문에 사용량 한도를 **더 빠르게 소모**합니다. 또한 일반 대화 한도의 적용을 받으며, 무료 플랜에서는 사용할 수 없습니다.
{{< /hint >}}

## 어떤 도구를 골라야 할까

상황별로 정리하면 다음과 같습니다.

1. **단순한 사실 하나만 빠르게** 필요하다 → 웹 검색
2. **인터넷 없이도 풀 수 있는 어려운 문제**를 깊게 생각해야 한다 → 확장 사고
3. **여러 자료를 모아 종합한 보고서**가 필요하다 → 리서치
4. **계획도 세우면서 폭넓은 조사**까지 필요하다 → 확장 사고 + 리서치 조합
