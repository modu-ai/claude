# 한국 기업 공개 디자인시스템 + 한글 폰트 서베이

디자인시스템·팔레트·한글 서체를 고를 때 **"실제로 가져다 쓸 수 있는가"**를 기준으로 정리한 참고 자료입니다. 한국 기업의 "디자인시스템"은 대부분 사내 전용이며, 코드/토큰/Figma까지 공개해 제3자가 재사용 가능한 곳은 소수입니다. **라이선스 함정**(브랜드 자산·토스 전용 서체)이 특히 많아 별도 경고 섹션을 둡니다.

---

## 1. 재사용 가능 (오픈소스, 라이선스 클린)

| 시스템 | 라이선스 | 무엇을 쓰나 |
|---|---|---|
| **당근 SEED** (`github.com/daangn/seed-design`) | Apache-2.0 | **역할 기반 색상(Role Based Color) + 팔레트 색상 2단 구조** — 라이트/다크 적응·접근성 대비 표준. 슬라이드 팔레트 설계 **1차 레퍼런스**. Figma 라이브러리까지 개방 |
| **채널톡 Bezier** | react Apache-2.0 / compose·swift MIT | 세 플랫폼(Web/Android/iOS) 모두 오픈소스. npm/CocoaPods/Gradle로 바로 설치. 컴포넌트 라이브러리 — 색·아이콘은 코드에서 추출 |
| **원티드 Montage** (`github.com/wanteddev`) | 오픈소스 표방 (**사용 전 LICENSE 재확인**) | Figma Community UI Kit + TypeScript 저장소. Design Principles·컴포넌트 문서 체계적 |
| **정부 KRDS** (`github.com/KRDS-uiux/krds-uiux`) | 공공 (공공누리 유사, 사이트 재확인) | **WCAG 색 대비 기준·한글 타이포그래피 스펙이 매우 상세**. React/Vue Storybook. 한국어 프레젠테이션·공공성 UI 참고에 유용 |

> 당근 SEED의 "역할 기반 색상 + 팔레트 색상" 2단 구조는 우리 덱의 색상 역할 체계·라이트/다크 토큰 네이밍을 설계할 때 가장 먼저 참고할 대상입니다(문서화 품질 국내 최고 수준).

## 2. 참고만 (문서·스크린샷 인용, 자산 재사용 불가)

코드/토큰/Figma가 공개되지 않아 **원칙·스크린샷 인용만** 가능한 사내 시스템입니다. 재배포·재가공 불가로 간주합니다.

- **LINE LDS** (`designsystem.line.me`) — 퍼블릭 문서 사이트(Foundation·UX·Voice&Tone)는 상세하나 다운로드 가능한 코드/Figma/토큰 없음. 저작권 LINE 보유.
- **카카오** — 통합 디자인시스템 사이트 없음(기술블로그 회고만). 폰트만 오픈소스.
- **네이버 deFign** — 블로그·D2 발표로만 소개. 폰트만 공개.
- **배민(우아한공방)** — 우아콘 발표로만. 폰트 11종만 공개.
- **뱅크샐러드 BPL / 클래스101 Vibrant** — 사실상 비공개(블로그·랜딩만).

> 이들은 "국내 디자인시스템 사례 소개" 슬라이드의 텍스트/스크린샷 인용 소스로만 쓰고, 각 이미지 저작권은 각사 소유이므로 출처를 표기합니다.

## 3. 무료 상업용 한글 폰트

| 폰트 | 라이선스 | 톤·용도 |
|---|---|---|
| **Pretendard** (`github.com/orioncactus/pretendard`) | SIL OFL 1.1 | **사실상 표준 — 기본값 권장**. Thin~Black 풀 웨이트, 웹폰트 CDN. 상업 사용·수정·재배포 자유(폰트 단독 판매만 금지) |
| **배민 11종** (한나·주아·도현·연성·을지로체 등) | OFL 계열 | 캐주얼·친근한 톤(F&B·로컬 커머스·캐주얼 브랜드) |
| **나눔글꼴 / 마루 부리** (네이버) | OFL 계열 | 나눔고딕·나눔명조·마루부리(세리프). 보조·본문 |
| **Kakao Big / Small Sans** (`github.com/kakao/kakao-font`) | OFL-1.1 | 두 종. 상업 사용·수정·재배포 자유 |

> 모두 "폰트 자체의 유료 판매"만 금지이고 상업 프레젠테이션·브랜딩에 바로 사용 가능합니다. 기본은 Pretendard, 캐주얼 무드가 필요하면 배민 폰트를 보조로 매핑합니다.

## 4. ⚠️ LICENSE 함정 경고 (반드시 확인)

아래는 "공개돼 있다"는 사실만 보고 재사용하면 **명백한 라이선스 위반**이 되는 자산입니다.

- **Toss Product Sans** = 다운로드 경로 자체가 없음. 산돌 "브랜드 전용 폰트"로 토스 독점. **상업 이용(제3자) 불가**. 출처 불명 3rd-party 배포본을 쓰면 저작권 분쟁 리스크.
- **Toss TDS (Figma UI Kit `.fig` · `@toss/tds-react-native` npm)** = **"앱인토스(Apps in Toss)" 미니앱 개발 전용**. 다른 프로젝트 사용·상업 판매·복사/수정/재가공·재배포 전부 **명시적 금지**. 일반 프로젝트용으로 뜯어 쓰면 위반.
- **TossFace(토스페이스)** = 무료 상업 사용은 가능하나 **OFL이 아닌 커스텀 라이선스**. **수정·2차적저작물(파생 폰트) 제작 금지**, IP 등록·공모전 출품·소스코드 변환 금지. "있는 그대로" CDN으로 불러 쓰는 것만 허용(자체 아이콘 세트로 가공·재배포 금지). 커뮤니티에 "OFL이다"라는 오인 게시물 있으나 사실 아님.
- **카카오·토스 브랜드 로고/CI** (brand.toss.im, kakaobank 브랜드 리소스 등) = "브랜드 표기용"이며 명시적 오픈 라이선스 아님. **변형 금지**. 그 회사 사례를 **인용하는 슬라이드**에서 로고·컬러 값을 정확히 재현하는 용도로만 쓰고, 우리 자체 브랜드 자산으로 변형·재사용하지 않는다.

> 브랜드 자산(로고·CI)은 자유 이용 자산으로 오인해서는 안 됩니다 — 각사 사용 규정(변형 금지·배경색 규정)을 반드시 지킵니다.
