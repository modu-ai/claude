# 로그라인 패턴

> 로그라인은 작품을 한 문장으로 압축한 것이다. 제작사·심사자가 가장 먼저 읽는 줄이며, 여기서 후크가 없으면 다음 장을 넘기지 않는다. 아래 패턴은 출발점이지 틀이 아니다.

## 1. 기본 골격

로그라인은 대개 네 요소를 담는다: **주인공 + 상황(계기) + 목적 + 장애(위기)**.

> "[어떤 처지의] 주인공이 [계기]로 [목적]을 좇지만 [장애]에 부딪힌다."

## 2. 패턴 유형

| 패턴 | 형태 | 적합 장르 |
|------|------|-----------|
| **변신형** | 평범한 X가 Y를 계기로 전혀 다른 존재가 된다 | 성장·판타지 |
| **잠입형** | 정체를 숨긴 주인공이 적진/이질적 세계에 들어간다 | 스릴러·로맨스 |
| **시한형** | 제한 시간 안에 무언가를 막아야 한다 | 재난·범죄 |
| **관계형** | 어울릴 수 없는 두 사람이 얽힌다 | 로맨스·버디 |
| **복수형** | 잃은 자가 되갚으려 한다 | 액션·누아르 |
| **미스터리형** | 하나의 의문이 세계를 뒤집는다 | 미스터리·추리 |

## 3. 장르별 분기

- **드라마**: 감정·관계 중심 → 인물의 내적 목적을 로그라인에 실어라.
- **영화**: 하나의 강한 사건축 → 시각적·행동적 갈등을 앞세워라.
- **웹드라마**: 짧은 회차·즉각 후크 → 초반 궁금증을 로그라인에 노출.

## 4. 나쁜 예 / 좋은 예

| 나쁜 예 | 문제 | 좋은 예 |
|---------|------|---------|
| "한 남자의 인생과 사랑에 관한 이야기" | 모호·나열, 갈등 없음 | "은퇴한 형사가 딸의 실종 단서를 쫓다 자신이 묻은 과거와 마주한다." |
| "판타지 세계에서 벌어지는 대모험" | 주인공·목적·위기 없음 | "마법을 잃은 궁정 마법사가 왕국을 구할 유일한 방법이 자신을 파문한 스승과 손잡는 것임을 알게 된다." |

## 5. 점검 4문

1. 주인공이 누구이고 무엇을 원하는가?
2. 무엇이 그를 막는가(위기)?
3. 왜 지금 이 이야기인가(시의성·차별)?
4. 한 문장으로 읽었을 때 다음 장이 궁금한가?
