# router.md — 자연어 → 2-모드 라우팅 프로토콜 (12-plugin 패밀리)

## 개요
사용자의 자연어 요청을 분석하여 두 진입 모드 — `--cowork`(code를 제외한 모든 코워크 업무, 11개 직원) / `--code`(코더 개발환경) — 중 하나로 라우팅하고, `--cowork` 안에서는 **어느 AI 직원(플러그인)의 스킬 체인이 중심이 될지**를 결정하는 프로토콜. 단일 'MoAI-Claude, 모두의 클로드'(`modu-ai/claude`) 마켓플레이스의 12-plugin 패밀리가 대상이다. 라우팅된 플러그인의 `skills/` 안 SKILL.md가 곧 실행 단위다.

---

## 1. 라우팅 흐름

```
사용자 자연어 요청 (/project 또는 bare 진입)
    ↓
[1단계] 플래그 확인
    ├── --cowork / --code 명시 → 해당 모드 즉시 진입
    ├── (폐기 별칭) --designer 등 개별 직원 플래그 → --cowork로 처리하되 해당 직원 중심 설계
    └── 플래그 없음 → [2단계]
    ↓
[2단계] 프로젝트 컨텍스트 확인
    ├── CLAUDE.md의 "워크플로우 체인" / .moai/config 정의 산출물 체인 있음
    │       → 해당 체인으로 직접 실행
    └── 없음 → [3단계]
    ↓
[3단계] 모드 판정: 개발 키워드면 --code, 그 외 전부 --cowork
    ↓
[4단계] (--cowork일 때) 직원 키워드 매핑 → 중심 직원·스킬 체인 결정
    ├── 매칭 1개 직원 → 해당 직원 중심 설계
    ├── 매칭 2개+ → §3 모호성 해소 / §4 복합 요청
    └── 매칭 0개 → §5 폴백
```

## 3. 모호성 해소

키워드 매칭 결과 중심 직원 후보가 2개 이상일 때:

### 3-1 자동 해소
- 산출물 유형이 명시되면 해당 산출물을 만드는 직원 우선
- 예: "브랜드 카드뉴스 만들어줘" → 콘텐츠 산출물 명시 → 마케터 중심 (`content-card-news`)
- 예: "디자인 시스템 기반 랜딩 페이지" → 디자이너(시스템 셋업) 후 마케터(랜딩 체인) 순차

### 3-2 사용자 확인
자동 해소가 어려우면 AskUserQuestion(1질문, 후보 직원 최대 4개 + Other). 모드 자체(--cowork/--code)가 불명확할 때만 2-모드 질문을 한다.

## 4. 복합 요청 처리 (크로스 직원)

`--cowork` 셋업은 원래 다중 직원 워크플로우를 전제로 한다. 요청이 여러 직원에 걸치면 **하나의 CLAUDE.md 워크플로우 표에 직원별 체인을 함께 설계**하고, 반복 작업 유형별 커스텀 에이전트(`.claude/agents/`)가 체인을 호출하도록 배선한다.

```
"스타트업이라 사업계획서랑 웹사이트 디자인 둘 다 필요해"
→ /project --cowork 1회 셋업:
   워크플로우 표 = 코워커(사업계획서 체인) + 디자이너(브랜드 셋업 체인)
   커스텀 에이전트 = biz-plan-writer, brand-system-keeper
```

개발이 섞이면 (`"서비스 개발도 해야 해"`) → `--code`는 별도 진입으로 안내한다 (코더 스캐폴드는 cowork 셋업과 독립).

### 4-1 심층 분석 판단
복합 요청이거나 3개+ 직원 체인이 걸치면 `ultrathink`(심층 분석)로 최적 설계를 결정한다.

## 6. 검증 깊이 연동

라우팅과 함께 **검증 깊이 등급**도 정한다 (상세: `execution-protocol.md`):
- 법률·계약·NDA·개인정보·세무·재무·정부지원·의료·규제 키워드, 또는 2개+ 직원 체인 → **DEEP** (근거 게이트 + 면책 + 전달 전 확인)
- 일반 산출물(보고서·카피·문서) → **NORMAL** (근거 게이트 활성)
- 단순 조회·요약 → **QUICK** (근거 게이트 스킵)

키워드는 seed일 뿐이며, 트리거 단어 없이 표현된 고위험 요청도 맥락으로 DEEP 판단한다.

## 7. 버전 정보

**2-모드(--cowork/--code) / 12-plugin / 단일 modu-ai/claude 마켓플레이스 라우팅** (v0.4.0). 모드 판정 → (--cowork일 때) 중심 직원 매핑 → 셋업 프로토콜 진입 + 커스텀 에이전트 배선. 별도의 실행 모듈 계층은 없다.
