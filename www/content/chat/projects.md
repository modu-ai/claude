서류가 책상 여기저기에 흩어져 있으면 필요한 자료를 찾는 데만 한참이 걸리죠. 주제별로 폴더를 만들어 관련 대화와 파일을 한곳에 모아 두면 훨씬 편합니다. Claude의 **Projects**가 바로 그런 "주제별 작업 공간"입니다.

Projects는 각각 자체 대화 기록과 지식 베이스를 가진 독립된 공간입니다. 문서·텍스트·코드 같은 파일을 올려 두면 Claude가 그 자료를 바탕으로 대답하고, 프로젝트마다 별도의 지침을 정해 Claude의 말투나 관점도 맞춤 설정할 수 있습니다. 무료 계정을 포함한 모든 사용자가 사용할 수 있습니다.

## Projects란? — "주제별 폴더" 같은 개념

Projects는 Claude와의 여러 대화와 파일을 한 주제 아래 모아서 관리하는 기능입니다. 각 프로젝트는 자기만의 대화 기록과 지식 베이스를 가지므로, 한 주제에 집중된 환경에서 대화를 이어갈 수 있습니다.

### 예시

- **프로젝트 1**: "마케팅 캠페인 Q2"
  - 대화 1: 아이디어 브레인스토밍
  - 대화 2: 예산 계획
  - 파일: 분석 자료, 제안서

- **프로젝트 2**: "계약서 검토"
  - 대화 1: 계약서 분석
  - 대화 2: 협상 전략
  - 파일: 계약서, 유사 사례

{{< hint type="note" >}}
무료 계정은 최대 5개의 프로젝트를 만들 수 있습니다. 유료 플랜에는 별도의 개수 제한이 명시되어 있지 않습니다.
{{< /hint >}}

## Project 만드는 법

1. Claude 사이드바 상단의 **Create New Project** 클릭
2. 프로젝트 이름 입력 (예: "분기별 예산 계획")
3. 선택사항: 프로젝트 설명 추가
4. **Create** 클릭
5. 이 프로젝트 안에서 대화 시작

## Project의 이점

| 이점 | 설명 |
|---|---|
| **문맥 유지** | 같은 Project 내 대화가 별도 기록으로 관리되어 주제별로 이어 작업 |
| **지식 베이스** | 문서·텍스트·코드 등 파일을 올려 Claude가 참고하도록 구성 |
| **맞춤 지침** | 프로젝트별 지침으로 Claude의 말투·관점을 조정 |
| **재사용** | 이전 대화를 쉽게 찾아 이어서 작업 |
| **팀 공유** (Team·Enterprise) | 같은 Project를 여러 사람이 함께 사용 |

## 프로젝트 지침 설정하기

프로젝트마다 **지침(instructions)** 을 정해 두면, 그 프로젝트 안의 모든 대화에 일관된 규칙을 적용할 수 있습니다. 매번 같은 설명을 반복하지 않아도 되니 편리합니다.

```
프로젝트 지침 예시:
"너는 우리 회사의 콘텐츠 담당자야.
모든 답변은 친근한 존댓말로, 전문 용어는 쉽게 풀어서 설명해 줘.
브랜드 이름은 항상 'OO'로 표기해."
```

이렇게 지침을 정해 두면 Claude가 해당 프로젝트에서 늘 그 말투와 관점으로 대답합니다.

## 대용량 지식 활용 — RAG

프로젝트에 자료를 많이 올리다 보면 한 번에 처리할 수 있는 분량을 넘어설 수 있습니다. 이때 **RAG(검색 증강 생성, Retrieval Augmented Generation)** 기술이 자동으로 작동합니다.

RAG는 올려 둔 문서 전체를 매번 통째로 읽는 대신, 질문과 관련된 부분만 똑똑하게 골라서 가져옵니다. 덕분에 표준 한도보다 **최대 10배** 많은 지식을 담아 두고도 답변의 품질과 속도를 그대로 유지할 수 있습니다.

### RAG의 특징

| 항목 | 내용 |
|---|---|
| **자동 작동** | 프로젝트 지식이 표준 한도를 넘으면 별도 설정 없이 켜짐 |
| **용량 확장** | 표준 한도 대비 최대 10배까지 지식 수용 |
| **품질 유지** | 정확도와 응답 속도가 일반 방식과 동일 |
| **도구 호환** | 웹 검색, 확장된 사고, Research 등 모든 Claude 도구와 함께 작동 |
| **기존 프로젝트 적용** | 이미 만든 프로젝트도 지식이 한도를 넘으면 자동 혜택 |

{{< hint type="note" >}}
RAG는 프로젝트 지식이 표준 한도를 넘어설 때만 작동합니다. 한도 안이라면 평소처럼 자료 전체를 직접 참고합니다. RAG 기능은 Free·Pro·Max·Team·Enterprise 등 모든 구독 등급에서 사용할 수 있습니다.
{{< /hint >}}

## 프로젝트 공유와 권한 (Team·Enterprise)

Team과 Enterprise 플랜에서는 프로젝트를 다른 구성원과 함께 사용할 수 있습니다. 특정 멤버에게만 공유하거나 조직 전체에 공개할 수 있고, 공유 탭·이메일 알림·접근 관리 같은 협업 기능을 제공합니다.

권한은 두 단계로 나뉩니다.

| 권한 | 할 수 있는 일 |
|---|---|
| **Can use** (사용) | 프로젝트를 보고 대화만 가능 |
| **Can edit** (편집) | 프로젝트를 수정하고 자료를 기여 가능 |

{{< hint type="warning" >}}
조직 전체 공유는 소유자(Owner) 또는 주 소유자(Primary Owner)가 비활성화할 수 있습니다. 회사 정책에 따라 전체 공개 옵션이 보이지 않을 수 있습니다.
{{< /hint >}}

## 3가지 실제 사용 사례

### 사용 사례 1: 업무 프로젝트

**프로젝트 이름**: "신제품 런칭 2026"

```
대화 1: "신제품의 주요 기능 3개와 가치 제안을 정리해 줄 수 있나요?"
대화 2: "위에서 정한 기능을 바탕으로 마케팅 캠페인 개요를 만들어 줄 수 있나요?"
대화 3: "위 캠페인에 맞는 SNS 포스트 5개를 작성해 줄 수 있나요?"

파일: 제품 스펙 → 마케팅 전략 → 최종 포스트
```

### 사용 사례 2: 학습 주제

**프로젝트 이름**: "Python 배우기"

```
대화 1: "Python 기초를 알려 줄 수 있나요?"
대화 2: "리스트와 딕셔너리의 차이를 설명해 줄 수 있나요?"
대화 3: "간단한 데이터 분석 코드를 보여 줄 수 있나요?"

파일: 학습 자료, 코드 예시, 연습 문제
```

### 사용 사례 3: 글쓰기 프로젝트

**프로젝트 이름**: "사업계획서 작성"

```
대화 1: "사업계획서의 구조를 알려 줄 수 있나요?"
대화 2: "Executive Summary를 작성해 줄 수 있나요?"
대화 3: "시장 분석 섹션을 보강해 줄 수 있나요?"

파일: 시장 데이터, 작성 중인 문서, 참고 자료
```
